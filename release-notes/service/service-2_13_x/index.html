Service 2.13.0​

January 22, 2025

Release of new features​

It provides a guide to the new features released in Service 2.13.0.

Database​

New Launched the IBM DB2 monitoring product.

Note

For more information about the IBM DB2 monitoring product, see the following.

Server​

  • New Added the Metrics Explorer menu that can analyze the past data more intuitively.

    • Added categories in the Server Inventory menu.

    • Added the modal to guide the Metrics Explorer feature.

    Note

    This feature can be seen in the Laboratory > Metrics Explorer path. See Next document for more information.

Kubernetes​

  • New Added the kube-scheduler Dashboard menu to view monitoring data for kube-scheduler, one of the system components of Kubernetes.

    Note

    • This feature can be seen in the Cluster > kube-scheduler Dashboard path. It is supported in Kubernetes agent 1.8.1 or later.

    • For more information about the kube-scheduler Dashboard menu, see the following.

  • New Added the Container Timeline menu that can display the container status over time.

    Note

    This feature can be seen in the Container > Container Timeline path. For more information, see the following.

  • New Added the Pod Timeline menu that can display the Pod status over time.

    Note

    This feature can be seen in the Workload > Pod Timeline path. For more information, see the following.

  • New Added a new Persistent Volume-related menu under Cluster.

    • Cluster > Persistent Volume List

    • Cluster > Persistent Volume Claim List

    Note

    • This feature is supported in Kubernetes agent 1.8.1 or later.

    • For more information about the Persistent Volume List and Persistent Volume Claim List menus, see the following.

Common​

  • Feature In Analysis > Metrics Chart New, improved the menu features.

    • Added a specific time when the Max value of the hint data is recorded upon mouse over in the chart data.

    • Added the Select/Unselect button to the entire legend list.

    • Added the feature to select colors for chart lines.

    • Added the Copy CSV button to the More menu.

    • Added the Copy chart image button to the More menu.

    Note

    For more information about Metrics Chart New, see the following.

  • Feature In Alert > Notifications, added the feature to select the notification level via a third-party plugin.

  • Feature Added the Alert > Event History menu and changed and added the real-time notification link.

    Note

    As in the applied Server and Database products, the links to the notification-related menus for Application, Kubernetes, Browser, and URL products.

  • Changed Replaced the time selector feature in the Integrated Report menu.

    • Daily report: You can select the start/end date and time.

    • Weekly report: Simply select a start date. Then the end date is selected automatically after calculation.

    • Monthly report: Simply select a start date. Then the end date is selected automatically after calculation.

      • If the start date is the 1st date, up to the end of the month is automatically selected.

      • If the start date is not the 1st date, up to the previous date of the next month is automatically selected.

    Note

    Example

    • If the start date is January 1, from January 1 00:00:00 to January 31 23:59:59 is automatically selected.

    • If the start date is January 10, from January 10 00:00:00 to February 9 23:59:59 is automatically selected.

  • Fixed Fixed the issue where the View resolved events switch UI does not appear when there is no data in the notification list widget of the Integrated Flex Board or Flex Board menu.

  • Fixed In Usage > Balance, fixed the issue where the list does not appear.

  • Fixed Modified the order of product icons in the search results for the project list.

Application​

  • Feature In Dashboard > Transaction Map, added the feature to adjust the maximum value of the Y-axis on the chart by directly entering it.

  • Changed In Dashboard > Transaction Map, when adjusting the maximum Y-axis value with the or button, improved the feature so that adjustments can be made only in 10-second intervals if the value exceeds 30 seconds.

    • The segment from 10 to 50 seconds is moved at 10 second intervals.

    • The segment from 50 to 300 seconds is moved at 50 second intervals.

    • For the segment of 300 or more seconds, it is moved at 100 second intervals.

    Note

    For more information about the Transaction Map menu, see the following.

  • Feature In Statistics > Transaction, added the Avg HTTP call time, Avg SQL time, and Avg SQL fetch time columns calculated based on the number of transactions.

    Note

    For the changed items, see the following.

    Changing the column name

    • Avg HTTP call time → Average time per HTTP call: Total HTTP call time / Total number of HTTP calls in transactions

    • Avg SQL time → Average time per SQL: Total SQL time / Total number of SQLs in transactions

    • Avg SQL fetch time → Average time per SQL patch: Total SQL patch time / Total number of SQL patches in transactions

    Newly added average value columns

    • Avg HTTP call time: Total HTTP call time / Number of transactions

    • Avg SQL time: 총 SQL 시간 / 트랜잭션 발생 건수

    • Avg SQL fetch time: Total SQL patch time / Number of transactions

  • Feature Added the table columns in submenus under Statistics and improved to display the columns also in the downloaded CSV file if columns are added through the Column Settings feature.

    See the following for the columns added by each submenu.

    • Domain: Time sum (ms)

    • Error: Agent, Agent node, Agent kind, Initial creation time

    • Remote HTTP Call: Std dev (ms)

    • Login: Time sum (ms)

    • Referrer: Time sum (ms)

    Note

    Initial creation time: It is the statistical time (5 minutes or 1 hour) when the transaction error was first generated during the query period.

  • Changed Added a guide message in the initial screen of Sitemap > Netstat Topology.

  • Fixed In Integrated Flex Board, fixed the error that occurs when the Active Transaction TopN widget is clicked.

  • Fixed Fixed the issue where when checking SQL steps in the Table view tab of the Trace analysis window, if one of the substitution variables or bind variables in the SQL statement is released, all of them are released.

  • Fixed In Analysis > Stack, fixed the issue where some checkboxes are partially cropped.

  • Fixed Fixed the error that occurs in the time selector when entering the Analysis > Transaction Search menu with an organization selected.
