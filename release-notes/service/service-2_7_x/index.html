Service 2.7.0​

2024년 07월 24일

신규 기능 출시​

Service 2.7.0 릴리스에 포함된 신규 기능을 소개합니다.

Database​

  • New Oracle V2 서비스 시작!!

    • 대시보드 메뉴의 UI 및 기능 대폭 개선했습니다.

    • 대시보드 메뉴에서 데이터베이스의 성능 지표 정보를 확인할 수 있습니다.

    • 데이터베이스 로그를 검색하고 추이를 확인할 수 있는 로그 메뉴를 추가했고, 로그 관련 알림을 설정할 수 있습니다.

    노트

    • Oracle V2의 새로운 기능에 대한 자세한 내용은 다음 문서를 참조하세요.

    • Oracle V2를 이용하기 위한 에이전트의 최소 버전은 1.9.0입니다. 에이전트 업데이트에 대한 자세한 내용은 다음 문서를 참조하세요.

  • New Oracle Pro 서비스 시작!!

    Direct Memory Access(DMA) 기술을 활용한 접근 방식으로 서버의 메모리에 직접 접근하여 필요한 데이터를 추출합니다. 이 방법은 데이터베이스 서버에 부하를 최소화하거나 발생시키지 않습니다. 메모리에 직접 접근함으로써 데이터를 거의 실시간으로 수집할 수 있으며, 시스템의 성능을 더욱 세밀하게 모니터링하고 문제 발생 시 즉시 대응할 수 있습니다.

    노트

    Oracle Pro에 대한 자세한 내용은 다음 문서를 참조하세요.

Kubernetes​

  • New Kubernetes 클러스터 내에서 실행 중인 모든 서비스들의 상태와 정보를 종합적으로 제공하는 Service 목록 메뉴 추가

    노트

    • 이 기능은 워크로드 > Service 목록 메뉴 경로에서 확인할 수 있습니다. 서비스 구성과 현재 상태를 상세히 파악하여 클러스터 내 네트워크 구성을 이해할 수 있습니다.

    • Service 목록 메뉴에 대한 자세한 내용은 다음 문서를 참조하세요.

    • 이 기능은 쿠버네티스 마스터 에이전트 1.7.8 버전 이상에서 지원합니다.

  • New 노드 관점에서 디스크 모니터링을 위한 노드 디스크 목록 메뉴 추가

    노트

    • 이 기능은 클러스터 > 노드 디스크 목록 메뉴 경로에서 확인할 수 있습니다.

    • 노드 디스크 목록 메뉴에 대한 자세한 내용은 다음 문서를 참조하세요.

    • 이 기능은 쿠버네티스 마스터 에이전트 1.7.8 버전 이상에서 지원합니다.

공통​

  • Feature 사이드 메뉴에서 전체 프로젝트를 선택해 프로젝트 목록을 펼치면, 프로젝트 검색 입력란에 자동 포커스되는 기능 추가

  • Feature 사이드 메뉴의 통합 검색에서 에이전트를 IP 주소로 검색할 수 있는 기능 추가

  • Changed 보고서 메뉴의 보고서 메일 발송 예약 창에서 시간 문구를 타임존이 포함된 문구로 안내하도록 변경

    예, 04~06 → (UTC+9) 04~06

  • Changed 사이드 메뉴의 조직 목록에 조직 아이콘 노출하도록 변경

  • Changed 계정 관리 또는 경고 알림 > 이벤트 수신 설정 메뉴에서 모바일 기기 알림 테스트 시 표시되는 성공 메시지 문구 변경

    모바일 기기와 성공적으로 연결되었습니다. → 테스트 메시지를 모바일 기기로 전송 요청 하였습니다.

  • Fixed 경고 알림 > 이벤트 기록 메뉴의 컬럼 선택 팝오버에서 초기화(기본값) 버튼 동작 버그 수정

  • Fixed 경고 알림 > 이벤트 수신 설정 메뉴에서 사용자 본인 계정이 아닌 다른 사용자의 모바일 기기 테스트 및 삭제 버튼을 선택할 수 없도록 비활성화

  • Fixed 관리 > 프로젝트 멤버 메뉴에서 테마 변경 시 이름/소속 태그 색상이 반영되지 않는 문제 수정

  • Deprecated 경고 알림 > 이벤트 설정 메뉴의 이상치 탐지 탭에서 이벤트를 추가 또는 수정할 때 오브젝트 병합 선택 UI 제거

  • Deprecated 통합 대시보드 / 통합 메트릭스 보드

Application​

  • Feature 다음 메뉴 경로에서 조회 내용을 CSV 파일 다운로드 기능 추가

    • 인스턴스 성능 관리 > 라이브러리 버전
    • 인스턴스 성능 관리 > 실행 환경 변수

  • Feature 관리 > 에이전트 설정 메뉴에서 경로에서 선택한 에이전트의 에이전트 설정(whatap.conf) 파일 다운로드 기능 추가

  • Changed 트레이스 분석 창의 멀티 트랜잭션 탭에서 호스트 정보가 없으면 트랜잭션을 노출하지 않도록 변경
