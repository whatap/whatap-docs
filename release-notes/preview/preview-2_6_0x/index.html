Preview 2.6.0.a

Note

In preview.whatap.io, the following new features and enhancements can be experienced in advance.

Preview 2.6.0.a7​

June 17, 2024

Common​

  • Changed Changed the criteria for the detailed view and display in Analysis > Metrics Anomaly Detection.

    • Previous: Displays the chart for the dragged period.

    • Change: Displays the data for up to 6 hours.

  • Changed In Integrated Dashboard, Integrated Metrics Dashboard, changed the service termination message.

Application​

Feature In Dashboard > Application Dashboard, added the default preset modification feature.

Server​

Fixed In Management > Agent Installation, changed the variables for the Oracle Solaris agent installation command and applied the multilingual content.

Kubernetes​

Fixed In Cluster > Node List, fixed the issue where the checkbox cannot be selected if the checkbox in the column selection dropdown is overlapped with that in the list below.

Database​

Common​

Fixed Fixed the issue where the data is not displayed properly when you click Today while selecting a date to compare in Analysis > Database parameter.

Note

This feature is not applied to Redis, MongoDB, and Oracle DMA products.

MySQL​

Changed In Management > Agent Installation, improved the installation guide style.

Log​

Changed In Log > Log Configuration, unified Data Retention and the name of Data period into Data retention.

Browser​

Feature In Page Load Hitmap, AJAX Hitmap, and User Session Log Search, added the Session replay feature.

Preview 2.6.0.a6​

June 13, 2024

Common​

  • Feature Reorganized the UI/UX for payment method registration and modification menus (Profile icon > Usage > Payment Information).

  • Changed Improved the guidance message when there are no search results on the product selection screen for project creation.

  • Changed In Usage > Project Subscription Status, modified to display the Subscribe button with separated from the Status column.

Application​

Common​

  • Feature In Dashboard > Transaction Map, added the Custom feature that can add items in the TOP 30 filter after searching them.

  • Feature Enhanced the features under Statistics.

    • It saves the selected value by the last viewed agent selection option.

    • It changed the feature not to automatically search upon entry of each menu.

    • It changed the initial query period for each menu from Today to Previous day.

    • It changed the default sorting order for Transaction and SQL menus to Sum time.

  • Changed In Instance Performance Management > Loaded Classes, changed the title of the class search window (Class signature → Class details).

  • Changed The Application Dashboard menu of old version is scheduled to be removed.

    • Added the Deprecated tag in the Sitemap menu.

    • If you enter Sitemap > Application Dashboard Old, the guidance message for planned removal appears.

  • Changed Even if you do not select after applying the filter in the Statistics submenu, the filter is to be applied when you download the CSV file.

  • Fixed On the Trace analysis window, fixed the issue where the strings for OS, Client name, and Status are exposed as translation keys.

  • Fixed Fixed the issue where if you select the step timeline graph on the Trace analysis window, the corresponding step does not appear.

  • Fixed Fixed the issue where the search time for the screen that appears by selecting Detail after dragging the chart on the widget in Analysis > Metrics Chart, appears differently from the widget search time.

  • Fixed In Dashboard > Application Dashboard, fixed the issue where the loading indicator is applied to the widget when the guidance window appears by clicking the button of the widget.

  • Fixed Fixed the issue where data call is followed without a delay when there is no agent in Dashboard > Application Dashboard.

  • Fixed Fixed the issue where scrolling is not applied when the agent list in the agent selection (Application) option is greater than the vertical size of the screen.

  • Fixed In Sitemap > Topology > APP. Statistics Topology, fixed the issue where the feature does not work by clicking the time selection option and changed the UIs for the time selection option.

  • Fixed Fixed the issue where a whiteout occurs when entering the Analysis > Stack menu from a project that belongs to the organization.

Java​

Fixed Modified the Jetty installation guide in the Add JVM Option section of Management > Agent Installation: JAVA_OPTS → JAVA_OPTIONS.

Kubernetes​

  • Feature In Cluster > Node List, added capacityType as a subcolumn of Node info.

    Note

    This feature is applied only to cluster projects.

  • Changed The Container table in Cluster > Node Detail was modified to match the table in Container > Container List.

  • Changed In Cluster > Node Detail, displayed the sums for the following metrics for each container within the node at the top of the Container table.

    cpu_quota, cpu_request, cpu_total_milli, mem_limit, mem_request, mem_working_set

  • Changed Changed the terminology of the list-type menus as follows:

    • Memory → Memory

    • Network → Network

    • Disk → Disk

    Note

    There is no change when the language setting is English.

  • Changed In Workload > Container List, changed the column classification.

  • Changed In Workload > Container Application List, changed the column name of Oname to MicroOname.

  • Changed In the summary view of Dashboard > Container Map, changed the category sorting sequence in the Log > Search Log tab.

  • Fixed In the summary view of Dashboard > Container Map, fixed the issue where the search result of the Log > Search Log tab is not consistent with the search result of View Detailed.

  • Fixed In Management > Kubernetes Agent List, modified to enable the View Detailed button only when the agent is active.

Log​

Changed In Log > Log Search, changed the display method of the accumulated number of cases when searching logs.

  • If the next page can be viewed, number+ appears. Otherwise, number appears.

  • When viewing the next page, the accumulated number of cases appears as 9999+, 19999+, and 29999+.

URL​

Fixed Fixed the issue where the Alert status icon on the URL project screen disappears.

Database​

Oracle DMA​

  • Feature In Management > Agent Installation, provided the agent download feature according to the Oracle version.

  • Fixed In Analysis > SQL Analysis, fixed the issue where multiple filtering is not applied.

  • Fixed Fixed the issue where the chart disappears from the Plan Change History tab of Analysis > SQL Analysis, when the browser is a specific size.

  • Fixed In Management > Agent Installation, modified the message of the installation guide.

Preview 2.6.0.a5​

June 7, 2024

Application​

  • Fixed In Analysis > Hitmap, fixed the issue where a specified instance is not selected by default in the pop-up menu when tracing databases.

  • Fixed When going to the screen in presentation or custom theme mode, in Analysis > Daily APP. Status, fixed the whiteout issue.

Database​

Common​

  • Feature In Analysis > Database Parameter, added the feature to compare between the real-time data and the last saved data when Reference date and Comparison date are all set to the date of today.

    Note

    This feature is not applied to Redis, MongoDB, and Oracle DMA products.

  • Changed In Analysis > Database Parameter, modified to display the collection time in the table's header area.

  • Changed In Stat/Report > SQL Statistics, modified to select only one instance for the Instance option.

    Note

    The same changes in specifications are applied in the following products:

    • MySQL: Stat/Report > MYSQL SQL Statistics

    • PostgreSQL: Stat/Report > PG SQL Statistics

  • Fixed In Stat/Report > SQL Statistics, fixed the issue where filtering is not applied.

  • Fixed Fixed the issue where the s_equal string is found in Filter when going to the Stat/Report > SQL Statistics menu by clicking a link such as filter value from other menus.

MySQL​

  • Changed In the basic column in Stat/Report > SQL Statistics, modified to hide the elapse wait item.

  • Changed In Stat/Report > SQL Statistics, moved the sqlHash column to the utmost right.

Oracle Pro​

Changed Changed the elapse_time metric name to elapsed_time that are used in db_oracle_dma_sqlstat or db_oracle_dma_xview.

Oracle V2 / Oracle Pro / PostgreSQL V2​

  • Fixed Modified to terminate the real-time view and not to select the time selector when the Session detail window is opened in the Active sessions table on the dashboard with the session terminated.

  • Fixed Modified not to expose the Correlation analysis button when the Session detail window is opened in the Active sessions table on the dashboard with the session terminated.
