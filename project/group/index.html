Group Management

You can group multiple projects for management. A user can own multiple projects. To manage multiple projects, the concept, "Group" has been provided. If there are multiple projects, it is cumbersome to add users or grant permissions for each project. In this case, you can group two projects into a group and manage users by the group.

Note
  • A project can belong to a group.
  • A project may not belong to a group.

Project users can inherit group users, and the users can monitor all projects in the group. A user who is granted the role only on the project can monitor only the project, regardless of whether the user is in a group or not.

Adding a group​

  1. From the side menu, select All Projects > Group.

  2. On the Add a Group screen, enter the name of the group to create in Group name.

  3. Enter the group description in Group description. The Group description field is optional.

  4. Select a project to add to the group in Add project. You can search a desired project for addition.

  5. After adding the project, select Add Group.

Note

The projects that can be added to the group are the ones with Edit permission or not part of the group.

Adding a project into a group​

  1. On the upper right of a project entry in the Project List of Home screen, select .

  2. Select Add to group.

  3. Select a group to add a project in.

  4. Select Add to group.

Viewing the project list in the selected group​

  1. By selecting All Projects in the side menu, open the group and project list panel.

  2. In the panel, select the button on the right of the group to view.

    Only the projects included in the selected group appear in the list. In this status, you can use the Group Member or Group Management menu.

  3. To see the full list of projects again, select the Go to project list button at the bottom of the side menu.

Note

To see the projects that are not part of the group, scroll down the list panel.

Excluding projects from the group​

Excluding from Home screen​

  1. On the upper right of a project entry in the Project List of Home screen, select .

  2. Select Delete From Group.

  3. If the Delete project from group window appears, select OK.

Excluding from group management​

You can move projects in a group to other projects or exclude them.

  1. By selecting All Projects in the side menu, open the group and project list panel.

  2. In the panel, select the button on the right of the group to delete.

  3. In the side menu, select Group Management.

  4. In the Group management section, select projects to exclude.

  5. If Move Project (N) is displayed, select the button.

  6. When the Move Project window appears, in Select group, select a group to move or select Other projects.

  7. Select Confirm.

Note

If you select Other projects in Select group, the project is excluded from its group.

Deleting a group​

  1. By selecting All Projects in the side menu, open the group and project list panel.

  2. In the panel, select the button on the right of the group to delete.

  3. In the side menu, select Group Management.

  4. In the Group Info section, select Delete group.

  5. See the details of the Delete group message window, and then select Delete.

Caution
  • To delete a group, the user list is also deleted.
  • In case of the projects in the group, only the group information is deleted.
  • The deleted information cannot be restored.

Inviting group member​

  1. By selecting All Projects in the side menu, open the group and project list panel.

  2. In the panel, select the button on the right of the group to invite members to.

  3. In the side menu, select Group Member.

  4. Select Invite Member on the upper right of the screen.

  5. You can also add members by selecting a sub project in Group.

  6. In User role, set the roles of the user to invite and then select Apply.

  7. In the Email input field, enter the email address of the user to invite, and then select Add. You can continue to add other users.

  8. After adding users to invite, select Complete.

Note

  • The member visiting feature can be used only by the users with the Member management role. For more information about the member roles, see the following. For more information about the member roles, see the following.

  • The accounts that have not signed up for the WhaTap monitoring service display the New account tag and send emails for sign up.

  • On the right of the email in the member list, the tag like MFA or SSO is displayed depending on whether MFA is applied or SSO is integrated.

  • If there are invited members to cancel before completion, in List of members to invite, select .

  • You cannot invite any by entering email addresses that are not in the desired email format.

  • The members who have already been invited cannot be invited again, and the members in the parent organization or group are automatically added.

Checking members for each project​

  1. By selecting All Projects in the side menu, open the group and project list panel.

  2. In the panel, select the button on the right of the group to check members from.

  3. In the side menu, select Group Member.

  4. If you select a project in the group, you can check the members in the project.

Modifying a group member role​

  1. By selecting All Projects in the side menu, open the group and project list panel.

  2. In the panel, select the button on the right of the group that includes the member whose role to be modified.

  3. In the side menu, select Group Member.

  4. In the Group Member list, select on the right of the member to modify.

  5. If the Edit member role window appears, set the values for Member role and User role per project of a group.

    • Member role: You can set the roles for the group and the projects in the group.
    • User role per project of a group: Permissions of the selected members can be set for each project in the group.

  6. After all settings are finished, select Apply.

Note

Member role editing can be used only by the users with the Member management role. For more information about the member roles, see the following.

Deleting group members​

  1. By selecting All Projects in the side menu, open the group and project list panel.

  2. In the panel, select the button on the right of the group that include the member to delete.

  3. In the side menu, select Group Member.

  4. In the Group Member list, select on the right of the member to delete.

  5. If the message window appears, select Delete.

Note

  • To delete members from subgroups, click any checkbox in the message window and then select Delete.

  • If you have not the Member management role, you cannot delete other members. For more information about member roles, see the following.

Downloading the member list​

To download the member list as a CSV format file, keep the following steps:

  1. By selecting All Projects in the side menu, open the group and project list panel.

  2. In the panel, select the button on the right of the group that include the member to delete.

  3. In the side menu, select Group Member.

  4. Select on the upper right of the screen.

Save the member list to the local disk in a CSV format file.

Note

Member role downloading can be used only by the users with the Member management role. For more information about the member roles, see the following.
