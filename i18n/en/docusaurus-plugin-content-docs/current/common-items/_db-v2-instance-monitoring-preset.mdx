## Preset{#preset}

<InDoc product="postgresql,mysql,oracle">

You can save and recall user-defined widget settings and active session table arrangements as presets. The following presets are provided by default, and the **_Default_** presets cannot be changed.

- **_Default_**: It consists of major DB metrics.

- **_Default(xos)_**: It consists of key DB metrics and XOS metrics (CPU and memory utilization, memory SWAP usage, disk I/O, etc.).

- **_Default(rds)_**: It consists of key DB metrics and RDS CloudWatch metrics (CPU utilization, free memory space, free storage space, network, etc.).

### Save Preset

In the <Cmdname sid="preset" className="uitext" /> list, select a saved item to fetch the settings.

1. Add or modify the widget metrics.

2. On the upper right of the screen, select ![Save icon](/img/ic-save.svg). 

3. In <Cmdname sid="save_preset" className="uitext" />, change the name and then select <Cmdname sid="save" className="uitext" />.

:::note

- The **_Default_** preset cannot be modified. 

- Depending on the database environment, some presets may not be supported.

- Including the modified metric graph, the column order, column width, and filter application content of the active sessions are also applied for custom presets.

- You can save or modify the presets with the <Cmdname sid="edit_project" className="uitext" /> role. For more information about the member roles, see [the following](../project/project-structure#member-auth).

:::

### Creating a preset {#newpreset}

1. Place the widgets in the desired format on the dashboard. You can also resize and place only the widgets to be frequently checked.

2. On the upper right of the screen, click the ![Save icon](/img/ic-save.svg) icon.

3. Enter a new preset name.

4. Click <Cmdname sid="save" className="uitext" />. You can see the newly saved preset from the preset list.

:::note

- If any changes are made to the newly created preset, save the preset again. Click the ![Save icon](/img/ic-save.svg) icon and then save the preset with the same name. Any changes are overwritten in the existing preset.

- If you go to another menu without saving changes on the dashboard, the changes are not saved.

- Presets are saved on a project basis and can be shared with other users.

:::

### Deleting presets

If you select <Cmdname sid="preset" className="uitext" /> on the upper right of the screen, the preset list appears. Select ![Delete icon](/img/ico-remove.svg) on the right of the preset to delete.
