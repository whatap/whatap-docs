## XOS agent installation (optional) {#apply-cubrid-xos-agent}

</InDoc>

To additionally monitor the resources of the database server, run a separate XOS agent process on the database server to collect data.

:::note

- It can be applied to only the OS environment running on the x86 architecture.

- The additional agent installation process is **optional**.

<Xclude product="oracle-pro">

- For more information about the XOS agent configuration options, see [the following](agent-xos-settings).

</Xclude>

<InDoc product="postgresql,mysql,oracle,mssql">

- To additionally collect and monitor monitoring metrics provided by cloud services on the dashboard of the database project, see [the following](cloudsettings).

</InDoc>

:::

### Configuring the whatap.conf file

Set the following options in the _whatap.conf_ file in the path where the DBX agent has been installed.

```ini title='whatap.conf'
xos=1
xos_port=3002
```

Move the _xos_ folder (`/unzip folder/xos/`) to the database server.

### Configuring the xos.conf file

Set the following options in the _xos.conf_ file in the _xos_ path moved to the database server.

```ini title='xos.conf'
dbx_ip={DB_Agent_IP}
dbx_port=3002 # default 3002
cpu_limit=0
mem_limit=10240
```

:::tip

In <Cmdname sid="side_agentInstall" className="uitext" />, when you enter the DB data to <UI>DB Agent IP</UI> and <UI>DB Agent Port</UI>, the agent options are automatically generated.

:::
