## Checking the metrics of the linked project {#checklinkedproject}

Metrics related to the added project can be checked in the <Cmdname sid="side_analysis" className="uitext" /> &gt; <Cmdname sid="side_hitmap" className="uitext" /> menu. You can also see performance metrics for the linked project while viewing transactions.

This feature is useful when analyzing transactions in the APM project. When you found a slow SQL, it is difficult to determine exactly in which DB the problem occurred in the environment using multiple similar DBs.

<ImgLang img="linked-project-db-hitmap.png" desc="Hitmap" />

1. Set the time and <Cmdname sid="application" className="uitext" /> for viewing the transactions, and then select ![Search Icon](/img/ico-btn-search.png).

2. In the <Cmdname sid="hitmap_transaction" className="uitext" /> section, select <Cmdname sid="BTN08519" className="uitext" /> on the upper right.

3. A new window appears where you can check the metrics for the correlated project.

   <ImgLang img="linked-project-db-instance-dahboard.png" desc="Correlated Project" />

Select the correlated project from the left list and then check the metrics in the <Cmdname sid="TTL08442" className="uitext" /> section. <Cmdname sid="TTL08442" className="uitext" /> allows you to view key performance metrics of the database associated with the transactions executed in the application.

<ImgLang img="linked-project-db-instance-dahboard-select-instance.png" desc="Correlated Project" />

To navigate to the dashboard menu for a selected project, select an instance and then click <Cmdname sid="BTN08444" className="uitext" /> on the upper right. Go to <Cmdname sid="side_dashboard" className="uitext" /> &gt; <Cmdname sid="side_dbMonitoring" className="uitext" />.

:::note

- The <Cmdname sid="hitmap_transaction" className="uitext" /> button in the <Cmdname sid="BTN08519" className="uitext" /> section appears only when there is an associated project. For more information about adding a correlated project, see [the following](#addlinkproject).

- You can add a project by selecting <Cmdname sid="TTL07151" className="uitext" /> at the top of the left <Cmdname sid="add" className="uitext" /> list.

- The <Cmdname sid="BTN08444" className="uitext" /> button is disabled when <Cmdname sid="all" className="uitext" /> is selected.

  <ImgLang img="linked-project-db-instance-dahboard-not-allowed-btn.png" desc="Instance Dashboard" />

- Depending on the database product type, you can navigate to the <Cmdname sid="side_analysis" className="uitext" /> &gt; <Cmdname sid="side_dbTrend" className="uitext" /> menu.

:::

</InDoc>

## Checking any linked project {#checklinkedproject}

It provides real-time details about active transactions in applications associated to active sessions in the database. Users can check specific metrics such as transaction ID, thread ID, client IP, and CPU usage time, along with basic information such as transaction URL and database connection information, DB type, HTTP method, and execution time. Additionally, you can monitor the SQL query execution time, number of SQL calls, DB connection time, and such, which allows you to comprehensively analyze the system performance.

In particular, it provides the call stack information and SQL queries through stack traces, helping you trace and resolve detailed causes when problems occur. This allows you to perform in-depth analysis of interactions between applications and databases, and to quickly diagnose performance bottlenecks.

### Tracing active transactions {#tracingtx}

Go to <Cmdname sid="side_dashboard" className="uitext" /> &gt; <Cmdname sid="side_dbMonitoring" className="uitext" />. The link analysis feature can be checked only in real-time query status.

<ProdImg img="db-linked-project-instance-monitoring.png" desc="Correlation analysis" />

1. Select an instance (agent) for query in the <Cmdname sid="instance" className="uitext" /> 
   option.

2. In the <Cmdname sid="active_sessions" className="uitext" /> table list, select ![Linked icon](/img/ic-correlations.svg) for the active session to correlate.

3. The <Cmdname sid="TTL07046" className="uitext" /> window appears, which provides transaction information associated with the selected active session.

Select the desired project from the <Cmdname sid="TTL07151" className="uitext" /> list to check the linked analysis information.

:::note

- You can go to the <Cmdname sid="side_appDashboard" className="uitext" /> menu for that project by selecting <Cmdname sid="BTN08404" className="uitext" /> on the upper right of the screen.

- If there is no transaction associated with the selected active session, the screen displays <Cmdname sid="no_data" className="uitext" />.

:::

### <Cmdname sid="TTL07046" /> Screen guidance {#TTL07046}

The <Cmdname sid="TTL07046" className="uitext" /> window displays information details for active transactions, as well as the call stack and SQL statements.

#### ***Active Transaction***

You can check information details of the linked active transactions.

<ImgLang img="db-linked-project-detail-active-tx.png" desc="Active Transaction" />

- ***DB Connection***: Database connection URL.

- ***Thread ID***: Unique identifier of the thread used to process the transaction.

- ***Sid***: ID of the selected active session.

  <InDoc product="postgresql">

  :::note

  ***Sid*** is the same value as the <UI>PID</UI> entry in the <Cmdname sid="active_sessions" className="uitext" /> table. It is also equivalent to the <UI>SID</UI> entry in the SQL information of the APM project.

  <ImgLang img="db-linked-projects-postgresql-pid-sid.png" desc="SID" />

  :::

  </InDoc>

- ***Thread Stat***: Current status of the thread.

- **DB Type\***: Type of the linked database platform.

:::note

- In **the versions before Java Agent 2.2.40**, the <UI>ThreadStat</UI> value appears in numeric format. To display in text format, install [Java Agent 2.2.40](../release-notes/java/java-2_2_40) or later in the related projects.

- For more information about others, see [the following](java/dashboard-hitmap-trace#anlaysis-trace-record-summary).

:::
