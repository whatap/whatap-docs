The log events that can be added are as follows:

- <Cmdname sid="realtime_log_event" className="uitext" />: Alerts are sent when the search value is found in the logs collected in real time.

- <Cmdname sid="composite_log_event" className="uitext" />: An alert is sent when more than a specific number of logs that meet conditions, are collected among recently collected logs.

## Common options for event addition {#log-event-add-common}

The following are common options that can be set when adding an event.

- <Cmdname sid="event_title" className="uitext" />: Enter the name of the event to add.

- <Cmdname sid="enable_events" className="uitext" />: You can set to enable or disable the alerts by clicking the toggle button.

- <Cmdname sid="level" className="uitext" />: Select one of <Cmdname sid="fatal" className="vslow" />, <Cmdname sid="warning" className="slow" /> and <Cmdname sid="info" className="info" />.

- <Cmdname sid="message" className="uitext" />: Enter a notification message to be displayed when events occur. By entering `${Tag}` or `${Field}`, you can apply the variable to the message. The key to enter in the variable must be included in <Cmdname sid="category" className="uitext" /> of the selected metrics data.

- <Cmdname sid="category" className="uitext" />: You can select the log type name (log folder name) from the list or enter it directly.

- <Cmdname sid="skiptime" className="uitext" />: This option can prevent excessive alerts from happening. No alerts are sent for the selected period after the first alert notification is generated. In addition, they are not recorded in <Cmdname sid="side_eventHistory" className="uitext" />.

- <Cmdname sid="event_tag" className="uitext" />: If you select an event receiving tag, alert notifications can be sent to project members and 3rd-party plug-ins with the tag. If the event receiving tag is not selected, alerts are sent to all project members.

  If the tag is not added, alerts are sent to all members. Click **_+_** <Cmdname sid="add_tag" className="uitext" /> and then select alert receivers in <Cmdname sid="event_tag_list" className="uitext" />. Select **_+_** <Cmdname sid="create_new_eventtag" className="uitext" /> to add new tags.

  :::note

  In <Cmdname sid="side_event" className="uitext" /> &gt; <Cmdname sid="side_eventNoti" className="uitext" />, you can set the tags in project members and 3rd-party plug-ins. For more information about <Cmdname sid="side_eventNoti" className="uitext" />, see [the following](set-receive-event).

  :::

## Add real-time log event {#rt-log-event}

<ImgLang img='log-alert-rt.png' desc='실시간 로그 이벤트 추가 sc' />

- <Cmdname sid="ITM06429" className="uitext" />: Indicates an identifier for accessing a specific value within the log data. You can select any in the list and directly enter it.

  > e.g. In case of accessing a value representing the HTTP response status code, <Cmdname sid="ITM06429" className="uitext" /> `status`

- <Cmdname sid="ITM06430" className="uitext" />: Indicates actual data that corresponds to the <Cmdname sid="ITM06429" className="uitext" />. An alert is sent if the log includes the specified word. You can select any in the list and directly enter it.

  > For example, in case of setting the <Cmdname sid="ITM06429" className="uitext" /> `status` <Cmdname sid="ITM06430" className="uitext" /> `200`, a warning notification is generated when collecting the log data with the HTTP response status code 200.

- <Cmdname sid="alert_object_filtering" className="uitext" />: Select <Cmdname sid="ITM06429" className="uitext" />, <Cmdname sid="operator" className="uitext" />, and <Cmdname sid="ITM06430" className="uitext" /> through the <Cmdname sid="metrics_rule_select" className="uitext" /> option to filter the targets. Otherwise, select the <Cmdname sid="metrics_rule_direct" className="uitext" /> option. If there is no input, it determines whether or not to generate notifications for all data collected in real time.

  > For example, among the logs in the `AppLog` category, it searches for the logs whose `level` is `ERROR`. An alert is sent when **_oid_** is **12345678** among the matching logs.

## Add composite log event {#cp-log-event}

<ImgLang img='log-alert-cp.png' desc='복합 로그 이벤트 추가 sc' />

- <Cmdname sid="template" className="uitext" />: Composite log templates are provided.

- <Cmdname sid="data_time_range" className="uitext" />: You can view the logs collected during the selected period. If you select 5 minutes as the time range, the event rule condition is checked after searching for the data collected during the last 5 minutes.

- <Cmdname sid="ITM06654" className="uitext" />

  - In <Cmdname sid="ITM06429" className="uitext" />, you can select any event rule. You can select any value that meets the selected <Cmdname sid="ITM06429" className="uitext" />.

  - If you add a same item in <Cmdname sid="ITM06429" className="uitext" />, the "OR" condition works. Otherwise, the "AND" condition works.

  - You can select <Cmdname sid="exclude" className="uitext" /> to exclude the selected search value from the event rules.

  - You can select **_+_** <Cmdname sid="add" className="uitext" /> to add or exclude multiple event rule conditions.  

- <Cmdname sid="alert_rule" className="uitext" />: Alerts are sent when events occur as many as the entered number and they occur by meeting the selected operator's condition.

  > For example, among the logs in the `AppLog` category, the logs that meet `Enter conditions` are filtered. If <Cmdname sid="exclude" className="uitext" /> is checked for condition, it indicates that the logs searched by the condition are excluded. Thus, the logs whose `level` is `ERROR` are excluded. Alerts are sent when the event is less than `5` among the logs collected in the last 10 minutes.

## Modifying the log event settings

1. Go to <Cmdname sid="side_event" className="uitext" /> &gt; <Cmdname sid="side_eventSetting" className="uitext" />.

2. Select <Cmdname sid="log" className="uitext" /> tab.

3. Select ![Edit icon](/img/ico-edit.svg) on the right of the event to modify in the log event list.

4. If the <Cmdname sid="event_setting" className="uitext" /> window appears, modify the options and then select <Cmdname sid="save" className="uitext" />.

To delete the selected log event, select <Cmdname sid="delete" className="uitext" /> on the upper right of the Event Setting window.

## Turning off the log event

1. Go to <Cmdname sid="side_event" className="uitext" /> &gt; <Cmdname sid="side_eventSetting" className="uitext" />.

2. Select <Cmdname sid="log" className="uitext" /> tab.

3. In the log event list, select the toggle button on the utmost right of the event to turn off the alert.

If you select the toggle button again, the alert is enabled.

## Exporting and importing the log event

After saving the log event settings in a json file, you can load them for reuse.

1. Go to <Cmdname sid="side_event" className="uitext" /> &gt; <Cmdname sid="side_eventSetting" className="uitext" />.

2. Select <Cmdname sid="log" className="uitext" /> tab.

3. Select **_JSON_** ![Export icon](/img/ico-export.svg) on the log event list. The <Cmdname sid="export_json" className="uitext" /> window appears.

4. Modify or edit the items to export.

5. On the upper right of the screen, select ![Export icon](/img/ico-export.svg) <Cmdname sid="export" className="uitext" />. Download the json file through your browser.

6. On the log event list, select ![Import icon](/img/ico-import.svg).

7. If the file selection window appears, select the downloaded json file.

8. If the <Cmdname sid="import_json" className="uitext" /> window appears, modify the content and then select ***+*** <Cmdname sid="add_to_list" className="uitext" />.

:::note 

**ID exists in the event. After deleting the ID, try again.**

- If a message appears, in the <Cmdname sid="import_json" className="uitext" /> window, delete the `id` and then select **_+_** <Cmdname sid="add_to_list" className="uitext" />.

- To overwrite an existing item, delete the `id` item and then select <Cmdname sid="overwrite" className="uitext" />.

:::
