On the initial screen of the WhaTap Monitoring service, select a project and then click <Cmdname sid="side_analysis" className="uitext" /> &gt; <Cmdname sid="side_perfTrend" className="uitext" />. You can see numerous metrics about the performance within the specific time range to view. 

<ImgLang img='performance-trend.png' desc='Performance Trend' />

### ![number 1](/img/number-01.png) Performance trend search conditions

- The time selector allows you to select a time range to search. The default search period is **_1 day_**.

  - If the selected range is 3 hours or less, a 5-second trend is displayed using raw data.

  - If the selected range is within 7 days, a 5-minute trend is displayed using 5-minute statistics.

  - If the selected range exceeds 7 days, a 1-hour trend is displayed using 1-hour statistics.

- The application selector allows you to select all or specific applications to view. 

- Select ![CSV Download icon](/img/ico-csv.svg) **_CSV_** download icon, you can download the list data displayed on the screen as a CSV file.

:::note

For more information on how to use the time selector, see [the following](usage-ui#timeselector).

:::

### ![number 2](/img/number-02.png) Performance Trend Chart

- You can see the specified lookup time range at the top of the chart.

- In the chart area, you can see the <Cmdname sid="rt_user" className="uitext" />, <Cmdname sid="tps" className="uitext" />, <Cmdname sid="resp_time" className="uitext" />, <Cmdname sid="cpu" className="uitext" />, <Cmdname sid="heap_memory" className="uitext" />, <Cmdname sid="active_transaction" className="uitext" />, <Cmdname sid="transaction" className="uitext" />, <UI>Top 10</UI>, <Cmdname sid="httpc" className="uitext" /> <UI>Top 10</UI>, <Cmdname sid="sql" className="uitext" /> <UI>Top 10</UI> and such.

- By selecting the ![Snapshot icon](/img/ic-camera.svg) <Cmdname sid="sms_alert_snapshot" className="uitext" /> icon, you can snapshot charts except for widget options.

#### <Cmdname sid="rt_user" />

Real-time users are counted based on unique browser IP addresses.

- When using raw data, the number of unique browser IPs that sent requests within 10 seconds is counted.

- When using 5-minute statistics, the number of unique browser IPs that sent requests in the past 5 minutes is counted.

- When using 1-hour statistics, the number of unique browser IPs that sent requests in the past hour is counted.

#### <Cmdname sid="tps" />

<Cmdname sid="tps" className="uitext" /> (Transaction Per Second) means the number of transactions processed per second.

#### <Cmdname sid="resp_time" />

It indicates the average response time for requests. If the average is very high, there may be many slow transactions or failures at any specified time.

#### <Cmdname sid="cpu" />

It indicates the CPU usage of the application server. You can see the trend in usage change.

Active transaction indicates a transaction in progress. For more information about active transactions, see [the following](dashboard#active-trs).

#### <Cmdname sid="transaction" /> Top 10

We use the transaction call statistics to provide you with the top 10 list with the most frequent transaction calls.

#### <Cmdname sid="httpc" /> Top 10

We use the HTTPC statistics to provide the top 10 list with the highest numbers of HTTP call URLs.

#### <Cmdname sid="sql" /> Top 10

It provides the top 10 list with the highest numbers of SQL calls using SQL statistics.
