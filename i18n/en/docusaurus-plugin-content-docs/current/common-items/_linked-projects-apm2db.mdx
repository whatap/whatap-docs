## Checking the metrics of the linked project {#checklinkedproject}

Metrics related to the added project can be checked in the following menu path. 

- <Cmdname sid="side_dashboard" className="uitext" /> &gt; <Cmdname sid="side_appDashboard" className="uitext" />

- <Cmdname sid="side_dashboard" className="uitext" /> &gt; <Cmdname sid="side_multi_active_tx" className="uitext" />

- <Cmdname sid="side_analysis" className="uitext" /> &gt; <Cmdname sid="side_hitmap" className="uitext" />

:::note

- For more information about the <Cmdname sid="side_appDashboard" className="uitext" /> menu, see [the following](dashboard-intro).

- For more information about the <Cmdname sid="side_multi_active_tx" className="uitext" /> menu, see [the following](dashboard-active-transaction).

- For more information about <Cmdname sid="side_hitmap" className="uitext" />, see [the following](analysis-hitmap).

:::

### <Cmdname sid="side_appDashboard" />

You can check the linked projects through the following widget in the <Cmdname sid="side_appDashboard" className="uitext" /> menu.

- <Cmdname sid="AA001" className="uitext" />

- <Cmdname sid="AA002" className="uitext" />

- <Cmdname sid="AA005" className="uitext" />

#### <Cmdname sid="AA001" /> / <Cmdname sid="AA002" /> widget

You can query the SQL performance of the database associated with active transactions. Go to <Cmdname sid="side_dashboard" className="uitext" /> &gt; <Cmdname sid="side_appDashboard" className="uitext" />.

<ImgLang img="linked-projects-check-widget-01.png" desc="Correlated project" />

<ImgLang img="linked-projects-check-widget-02.png" desc="Correlated project" />

1. On the dashboard, click the chart area of ​​the <Cmdname sid="AA001" className="uitext" /> or <Cmdname sid="AA002" className="uitext" /> widget.

2. When the <Cmdname sid="acttx" className="uitext" /> window appears, select transactions.

3. When the transaction details window appears, select <Cmdname sid="BTN08518" className="uitext" /> on the upper right.

4. When the <Cmdname sid="TTL07860" className="uitext" /> window appears, check the related metrics.

:::note

- The <Cmdname sid="BTN08518" className="uitext" /> button appears in the transaction details window only when the DB of the database project registered for linked analysis has executed the SQL of the selected transaction.

- For more information about the <Cmdname sid="TTL07860" className="uitext" /> window, see [the following](#TTL07860).

- In **the versions before Java Agent 2.2.40**, the <UI>ThreadStat</UI> values appear in numeric format on the <Cmdname sid="acttx" className="uitext" /> window. To display in text format, install [Java Agent 2.2.40](../release-notes/java/java-2_2_40) or later.

- The databases that support this feature are **PostgreSQL**, **Oracle**, **Oracle Pro**, and **MySQL** platforms.

:::

#### <Cmdname sid="AA005" /> Widget

You can view performance metrics of the database associated with transactions from the past. Go to <Cmdname sid="side_dashboard" className="uitext" /> &gt; <Cmdname sid="side_appDashboard" className="uitext" />.

##### Checking in the DB connection step

<ImgLang img="linked-projects-check-widget-03.png" desc="Correlated project" />

<ImgLang img="linked-projects-check-widget-04.png" desc="Correlated project" />

:::note

- The ![Link icon](/img/ic-correlations.svg) button appears on the <Cmdname sid="TTL06126" className="uitext" /> tab only when the DB of the database project registered for linked analysis executes the SQL statement of the selected transaction.

- For more information about the <Cmdname sid="TTL08442" className="uitext" /> window, see [the following](#TTL08442).

- For more information about the <Cmdname sid="TTL06209" className="uitext" /> window, see [the following](dashboard-hitmap-trace).

### Copying/pasting the bind variable {#copypastebindvar}

You can copy the bind variable value displayed in the SQL step and then paste it into the <UI>SQL Text</UI> tab of the <Cmdname sid="TTL07860" className="uitext" /> window. Check the SQL step in the <Cmdname sid="TTL06209" className="uitext" /> window.

<ImgLang img="linked-projects-copy-bind-var.png" desc="Bind variable" />

1. In the <UI>SQL</UI> window, select <Cmdname sid="BTN06764" className="uitext" /> and then enter the security key.

2. Check the bind variable and then select <Cmdname sid="BTN08840" className="uitext" />.

3. Select <Cmdname sid="BTN08518" className="uitext" />.

4. When the <Cmdname sid="TTL07860" className="uitext" /> window appears, select <Cmdname sid="BTN08846" className="uitext" /> on the <UI>SQL Text</UI> tab.

:::note

- For more information on how to view the SQL step in the <Cmdname sid="TTL07860" className="uitext" /> window, see [the following](#sqlstep).

- For more information about the agent configuration related to the security key, see [the following](install-agent#security).

:::

## <Cmdname sid="TTL07860" /> {#TTL07860}

<Cmdname sid="TTL07860" className="uitext" /> allows you to check the SQL performance of the database associated with the current transaction in the application.

<ImgLang img="linked-projects-database-info.png" desc="Correlated DB information" />

- The charts at the top of the screen provide charts for key metrics of the database. You can also select other metrics by selecting the checkbox.

- The ***Session Info / Lock Tree*** section provides various details for the database session.

  <details>

  <summary>Session Info / Lock Tree</summary>

  - ***query_hash***: Unique hash value for the SQL query in progress.

  - ***user***: User name connected to the database.

  - ***db***: Name of the connected database.

  - ***runtime***: Time (in milliseconds) that the session has been run.

  - ***query_param***: Parameter used in the query.

  - ***thread_id***: Unique identifier of the thread connected to this session.

  - ***name***: Name associated with the session.

  - ***rows***: Number of rows processed in the executed query.

  - ***state***: State of the session.

  - ***id***: Unique identifier of the current session.

  - ***host***: IP address of the server in which the database is running.

  - ***command***: It indicates the currently executing command or task.

  - ***os_id***: ID of the session used in the operating system.

  - ***type***: Type of the session.

  - ***connection_type***: Database connection type.

  - ***time***: Started time of the session.

  - ***oid***: Unique ID of the instance (agent) to which the session belongs.

  </details>

- The ***SQL Text*** tab displays the actual content of the executed SQL query.

  - ***Default Bind***: You can bind the part set as a variable in the query sentence to the default value.

  - ***Reset Bind***: You can release the bound default value and check the variable.

  - <Cmdname sid="BTN08846" className="uitext" />: You can paste the copied bind variable value. For more information, see [the following](#copypastebindvar).

  - ***Formatting***: You can improve readability by applying indentation and formatting to a SQL query statement.

- ***Plan***: Plan information can be checked. Enter **DB Name**, **User Name**, and **Password**, and then select ![Search icon](/img/ico-btn-search.png).
