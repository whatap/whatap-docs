## ![](/img/number-01.png) Time and category selection

In the ![number1](/img/number-01.png) area, you can specify the <Cmdname sid="time_selector" className="uitext" />, <Cmdname sid="max_count" className="uitext" />, and <Cmdname sid="category" className="uitext" /> for the collected metrics. <Cmdname sid="time_selector" className="uitext" /> and <Cmdname sid="category" className="uitext" /> must have been specified.

- <Cmdname sid="time_selector" className="uitext" />: You can query by specifying the time when metrics were collected. The default value is `1 hour`. In addition to the lookup time provided as the default option, you can specify the date and time after selecting the <Cmdname sid="time_selector" className="uitext" /> tab.

  :::note

  For more information on how to use the <Cmdname sid="time_selector" className="uitext" /> option.

  :::

- <Cmdname sid="max_count" className="uitext" />: You can specify the maximum number of metrics to search in the ![number3](/img/number-03.png) list. You can set the count up to `10`, `50`, `100`, `200`, `300`, `1000`, `2000`, and `3000`.

- <Cmdname sid="category" className="uitext" />: It is a classification unit for the related metrics. You can set the desired <Cmdname sid="category" className="uitext" /> after selecting the <Cmdname sid="category" className="uitext" /> tab.

- ![Refresh icon](/img/ic-refresh-w-bg.svg) <Cmdname sid="refresh_tag_options" className="uitext" />: <Cmdname sid="category" className="uitext" />, <Cmdname sid="tags" className="uitext" />, and <Cmdname sid="field" className="uitext" /> options can be fetched.

## ![](/img/number-02.png) Tag and field selection

In the ![number2](/img/number-02.png) area, select <Cmdname sid="tags" className="uitext" /> and <Cmdname sid="field" className="uitext" />. The default setting is "Select All," unless specified separately by user. 

- <Cmdname sid="tags" className="uitext" />: Unique data that can identify the collected targets.

- <Cmdname sid="field" className="uitext" />: Metrics collected from the monitored targets.

- <Cmdname sid="filter" className="uitext" />: Select ![Filtering icon](/img/ico-filter.svg) <Cmdname sid="filter_by_tag_value" className="uitext" /> and then set the tag value for filtering.

  > For example, you can search filtered data by setting the value of `oname` to `demo-8101`.

- ![Search icon](/img/ic-search-on-press.svg) <Cmdname sid="search" className="uitext" />: If you set the conditions and then select <Cmdname sid="search" className="uitext" />, you can search the source data of the metric in the ![number3](/img/number-03.png) area. 

- ![CSV Download icon](/img/ic-csv-download-ms.svg) : You can download the source data of the metric as a CSV file.

## ![](/img/number-03.png) Metrics table

Because the metrics to be collected cannot be specified in advance, it is important to check the source data for all metrics being collected. After setting the desired conditions in the condition area above, you can search the original data of the metric in a table format in the ![number3](/img/number-03.png) area. Columns in the table are changed when you specify the conditions for <Cmdname sid="tags" className="uitext" /> and <Cmdname sid="field" className="uitext" />.

:::note

- To search metrics, <Cmdname sid="time_selector" className="uitext" /> and <Cmdname sid="category" className="uitext" /> must have been specified. 

- To search metrics, <Cmdname sid="tags" className="uitext" /> and <Cmdname sid="field" className="uitext" /> are optional.

<InDoc product="npm">

- Among network performance metrics, the `npm_process_tag_data` and `npm_process_tag_list` categories do not provide the item, <Cmdname sid="tag" className="uitext" />.

</InDoc>

:::
