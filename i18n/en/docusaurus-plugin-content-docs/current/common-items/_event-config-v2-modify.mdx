## Modifying event {#modify-event}

You can modify the metric values applied in basic event templates for changing the warning notification occurrence criteria. In addition, you can change the notification occurrence criteria through various option settings. Select ![Edit icon](/img/ico-edit.svg) on the event to modify. The event editing screen appears. Modify the following items and then select <Cmdname sid="save" className="uitext" />.

### <Cmdname sid="TTL07362" /> {#TTL07362}

You can set the criteria for event occurrence conditions.

<ImgLang img='event-config-v2-condition-common.png' desc='Define event condition' />

- <Cmdname sid="simulation" className="uitext" />: If you set event conditions and then select the button, the result of the set event can be predicted.

  <ImgLang img='event-config-v2-simulation-common.png' desc='Event simulation' />

  :::note

  For more information on how to use the time selector, see [the following](usage-ui#timeselector).

  :::

- <Cmdname sid="TTL07365" className="uitext" replace="</br>" />: You can set the thresholds for the <span class="slow">Warning</span> or <span class="vslow">Critical</span> notifications. Set an operator and then enter thresholds.

  - Operators: `>` (greater than), `>=` (greater than or equal to), `<` (less than), `<=` (less than or equal to), `==` (equal to)

  - You can enter a positive number, negative number, or 0 as the threshold, but whether to enter a negative number may differ depending on the selected event template.

  :::note

  - Depending on the provided event template, the notification levels that can be set may differ.

  - You can select one of <span class="slow">Warning</span> and <span class="vslow">Critical</span>. However, you cannot disable both two items.

  :::

- <Cmdname sid="TTL07366" className="uitext" />: You can send notifications based on the count that meets the specified event conditions.

  - <Cmdname sid="ITM07369" className="uitext" />: Notifications are sent each time any event occurs.

  - <Cmdname sid="TTL07366" className="uitext" />: Notifications are sent each time any event occurs as many as you specified.

- <Cmdname sid="TTL07367" className="uitext" />: No notification is sent for the selected time after receiving the notification.

  :::note

  If the <Cmdname sid="CNT08326" className="uitext" /> feature is active, notifications are not sent for the set period after sending a notification in <b>RECOVERED</b> state.

  :::

- <Cmdname sid="CNT08326" className="uitext" />: When this option is enabled, the events appear as <Cmdname sid="in_progress" className="uitext" /> in the <Cmdname sid="side_eventHistory" className="uitext" /> menu. When an event of <span className="vslow">Critical</span> or <span className="slow">Warning</span> level is resolved, normal (**RECOVERED**) notifications are received. This function can be enabled or disabled by selecting the toggle button.

  <ImgLang img="metric-warning-notice-new-recovered.png" desc="Resolved notification" />

  :::note

  For more information about <Cmdname sid="side_eventHistory" className="uitext" />, see [the following](set-event-history).

  :::

- **_Event action time_**: If you set an event action time tag, notifications with the tag trigger events during the time zone.\
  By clicking **_+ Add_**, you can see the list of tags in **_Event action time_** and create a new tag. 

  - To create a tag, click **_+ Create new tag_** in **_Event action time_**, select the tag name, day of the week, time, and color, and then click the Apply button.

  - To edit or delete tags, click ![Edit icon](/img/ico-edit.svg) in the tag list.

  <ImgLang img="event-operation-time.png" desc="Resolved notification" />

    :::warning

    The tags that are in use in the event settings cannot be deleted.

    :::

:::tip

The criteria for sending notifications differ depending on the following conditions:

| <Cmdname sid="TTL07366" className="uitext" />                                                 | <Cmdname sid="CNT08326" className="uitext" /> | Action                                                                                                                                                  |
| --------------------------------------------------------------------------------------------- | :-------------------------------------------: | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| <Cmdname sid="ITM07369" className="uitext" />                                                 |                     **On**                    | Notifications are sent when the specified event conditions are met or resolved.                                                                         |
| ^                                                                                             |                    **Off**                    | The notifications are sent whenever the specified conditions are met.                                                                                   |
| <Cmdname sid="TTL07366" className="uitext" /> N <Cmdname sid="ITM07370" className="uitext" /> |                     **On**                    | Notifications are sent when a condition is met at least N times or an event is resolved at least N times.                                               |
| ^                                                                                             |                    **Off**                    | A notification is sent if the event conditions you set are met more than N times. After sending a notification, the count is reset and it is recounted. |

:::

### <Cmdname sid="TTL07364" /> {#TTL07364}

You can write a title and message for the event received as a notification, and then select the recipients.

<ImgLang img='event-config-v2-info-receive-common.png' desc='Basic information and notification setting' />

- <Cmdname sid="enable_events" className="uitext" />: You can enable the current events.

- <Cmdname sid="event_title" className="uitext" />: The event name of the basic template has been specified. You can edit the event name as you want.

- <Cmdname sid="TTL06883" className="uitext" />: The message of the basic template has been specified. You can edit the message as you want. If the <LinkImage img="ico-timing.svg" className="ico" desc="time icon" /> button is clicked, you can see the history of previously entered messages.

  :::tip

  You can write a message by entering the <code>&#36;&#123;Tag&#125;</code> or <code>&#36;&#123;Field&#125;</code> variable in the message field.

  In <Cmdname sid="analysis" className="uitext" /> &gt; <Cmdname sid="tag_count" className="uitext" />, select <Cmdname sid="category" className="uitext" /> and then check the <code>&amp;#36;&amp;#123;Tag&amp;#125;</code> or <code>&amp;#36;&amp;#123;Field&amp;#125;</code> variable that can be entered. For the <Cmdname sid="category" className="uitext" /> name of the current event template, see the <b>Category</b> in <Link to="#basic-template">the following</Link>

  :::

- <Cmdname sid="alert_test" className="uitext" />: When receiving notifications as events, you can pre-check the specified event name and message. Testing is possible only when the required items (<Cmdname sid="TTL07365" className="uitext" replace="</br>" />, <Cmdname sid="event_title" className="uitext" />, and <Cmdname sid="TTL06883" className="uitext" />) are all entered.

  :::note

  - During testing, substitutions for actual metric values ​​or variables do not work. 

  - Test notifications are sent only to users with the recipient tag set in the <Cmdname sid="TTL07415" className="uitext" /> option. If you select <Cmdname sid="ITM07419" className="uitext" />, a test notification is sent to all users.

  :::

- <Cmdname sid="TTL07415" className="uitext" />: You can select members to receive notifications from current events.

  - <Cmdname sid="ITM07419" className="uitext" />: You can send notifications to all the members in the project.

  - <Cmdname sid="ITM07420" className="uitext" />: It sends notifications to project members and 3rd-party plugins with the selected tags. Click <Cmdname sid="add_tag" className="uitext" /> or <LinkImage img="ico-add.svg" className="ico" /> to select a tag from the tag list.

  :::note

  In <Cmdname sid="event_statistic" className="uitext" /> &gt; <Cmdname sid="eventNoti" className="uitext" />, you can set the tags in project members and 3rd-party plug-ins. For more information, see <Link to="set-receive-event#set-notice-by-team-or-users">the following</Link>.

  :::
