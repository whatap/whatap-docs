The following explains how to create and manage event notifications based on metrics. Metric events are used to configure the events more specific and complex than the [Default template](warning-notice-v2). It provides the feature that allows you to define various events occurring in the monitoring targets according to conditions and set them to receive notifications efficiently.

## Adding event

To add a metrics event, select the <Cmdname sid="add_notification" className="uitext" /> button on the upper right of the screen. When the Add Event screen appears, proceed with configuration by following the steps below:

1. <Cmdname sid="TTL07362" className="uitext" />: Set the event conditions to receive notifications.

2. <Cmdname sid="TTL07363" className="uitext" />: Select the monitoring targets to generate events.

3. <Cmdname sid="TTL07364" className="uitext" />: Set the event name, message, and recipients.

### <Cmdname sid="TTL07363" /> {#TTL07363}

Select monitoring targets where events occur. If not entered, events are monitored and notifications are sent across the entire project. This may result in lots of notifications. Set specific targets to avoid excessive notifications.

<ImgLang img="metric-warning-notice-new-select-target.png" desc="Select target" />

Event targets can be specified based on the following tags (`Tag`). The tag (`Tag`) is the data containing unique information to distinguish the targets to collect. You can use values ​​such as IP, Oname, and host data that have little change history.

- <Cmdname sid="metrics_rule_select" className="uitext" />: You can specify targets by selecting tag names and operators and entering values.

  <Cmdname sid="add" className="uitext" />: You can add targets for the event. You can select `&&`(and) or `||`(or) as the condition.

- <Cmdname sid="metrics_rule_direct" className="uitext" />: You can specify targets by directly entering the tag names, operators, and values. If you select this option, the tags (<Cmdname sid="metrics_event_tag_button" className="uitext" />) that can be entered are provided.

:::tip

Set event targets by noting the following example.

```text
ex. endsWith(okindName, 'example_name') && container == 'prod.billing'
ex. ${4xxErrorType} == '401'
```

:::

:::note

- Depending on the value selected in <Cmdname sid="select_category" className="uitext" /> of the <Cmdname sid="TTL07362" className="uitext" /> section, available tags may differ.

- If the event target is changed, the number of notifications may also change. Run the <Cmdname sid="simulation" className="uitext" /> button again to see the result.

- For more information about the selection of event targets and use of operators, see [the following](#condition-guide).

## Modifying and deleting the event

1. Go to <Cmdname sid="side_event" className="uitext" /> &gt; <Cmdname sid="side_eventSetting" className="uitext" /> and then select the <Cmdname sid="metrics" className="uitext" /> tab.

2. In the event list, select ![Edit icon](/img/ico-edit.svg) on the utmost left of the item to edit or delete.

3. If the metrics event setting window appears, modify each option and then select <Cmdname sid="save" className="uitext" />.

   To delete the selected event, select <Cmdname sid="delete" className="uitext" /> on the upper right of the Event configuration window.

## Modifying to JSON format {#json}

You can modify metrics event settings in JSON format.

1. On the upper right of the screen, select <span class="uitext">JSON</span> <LinkImage img="ico-export.svg" className="ico" />.

2. When the editing window appears, modify the content in JSON format.

3. After all changes are made, select <Cmdname sid="save" className="uitext" /> on the upper right of the screen.

:::note

If the modified content does not match the JSON format, an error message appears at the bottom of the screen and the changes cannot be saved. The displayed error message may differ depending on the format.

<ImgLang img='event-config-v2-json-error.png' desc='JSON error' />

:::

The JSON data structure is as follows:

```json
[
  {
    "eventMessage": "APDEX: ${apdex100} > 10 ",
    "select": "",
    "receiver": [],
    "alertLabel": [
      "project"
    ],
    "rule": "apdex100 > 10",
    "silentSec": 0,
    "alertKey": [
      "project"
    ],
    "enabled": false,
    "eventTitle": "APDEX",
    "repeatDuration": 0,
    "eventLevelText": "Critical",
    "id": "z3f41ge464magg",
    "category": "app_counter_project{m5}",
    "repeatCount": 0,
    "stateful": false
  }
]
```
