## Basic event templates {#basic-template}

Basic event templates are provided based on the product of the created project. You can activate a desired event by selecting <Cmdname sid="TTL06099" className="uitext" /> on the utmost left of the event list.

<InDoc product="java,php,python,nodejs,dotnet,golang">

<ImgLang img='event-config-v2-apm.png' desc='Event list' />

The provided event templates are as follows. The default value set in <Cmdname sid="ITM07578" className="uitext" /> can be changed to a desired one.

- **_Active Transaction_**

  Category: [`app_counter`](metrics-app#app_counter)

  If the number of active transactions (`active_tx_count`) exceeds 100, a warning (<span class="slow">Warning</span>) level notification is sent.

- **_Server CPU_**

  Category: [`app_host_resource`](metrics-app#app_host_resource)

  If the host's CPU utilization (`cpu`) exceeds 70%, a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 90%, a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_Server Disk_**

  Category: [`app_host_resource`](metrics-app#app_host_resource)

  If the host's disk utilization (`disk`) exceeds 70%, a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 90%, a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_Server Memory_**

  Category: [`app_host_resource`](metrics-app#app_host_resource)

  If the host's memory utilization (`mem`) exceeds 70%, a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 90%, a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_Transaction Error Count_**

  Category: [`app_counter`](metrics-app#app_counter)

  If the number of transaction errors (`tx_error`) exceeds 10, a warning (<span class="slow">Warning</span>) level notification is sent.

- **_Transaction Response Time_**

  Category: [`app_counter`](metrics-app#app_counter)

  If the average response time (`resp_time`) for transactions continues to exceed 2,000 ms (2 seconds) for 10 consecutive times, a warning (<span class="slow">Warning</span>) level notification is sent.

:::note

For more information about the fields applied to the basic event templates, see [the following](metrics-app).

:::

The provided event templates are as follows. The default value set in <Cmdname sid="ITM07578" className="uitext" /> can be changed to a desired one.

- **_CPU_**

  Category: [`server_base`](metrics-server#server_base)

  If the CPU utilization (`cpu`) exceeds 70%, a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 90%, a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_CPU Steal_**

  Category: [`server_base`](metrics-server#server_base)

  If the [CPU steal utilization](../glossary#cpusteal)(`cpu_steal`) exceeds 5, a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 10, a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_Disk_**

  Category: [`server_disk`](metrics-server#server_disk)

  If the file system utilization (`usedPercent`) exceeds 90%, a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 95%, a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_Disk I/O_**

  Category: [`server_disk`](metrics-server#server_disk)

  If the [Disk I/O utilization](../glossary#diskio)(`ioPercent`) exceeds 90%, a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 95%, a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_Disk Inode_**

  Category: [`server_disk`](metrics-server#server_disk)

  If the disk Inode utilization (`usedInodePercent`) exceeds 90%, a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 95%, a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_Memory_**

  Category: [`server_base`](metrics-server#server_base)

  If the memory utilization (`memory_pused`) exceeds 70%, a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 90%, a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_Network BPS_**

  Category: [`server_network`](metrics-server#server_network)

  If the sum of network inbound traffic (`trafficIn`) and outbound traffic (`trafficOut`) exceeds 150,000,000 bytes (150 MB), a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 200,000,000 bytes (200 MB), a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_Network IOPS_**

  Category: [`server_network`](metrics-server#server_network)

  If the sum of network inbound packet (`packetIn`) and outbound packet (`packetOut`) exceeds 45,000 bytes, a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 50,000 bytes, a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_Process CPU_**

  Category: [`server_process`](metrics-server#server_process)

  If the total CPU utilization (`cpu`) exceeds 90%, a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 95%, a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_Process Count_**

  Category: [`server_process`](metrics-server#server_process)

  If the number of processes (`count`) is 2 or more, a warning (<span class="slow">Warning</span>) level notification is sent. If it is less than 1, a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_Process Memory_**

  Category: [`server_process`](metrics-server#server_process)

  If the process memory utilization (`memory`) exceeds 90%, a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 95%, a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_Swap Memory_**

  Category: [`server_base`](metrics-server#server_base)

  If the swap memory utilization (`memory_swappused`) exceeds 70%, a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 90%, a critical (<span class="vslow">Critical</span>) level notification is sent.

:::note

For more information about the fields applied to the basic event templates, see [the following](metrics-server).

:::

  :::note

  This template is only available if you have configured and run the XOS agent. For more information about the XOS agent configuration, see [the following](agent-xos-settings).

  :::

  Category: [`db_mysql_counter`](metrics-data-list)

If it exceeds 100 seconds, a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_Jitter_**

  Category: [`npm_process_tag_data`](metrics-npm#npm_process_tag_data)

  If the average number of packet round trip delay changes (`avg_jitter`) exceeds 30, a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 60, a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_Latency_**

  Category: [`npm_process_tag_data`](metrics-npm#npm_process_tag_data)

  If the average time of packet round trip delays (`avg_latency`) exceeds 200 ms (0.2 seconds), a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 400 ms (0.4 seconds), a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_Loss Packet_**

  Category: [`npm_process_tag_data`](metrics-npm#npm_process_tag_data)

  If the lost packet count (`lost_count`) exceeds 1, a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 10, a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_Retransmission Packet_**

  Category: [`npm_process_tag_data`](metrics-npm#npm_process_tag_data)

  The retransmitted packet count (`retransmission_count`) exceeds 1, a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 10, a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_Session Count_**

  Category: [`npm_process_tag_data`](metrics-npm#npm_process_tag_data)

  The number of sessions excluding duplicates (`session_count`) exceeds 100, a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 1000, a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_Total Packet Count_**

  Category: [`npm_process_tag_data`](metrics-npm#npm_process_tag_data)

  If the sum of the transmitted packet count (`send_packet_count`) and the received packet count (`recv_packet_count`) exceeds 50,000, a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 100,000, a critical (<span class="vslow">Critical</span>) level notification is sent.

- **_Total Packet Size_**

  Category: [`npm_process_tag_data`](metrics-npm#npm_process_tag_data)

  If the sum of the data size for transmitted packets (`send_packet_size`) and that for received packets (`recv_packet_size`) exceeds 7,680,000,000 bytes (7.68 GB), a warning (<span class="slow">Warning</span>) level notification is sent. If it exceeds 15,360,000,000 bytes (15.36 GB), a critical (<span class="vslow">Critical</span>) level notification is sent.

:::note

For more information about the fields applied to the basic event templates, see [the following](metrics-npm).
