## Checking the session information details {#checksessioninfo}

### <Cmdname sid="session_detail" /> {#session-details}

This guides you to the <Cmdname sid="session_detail" className="uitext" /> window that you can view information by selecting a session in the following menu path.

- <Cmdname sid="side_dashboard" className="uitext" /> &gt; <Cmdname sid="side_dbMonitoring" className="uitext" />

- <Cmdname sid="side_dashboard" className="uitext" /> &gt; <Cmdname sid="side_multiInstance" className="uitext" />

- <Cmdname sid="side_analysis" className="uitext" /> &gt; <Cmdname sid="side_dbTrendComparison" className="uitext" />

- <Cmdname sid="side_analysis" className="uitext" /> &gt; <Cmdname sid="side_dbLockTree" className="uitext" />

- <Cmdname sid="side_analysis" className="uitext" /> &gt; <Cmdname sid="side_dbWaitAnalysis" className="uitext" />

When a session is selected in the session table list, the <Cmdname sid="session_detail" className="uitext" /> window appears. You can see the information details about the selected session.

<InDoc product="oracle-pro,oracle"><ImgLang img='db-session-details-info-oracle-pro.png' desc='Session detail' /></InDoc>

- The session information is viewed in real time by default. If you select ![Pause icon](/img/ico-pause.svg) in the time selector, you can stop a real-time view and see the data based on the paused time. To refresh data while pausing a real-time view, select ![Refresh icon](/img/ico-refresh.svg).

- ***Kill Session***: You can stop the selected session. If the <Cmdname sid="session_kill" className="uitext" /> window appears, enter the password (***Parameter Key***) and then select <Cmdname sid="apply" className="uitext" />.

  :::note

  This feature is only available in real-time viewing mode.

  :::

- <Cmdname sid="TTL06721" className="uitext" />: You can go to the <Cmdname sid="side_sessionHistory" className="uitext" /> menu to see the records of the selected session.

  :::note

  For more information about <Cmdname sid="side_sessionHistory" className="uitext" />, see <Link to="analysis-session-history">the following</Link>.

  :::

- ***Session Stat***: It provides various performance metrics and statistical information for the selected session. It includes the type and number of tasks performed in the session, resource usage, and more.

- ***Delta*** / ***Sigma***: The change rate and statistical analysis information for the selected session appear. You can check the amount of change in session activities.

- ***Session Info***: You can see the basic information of the selected session and the information details including the status.

- ***Wait Info***: The waiting event and waiting time of the selected session are provided. You can also see information about the waiting events that occur when the database system is waiting to execute a specific task.

- ***SQL Text*** / ***Plan***: You can see the SQL query statements and plans of the selected session. For more information, see [the following](#query-details).

### <Cmdname sid="TTL06224" /> {#query-details}

To check the SQL query information of the current session or past sessions, check the <UI>query_text</UI> column. The <Cmdname sid="TTL06224" className="uitext" /> window appears to display the query statement and SQL plan.

<ImgLang img="db-sql-details-new-oracle-pro.png" desc="SQL 상세" />

- <Cmdname sid="BTN06225" className="uitext" />: You can go to the <Cmdname sid="side_dbSqlStat" className="uitext" /> menu, where you can view statistical data based on the query.

- ***Formatting***: You can improve readability by applying indentation and formatting to a SQL query statement.

- ***Runtime Plan***: It provides the execution plan and runtime information for a selected SQL query. It provides the information details such as execution count, average execution time, and average physical reads.

  - <Cmdname sid="select_1_hour" className="uitext" /> / <Cmdname sid="select_1_day" className="uitext" />: It provides statistical information based on the past 1 hour or day.

- ***Explain Plan***: It displays the execution plan predicted by the optimizer. It provides information such as cost, job, object name, and cardinality.

- ***Plan History***: You can check the history of the execution plans of the SQL queries executed in the database.

- ***Bind Capture***: You can see the values ​​of the bind variables used in SQL queries executed in the database. This allows you to see actual content of query executions.

  :::note

  This is a value captured in the database (`v$sql_bind_capture`), not a bind value executed in real time. If the number of cases is too large, it displays up to 100.

  :::

### Tuning guide {#tunning-guide}

You can check the tuning guide in the Plan tree of the<Cmdname sid="TTL06224" className="uitext" /> window. Select ![Tuning icon](/img/ic-chat-red.svg). Proceed with database optimization by referring to the tuning guide provided.

<ImgLang img="db-sql-details-tunning-guide-oracle-pro.png" desc="Tunning guide" />

:::note

The tuning guide can be viewed when the nodes in the Plan Tree are `TABLE ACCESS (FULL)` and `INDEX (SKIP SCAN)` and the nodes contain the keyword, `CARTESIAN`.

:::
