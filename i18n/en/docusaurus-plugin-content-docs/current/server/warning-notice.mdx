## Adding a server event{#add-server-event}

Home &gt; Select Project &gt; <Cmdname sid="side_event" className="uitext" /> &gt; <Cmdname sid="side_eventSetting" className="uitext" /> &gt; <Cmdname sid="server" className="uitext" /> tab &gt; Select **_+_** <Cmdname sid="add_alert_setting" className="uitext" />

You can receive alerts by setting the global events for specific servers. 

<ImgLang img='server-server-event.png' desc='Adding a server event'/>

1. If the <Cmdname sid="add_alert_setting" className="uitext" /> window appears, enter the ![number1](/img/number-01.png) event <Cmdname sid="title" className="uitext" />.

2. Check the ![number2](/img/number-02.png) <Cmdname sid="sms_server_list" className="uitext" /> or select a desired server for monitoring through the search bar. 

3. In the ![number3](/img/number-03.png) <Cmdname sid="alert_rule" className="uitext" /> tab, set the <Cmdname sid="condition" className="uitext" />, <Cmdname sid="level" className="uitext" />, <Cmdname sid="sms_alert_duration" className="uitext" />, and <Cmdname sid="enable" className="uitext" /> of the desired <Cmdname sid="indicator" className="uitext" />.

   - <Cmdname sid="level" className="uitext" />: You can select a level of <Cmdname sid="sms_alert_warn" className="slow" /> and <Cmdname sid="sms_alert_fatal" className="vslow" />.

   - <Cmdname sid="sms_alert_duration" className="uitext" />: After the set <Cmdname sid="sms_alert_duration" className="uitext" />, the server status is set to <Cmdname sid="sms_alert_warn" className="slow" /> or <Cmdname sid="sms_alert_fatal" className="vslow" />.

   For guide to <Cmdname sid="indicator" className="uitext" />, see the following.

   - <Cmdname sid="restart" className="uitext" />: This event occurs when the server restarts.

   - <Cmdname sid="down" className="uitext" />: This is a communication failure event that occurs when normal data is not received from the agent for the <Cmdname sid="sms_alert_duration" className="uitext" />.

   - <Cmdname sid="port" className="uitext" />: Port access unavailable event. Registered agents in the project check each other's status through **10120 port**. This event occurs when communication fails for the <Cmdname sid="sms_alert_duration" className="uitext" />.

   - <Cmdname sid="cpu" className="uitext" />, <Cmdname sid="mem" className="uitext" />, <Cmdname sid="swap" className="uitext" />: Set the usage events such as CPU, memory, and SWAP memory.

   - <UI>steal</UI>: Set the event for CPU steal time. Only Linux-based OS can be set.

   - <Cmdname sid="diskIo" className="uitext" />, <Cmdname sid="diskQuota" className="uitext" />, <UI>inode</UI>: Set the event based on the disk usage, I/O usage, and Inode usage.

   - <Cmdname sid="networkIops" className="uitext" />, <Cmdname sid="networkBps" className="uitext" />: Set the network usage-related event. It can be set based on the traffic and packets per second.

4. Set the ![number 4](/img/number-04.png) <Cmdname sid="event_tag" className="uitext" />. 

   - If this tag is selected, notifications can be sent to project members and 3rd-party plug-ins with the corresponding tags. If the event receiving tag is not selected, alerts are sent to all project members. 

   - Click <span class="uitext">+</span> <Cmdname sid="add_tag" className="uitext" /> and then select target alert receivers in <Cmdname sid="event_tag_list" className="uitext" />. You can also select <span class="uitext">+</span> <Cmdname sid="create_new_eventtag" className="uitext" /> to add new tags.

   - In <Cmdname sid="event_statistic" className="uitext" /> &gt; <Cmdname sid="event_receive_configuration" className="uitext" />, you can set the tags in project members and 3rd-party plug-ins. For more information about <Cmdname sid="event_receive_configuration" className="uitext" />, see <Link to="set-receive-event#set-notice-by-team-or-users">the following</Link>.

5. If you finished the event configuration, select ![number5](/img/number-05.png) <Cmdname sid="save" className="uitext" />.

:::caution

One event can be assigned to a server.

:::

## Adding process event{#process-event}

Home &gt; Select Project &gt; <Cmdname sid="side_event" className="uitext" /> &gt; <Cmdname sid="side_eventSetting" className="uitext" /> &gt; <Cmdname sid="process" className="uitext" /> tab &gt; Select **_+_** <Cmdname sid="add_alert_setting" className="uitext" />

You can receive alerts by setting the events for specific processes. If you specify conditions based on the key metrics such as <Cmdname sid="processcount" className="uitext" />, <Cmdname sid="processcpu" className="uitext" /> usage, and <Cmdname sid="processmem" className="uitext" /> usage, you can prevent the processes from terminating unexpectedly or experiencing rapid performance fluctuations. By doing so, you can receive alerts in real time.

You can assess the severity of events by setting the warning and critical levels and the <Cmdname sid="sms_alert_duration" className="uitext" /> for which the set thresholds remain in effect. You can also send alerts to specific project members by using the <Cmdname sid="event_tag" className="uitext" />. This allows you to efficiently take prioritized actions.

   For guide to <Cmdname sid="indicator" className="uitext" />, see the following.

   - <Cmdname sid="processcount" className="uitext" />: You can set an event for the number of processes with the same name. Set the number of <Cmdname sid="sms_alert_fatal" className="uitext" /> and <Cmdname sid="sms_alert_warn" className="uitext" /> conditions.
   - <Cmdname sid="processcpu" className="uitext" />: You can set the events depending on the CPU usage.
   - <Cmdname sid="processmem" className="uitext" />: You can set the events depending on the memory usage.

   - If this tag is selected, notifications can be sent to project members and 3rd-party plug-ins with the corresponding tags. If the event receiving tag is not selected, alerts are sent to all project members. 

   - Click <span class="uitext">+</span> <Cmdname sid="add_tag" className="uitext" /> and then select target alert receivers in <Cmdname sid="event_tag_list" className="uitext" />. You can also select <span class="uitext">+</span> <Cmdname sid="create_new_eventtag" className="uitext" /> to add new tags.

   - In <Cmdname sid="event_statistic" className="uitext" /> &gt; <Cmdname sid="event_receive_configuration" className="uitext" />, you can set the tags in project members and 3rd-party plug-ins. For more information about <Cmdname sid="event_receive_configuration" className="uitext" />, see <Link to="set-receive-event#set-notice-by-team-or-users">the following</Link>.

- If you have incorrectly set the <Cmdname sid="process" className="uitext" /> and <Cmdname sid="process_owner" className="uitext" />, disable these alerts to cancel activation, and then set the alert ***conditions*** again.

:::

## Adding the log files and Windows events

Home &gt; Select Project &gt; <Cmdname sid="side_event" className="uitext" /> &gt; <Cmdname sid="side_eventSetting" className="uitext" /> &gt; <Cmdname sid="sm_log_event_tab" className="uitext" /> tab &gt; Select **_+_** <Cmdname sid="add_alert_setting" className="uitext" />

You can manage specific log files and Windows event settings.

### Log File{#log-file-event}

To add a <Cmdname sid="log_file" className="uitext" /> event, set the <Cmdname sid="metrics_event_rules" className="uitext" /> seeing the following.

<ImgLang img='server-log-file.png' desc='Adding log file event'/>

1. If the <Cmdname sid="add_alert_setting" className="uitext" /> window appears, enter the ![number1](/img/number-01.png) event <Cmdname sid="title" className="uitext" />.

2. Check the ![number2](/img/number-02.png) <Cmdname sid="sms_server_list" className="uitext" /> or select a desired server for monitoring through the search bar. 

   :::note

   When enabling the **_Project Default Policy_** option ![Icon](/img/ic-toggle-on.svg), apply the default alert policy to the project administration server.

   :::

3. ![number 3](/img/number-03.png) In the <Cmdname sid="alert_rule" className="uitext" /> tab, select <Cmdname sid="log_file" className="uitext" /> in <Cmdname sid="sms_alert_log_rule_type" className="uitext" />.

4. See the example and then enter the <Cmdname sid="sms_alert_log_rule_file_path" className="uitext" /> following a regular expression rule. 

   > e.g. _/var/log/mylogfile_%Y-%m-%d_%H:%M:%S.log_

5. Enter the <Cmdname sid="sms_alert_log_rule_keyword" className="uitext" /> to trigger events. Multiple items can be specified.

6. Enter the <Cmdname sid="sms_alert_log_rule_keyword_exclude" className="uitext" /> to be excluded from alerts. Multiple items can be specified.

7. Enter the <Cmdname sid="sms_alert_log_rule_keyword_option_repeat" className="uitext" /> and <Cmdname sid="sms_alert_log_rule_keyword_option_duration" className="uitext" /> for alerts how many seconds and how many times when detecting the specified **keyword**. 

8. Select an event <Cmdname sid="sms_alert_severity" className="uitext" />. You can set the level of <Cmdname sid="sms_alert_fatal" className="vslow" /> or <Cmdname sid="sms_alert_warn" className="slow" />.

9. Select ![number4](/img/number-04.png) <Cmdname sid="save" className="uitext" />.

### Window Event{#windows-event}

To add an <Cmdname sid="window_event" className="uitext" />, set the <Cmdname sid="metrics_event_rules" className="uitext" /> seeing the following. It is available only in the Windows environment.

<ImgLang img='server-log-windows.png' desc='Adding Windows events'/>

1. If the <Cmdname sid="add_alert_setting" className="uitext" /> window appears, enter the ![number1](/img/number-01.png) event <Cmdname sid="title" className="uitext" />.

2. Check the ![number2](/img/number-02.png) <Cmdname sid="sms_server_list" className="uitext" /> or select a desired server for monitoring through the search bar. 

   :::note

   When enabling the **_Project Default Policy_** option ![Icon](/img/ic-toggle-on.svg), apply the default alert policy to the project administration server.

   :::

3. ![number 3](/img/number-03.png) In the <Cmdname sid="alert_rule" className="uitext" /> tab, select <Cmdname sid="window_event" className="uitext" /> in <Cmdname sid="sms_alert_log_rule_type" className="uitext" />.

4. Select <Cmdname sid="sms_alert_log_rule_event_category" className="uitext" /> by referencing the following example.

   <ImgLang img='server-log-windows-a-01.png' desc='Log name'/>

5. Noting the following example, set the values for <Cmdname sid="sms_alert_log_rule_event_level" className="uitext" />, <Cmdname sid="sms_alert_log_rule_eventsource" className="uitext" />, <Cmdname sid="sms_alert_log_rule_eventid" className="uitext" />, and <Cmdname sid="sms_alert_log_rule_keyword" className="uitext" />. For the <Cmdname sid="sms_alert_log_rule_keyword" className="uitext" />, multiple items can be specified.

   <ImgLang img='server-log-windows-a-02.png' desc='Level, event source, etc'/>

6. Enter the <Cmdname sid="sms_alert_log_rule_keyword_exclude" className="uitext" /> to be excluded from alerts. Multiple items can be specified.

7. Select an event <Cmdname sid="sms_alert_severity" className="uitext" />. You can set the level of <Cmdname sid="sms_alert_fatal" className="vslow" /> or <Cmdname sid="sms_alert_warn" className="slow" />.

8. Select ![number4](/img/number-04.png) <Cmdname sid="save" className="uitext" />.

#### Log File Throughput

The file log throughput is set as follows based on the log lines. This is to prevent excessive CPU use of the WhaTap server monitoring agent when too many logs occur. Excessive amount is not processed.

- Version 1.3.6 or earlier: Exceeding 60 lines per 2 seconds are processed. 
- Version 1.3.7 or later: Exceeding 1000 lines per 2 seconds are processed. 

You can adjust the excess per 2 seconds with the following option.

- **Linux**

  > `log.skip.threshold=number of lines that can be processed per 2 seconds` in the _/usr/whatap/infra/conf/whatap.conf_ file

- **Windows**

  > `log.skip.threshold=number of lines that can be processed per 2 seconds` in the _C:\Program Files\WhatapInfra\whatap.conf_ file

## Modifying an event{#event-edit}

1. On the right of the event list, select ![Edit icon](/img/ico-edit.svg) <Cmdname sid="edit" className="uitext" />. 

2. In the <Cmdname sid="alert_rule" className="uitext" /> tab, modify the <Cmdname sid="condition" className="uitext" />, <Cmdname sid="level" className="uitext" />, <Cmdname sid="sms_alert_duration" className="uitext" />, and <Cmdname sid="enable" className="uitext" /> of the desired <Cmdname sid="indicator" className="uitext" />.

3. After modification, select <Cmdname sid="save" className="uitext" />.

:::note

- If there is any modification for a specific metric, all active alerts for the metric are <span class='vslow'>disabled</span>.
- After modification, a new alert is triggered when any data for the set conditions is received.
- If any process alert data of <Cmdname sid="title" className="uitext" />, <Cmdname sid="process" className="uitext" />, and <Cmdname sid="process_owner" className="uitext" /> has been modified, all alerts for the condition are <span class='vslow'>disabled</span>.

:::

## Deleting an event{#event-delete}

1. On the right of the event list, select ![Edit icon](/img/ico-edit.svg) <Cmdname sid="edit" className="uitext" />. 

2. On the upper right of the modification screen, select ![Delete icon](/img/ic-delete-critical.svg) <Cmdname sid="delete" className="uitext" />.

## Copying an event{#event-copy}

1. On the right of the event list, select ![Copy icon](/img/ico-copy.svg) <Cmdname sid="copy" className="uitext" />.

2. When the <Cmdname sid="TTL06597" className="uitext" /> window appears, select a desired project or select <Cmdname sid="BTN06571" className="uitext" /> at the bottom.

3. If a project to copy the event settings to has been set, select <Cmdname sid="copy" className="uitext" />.

:::note

- When duplicate events are copied within the same project, as many events as the number of copies are created.

- The copied event name is added with the suffix **_(copy)_**. 
  > e.g. Previous event name **_Default_** → Copied event name **_Default(copy)_**

:::
