Home &gt; Select Project &gt; <Cmdname sid="side_report" className="uitext" />

If you click ![down-arrow](/img/down-arrow.svg) in <Cmdname sid="report" className="uitext" /> on the screen, you can see various analysis reports classified by daily, weekly, and monthly. Select a desired report format, and then set the date and time to view and filtering options. You can check the selected report by selecting ![Search icon](/img/ico-btn-search.png).

## Daily server report {#S001_ServerDaily_TagReport}

In <Cmdname sid="S001_ServerDaily_TagReport" className="uitext" report />, you can see the usage of server resources for the day. You can search by setting the start date, start time, and end time.

### Intro {#overview}

<ImgLang img='server_daily_report_01.png' desc='Daily Server Report summary'/>

You can see a summary of project names, number of hosts, and number of alerts. For the numbers of hosts and notifications, you can see how much has changed compared to the previous day.

- <Cmdname sid="host_count" className="b600" report/>: Total number of servers during the query period
- <Cmdname sid="alert_count" className="b600" report/>: Total number of alerts during the query period

### Server list {#server_list}

<ImgLang img='server_daily_report_02.png' desc='Daily Server Report - Server lists'/>

You can check the daily key metrics for each server. In case of the maximum CPU (CPU Max) and maximum memory (MEM Max), the time at which the corresponding figures are displayed appears together.

The numbers of alerts can be compared by retrieving data of 7 days ago, 1 day ago, and the day of inquiry.

:::note

<span class='loger'>Red</span> appears when the average and maximum values for CPU and memory are above 90%.

:::

### Alert history {#alert_history}

<ImgLang img='server_daily_report_03.png' desc='Alert List'/>

You can check the occurrence time, rating, server name, and cause of the alert in the notification history during the query period.
For more information about each alert, see <Cmdname sid="side_event" className="uitext" /> &gt; <Cmdname sid="side_eventHistory" className="uitext" />.
If there is no alert record for the project, the message, "<Cmdname sid="no_data" className="uitext" report />" may appear.

:::note

According to the notification level, <span class='loger'>Critical</span> notifications appear in <span class='loger'>red</span>, <span class='wer'>Warning</span> notifications <span class='wer'>orange</span>, and <span class='mstop'>Info</span> notifications <span class='mstop'>gray</span>.

:::

## Weekly server report {#S100_ServerWeeklyReport}

In <Cmdname sid="S100_ServerWeeklyReport" className="uitext" report/>, you can see the usage of server CPU and memory for the week.
You can enter the start date and see from 7 days prior to the day of inquiry.
It searches data for 7 days from the start date.

### Full resource usage list {#total_resource_used_list}

#### <Cmdname sid="_cpu" report/>

<ImgLang img='server_weekly_report_cpu.png' desc='Weekly total resource usage list - CPU'/>

It views the CPU utilization for each server by day during the query period.
Queried items are CPU average, 95% of maximum, and maximum. Each value is represented as a percentage.

:::note 

**95% of maximum**

It indicates the maximum value excluding the top 5% data. This is used to exclude the momentary high figures due to any reason such as server restart.

:::

#### <Cmdname sid="_mem" report/>

<ImgLang img='server_weekly_report_memory.png' desc='Weekly total resource usage list - Memory'/>

It views the memory usage for each server by day during the query period.
Queried items are memory average and maximum value. Each value is represented as a percentage.

#### <Cmdname sid="_disk" report/>

<ImgLang img='server_weekly_report_disk.png' desc='Weekly total resource usage list - Disk'/>

Metrics per file system mounted on each server during the query period.
It views the mount location, total disk size, and utilization. Each value is represented as a percentage.

:::note

- If the value is 90% or more, it is displayed in <span class='loger'>red</span>. 
- `okindName` appears in the group name, and may be blank if `okindName` is not set.

:::

## Monthly server report {#S200_ServerMonthlyReport}

In <Cmdname sid="S200_ServerMonthlyReport" className="uitext" report />, you can see the usage and data of server resources for the month.
You can enter the start date and see from 30 days prior to the day of inquiry.

### Intro {#overview}

<ImgLang img='server_monthly_report_01.png' desc='Monthly server report - summary'/>

You can see the aggregate the usage of server resources for the month.

- <Cmdname sid="total_host" className="b600" report/>: Total number of servers during the query period
- <Cmdname sid="total_core" className="b600" report/>: Total number of cores during the query period
- <Cmdname sid="total_disk" className="b600" report/>: Size of the mounted file systems for the query period
- <Cmdname sid="total_alarms" className="b600" report/>: Total number of alerts during the query period

### Server information {#server_info}

<ImgLang img='server_monthly_report_02.png' desc='Server information'/>

It displays the IP address, OS data, CPU data, and memory usage of the server run during the query period.

### Alert details {#alarms_list}

<ImgLang img='server_monthly_report_03.png' desc='Alert History'/>

You can check the total number of alerts for all servers during the query period, and you can view the alerts generated by server for each week.

### List of servers in use {#server_usage_history}

<ImgLang img='server_monthly_report_04.png' desc='Server Usage List'/>

You can check the CPU average usage and its maximum value, and memory average usage and its maximum value for each server at a glance.

:::note

If the value is 90% or more, it is displayed in <span class='loger'>red</span>. 

:::
