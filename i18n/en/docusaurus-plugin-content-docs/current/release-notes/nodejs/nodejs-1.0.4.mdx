October 16, 2025

- <Status priority='1'>New</Status> **hook_method_patterns**

    - Option that allows monitoring of specific methods in user applications.

    - When the method path is specified in the configuration file, the execution time and call arguments of that method are collected.

    - Default value: `false`

        :::note

        **Configuration Example**  
        - Class method: `./path/filename/className.methodName`  
        - Regular function: `./path/filename/functionName`  
        - Use commas to specify multiple items.

        :::

        :::caution

        **Caution**  
        - For TypeScript projects, specify the built file path (for example, `./dist/services/user.service`).  
        - Restart the application if you remove this option after enabling it.

        :::

- <Status priority='0'>Fixed</Status> Improved management of `whatap_nodejs` module processes when running multiple instances in PM2 cluster mode.

    - Only the first instance starts the `whatap_nodejs` agent.  
    - The remaining instances automatically connect to the agent already running.
