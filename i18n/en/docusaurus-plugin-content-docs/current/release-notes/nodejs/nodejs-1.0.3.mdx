September 25, 2025

- <Status priority='1'>New</Status> **exit_with_parent_process_enabled**
    - `false`: default
    - `true`: when the application exits, the whatap_nodejs process terminates

        :::caution

        **Caution**  
        Node.js Agent v1.0.3 or higher

        :::

- <Status priority='1'>Changed</Status> Introduced an independent collector process

    - **Improved application name configuration**

        - ONAME setting: You can specify the application name with the `app_name` setting in *whatap.conf*.

            ```bash
            ## whatap.conf
            app_name=MyNodeApp              # Application name, default: NODE
            app_process_name=node           # Process name (for example, node)

            ## Enter the application's root path in the WHATAP_HOME environment variable.
            ## Default: process.cwd()
            export WHATAP_HOME={app_root_path}
            ```

    - **New data collection architecture**

        - Independent collector process: A separate `whatap_nodejs` process aggregates data and sends it to the collection server.
        - Improved performance: It runs as a process independent of the Node.js application, which minimizes impact on main application performance.

    - **Improved PM2 cluster mode support**

        - Unified monitoring: When two or more instances run in PM2 cluster mode, they are consolidated and displayed under one `app_name`.
        - Efficient resource management: A single collector process handles data from multiple instances to optimize resource usage.

        :::note
        **Note**  
        Even when you run with PM2, set `app_process_name` to `node`. This is for identification in WhaTap monitoring rather than the actual process name.
        :::

    - **Compatibility**
        - **Node.js version**: 17.0.0 or higher
        - **Platform**: macOS (64bit), Linux (64bit)
        - **Supported frameworks**: Express, Koa, Fastify, Socket.IO, WebSocket, GraphQL
        - **Supported databases**: MySQL, PostgreSQL, MSSQL, Oracle, MariaDB, MongoDB, Redis

- <Status priority='0'>Fixed</Status> Fixed an issue where the Container ID was not mapped in Kubernetes container environments

    :::caution

    **Caution**  
    Supported from Kubernetes agent v1.8.8 or higher

    :::
