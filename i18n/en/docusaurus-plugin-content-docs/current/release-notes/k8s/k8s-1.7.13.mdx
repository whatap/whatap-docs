October 21, 2024

- <Status priority='1'>New</Status> Added the <Cmdname sid="side_analysis" className="uitext" /> > <Cmdname sid="side_metricsSearch" className="uitext" /> `kube_ingress` category.

  It collects the metadata and the related information for Ingress resources. This category is scheduled to be used for implementation of dashboards related to Ingress lists and routing policies.

  :::note 

  For more information about the Ingress (`kube_ingress`) category, see [the following](../../kubernetes/metrics-kubernetes/#kube_ingress). 
  ::: 

- <Status priority='1'>New</Status> Added the container identification-related metrics in the <Cmdname sid="side_analysis" className="uitext" /> > <Cmdname sid="side_metricsSearch" className="uitext" /> `kube_oom_event` category.

  You can identify containers when an OOMKilled event occurs by adding values for ***namespace***, ***podUid***, and ***containerName*** as ***tags***.

- <Status priority='1'>New</Status> Added the port-related metrics to the <Cmdname sid="side_analysis" className="uitext" /> > <Cmdname sid="side_metricsSearch" className="uitext" /> `container` category.

  The metrics related to **containerPort** used by containers have been added. Kubernetes services perform routing based on container ports within the Pod, so these metrics can be used to express relationships between services, Pods, and containers.

:::info
