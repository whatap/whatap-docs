> **Includes Java Agent v2.2.52**

February 17, 2025

- <Status priority='1'>New</Status> Supported for collecting metrics related to Kubernetes custom resources via additional features.

  - Added the `collect_custom_resources` option to the whatap-master-agent's container by applying additional features instead of basic features.

    **collect_custom_resources** <span class='type'>bool</span>

    Default `false`

    When set to `true`, you can check the `kube_cr` and `kube_crd` categories in the <Cmdname sid="side_metricsSearch" className="uitext" /> menu. The collected data can be seen in <Cmdname sid="side_objectManifest" className="uitext" /> and **Tag count**. 

    :::note 

    - `kube_crd`: It collects the CustomResourceDefinition (CRD) metrics.

    - `kube_cr`: It collects the custom resource (CR) metrics defined as CRD.

    :::

- <Status priority='1'>New</Status> Improved the GPU monitoring by adding the `DCGM_FI_DEV_POWER_USAGE` data to the `container_gpu` category.

- <Status priority='1'>New</Status> Added the `#whatapSysLog` category in the log to provide the user environment's runtime information when running the node agent container.

- <Status priority='0'>Fixed</Status> Fixed the `kube_pod_volume` related bugs.

  - Fixed the bug where all data is collected regardless of the cluster project or namespace.

  - Fixed some data missing cases caused by synchronization issues.

- <Status priority='0'>Fixed</Status> Fixed the issue where container image names are not parsed correctly.

:::info

:::
