March 27, 2025

- <Status priority='1'>Feature</Status> Added the status code collection feature and expanded the exception tracing scope in `undertow-server 2.3.0` or later.

- <Status priority='1'>Changed</Status> Changed not to generate the traceparent header if there is no `mtid`.

- <Status priority='0'>Fixed</Status> Fixed the issue of transaction data being collected repeatedly when implementing `spring-webflux 6.0` or later under `undertow-server 2.3.0` or later.

  ```makefile
  ## example using spring-boot-3.4.3 on undertow-2.3.18.Final
  weaving=spring-boot-3.2,undertow-2.3.0  
  ```

- <Status priority='0'>Fixed</Status> Fixed the issue of transaction data being collected repeatedly when calling the Jakarta Filter under `undertow-server 2.3.0` or later.

- <Status priority='0'>Fixed</Status> Fixed the issue where the collection server is not connected when `whatap.oname` has not been specified when running `tomcat` or `embedded-tomcat`.

- <Status priority='1'>Deprecated</Status> Ended the configuration support of `webflux`, and the existing configuration has been replaced with the `spring-boot` configuration.
