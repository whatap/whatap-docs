June 25, 2025

* <Status priority='1'>New</Status> Added the <code>app_context_stat</code> category to matrix metrics.

    * **app_context_enabled** <span class='type'>Boolean</span> 

        * Default: <code>false</code>

        * When set to <code>true</code>, enables the full <code>app_context_stat</code> data collection feature.
    
    * **app_context_parser**
    
        * Use one of: <code>default</code>, <code>prefix</code>, <code>match</code>

        * <code>default</code>: Default parser (extracts the first path segment)

        * <code>prefix</code>: Prefix-matching parser (most flexible)

        * <code>match</code>: Exact-matching parser (matches only fully identical paths)
    
    * **app_context_parser_reset** <span class='type'>Number</span> 
    
        * Default: <code>0</code>

        * A counter that forcibly resets the parserâ€™s internal cache and configuration.
    
    * **app_context_path_depth** <span class='type'>Number</span> 
    
        * Default: <code>1</code>

        * Specifies the depth of path segments to extract from the URL (used only with the default parser)

    * **app_context_path_set** <span class='type'>String</span> 

        * Defines a mapping of monitored paths and their display names. Use a comma (<code>,</code>) to separate multiple values.

        * Example: <code>name1@url1,name2@url2,url3,name4@url4</code>

* <Status priority='1'>New</Status> Added an option to display the original URL in step traces when URL normalization is applied.

    * **step_origin_url_enabled** <span class='type'>Boolean</span> 
    
        * Default: <code>false</code>

* <Status priority='0'>Fixed</Status> Fixed an issue where the original URL was not displayed in record summaries when URL normalization was applied.
