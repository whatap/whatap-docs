## Service 2.7.0

<div class='indentTab'>

### Release of new features

It provides a guide to the new features released in Service 2.7.0.

#### `Database`

- <Status priority='1'>New</Status> **Oracle V2** services now launched!!

  - The UIs and features of <Cmdname sid="side_dashboard" className="uitext" /> have been greatly improved.

  - You can see the database performance metrics in <Cmdname sid="side_dashboard" className="uitext" />.

  - Added the <Cmdname sid="side_log" className="uitext" /> menu that can search for database logs and their trend and configure the log notifications.

  :::note

  - For more information about new features in **Oracle V2**, see [the following](../../oracle/monitoring-intro#new-feature).

  - The minimum version of the DBX agent to use the **Oracle V2** is **1.9.0**. For more information about the agent update, see [the following](../../oracle/agent-manage).

  :::

- <Status priority='1'>New</Status> **Oracle Pro** services now launched!!

  This approach uses the Direct Memory Access (DMA) technique to directly access the server's memory and extract the required data. This method minimizes or does not cause load on the database server. By accessing memory directly, data can be collected almost in real time, allowing to monitor the system performance more precisely and to immediately respond upon any problem.

  :::note

  For more information about **Oracle Pro**, see [the following](../../oracle-pro/monitoring-intro).

  :::

#### `Kubernetes`

- <Status priority='1'>New</Status> Added the <Cmdname sid="side_kubeServiceList" className="uitext" /> menu that provides comprehensive status and information on all services running in the Kubernetes cluster.

  :::note

  - This feature can be seen in the <Cmdname sid="side_workload" className="uitext" /> &gt; <Cmdname sid="side_kubeServiceList" className="uitext" /> path. You can understand the network configuration within the cluster by detailing the service configuration and current status.

  - For more information about the <Cmdname sid="side_kubeServiceList" className="uitext" />, see [the following](../../kubernetes/service-list).

  - This function is supported in Kubernetes master agent 1.7.8 or later.

  :::

- <Status priority='1'>New</Status> Added the <Cmdname sid="side_nodeDiskList" className="uitext" /> menu for disk monitoring from the node perspective.

  :::note

  - This feature can be seen in the <Cmdname sid="side_cluster" className="uitext" /> &gt; <Cmdname sid="side_nodeDiskList" className="uitext" /> path.

  - For more information about the <Cmdname sid="side_nodeDiskList" className="uitext" />, see [the following](../../kubernetes/node-disk-list).

  - This function is supported in Kubernetes master agent 1.7.8 or later.

  :::
