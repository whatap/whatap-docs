April 3, 2025

- <Status priority='1'>New</Status> Added the option to handle code conversion when Korean characters are broken.

  - Defined character encoding for `codefrom=` and `codeto=`.

  - Selected and specified out of the results of the `iconv -l` command.

  - e.g. `codefrom=EUC-KR`, `codeto=UTF-8`

- <Status priority='1'>Feature</Status> Changed the default value for `plan_elapse` and `bind_elapse` from 5 seconds to 3 seconds.

- <Status priority='1'>Feature</Status> Changed the default `sqlstat` transmission cycle from 5 minutes to 1 minute.

- <Status priority='1'>Feature</Status> Added `!rectype`, `@id`, and `@stime` to `sqlstat`.

- <Status priority='0'>Fixed</Status> Fixed the issue where parallel query information is not collected in version 19 or later.

- <Status priority='0'>Fixed</Status> Fixed the issue where metrics are not searched by multi DB when there is only one pdb.

- <Status priority='0'>Fixed</Status> Added `result2` to `session list request/reply` that calls the agent in real time.

- <Status priority='0'>Fixed</Status> Fixed the issue where the `temp tablespace` value is displayed abnormally.

  - Modified the collection query logic with `distinct` because of Oracle bugs (18c, 19c).

- <Status priority='0'>Fixed</Status> Reorganized the `_no_5m_hour_` category to avoid unnecessary 5-minute data generation.

- <Status priority='0'>Fixed</Status> When searching for plans in real time by using the `plan_hash` value, modified to search using the latest `timestamp` value if there is no `child_number`.
