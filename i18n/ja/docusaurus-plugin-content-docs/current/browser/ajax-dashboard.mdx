AJAXとは、非同期ジャバスクリプトとXML(Asynchronous JavaScript And XML)です。 サーバーと通信するために`XMLHttpRequest`オブジェクトの使用を意味します。 

AJAXは、リソース データの動的な画面を構成する上で重要なデータロード方式です。 AJAXが正常に呼び出されない場合、エンドユーザーに画面が正常に表示されません。 当社が収集する情報は、ユーザー体験に悪影響を及ぼしているものを特定するのに役立ちます。

## AJAX件数

<ImgLang img='rum-ajax-count.png' desc='AJAX件数' />

リアルタイムでAJAXのロード数を確認することができます。

- ウィジェットの右上にある ![詳細を見るアイコン](/img/ico-detail-mode.svg)ボタンを選択し、時間帯を設定してAJAXのロード数を表示することができます。 
- データ検索基準を ***AJAX Host*** または ***全体*** AJAXに変更するにはウィジェット右上の ***AJAX Path*** ボタンを選択します。
- ***AJAX Path***、***AJAX Host*** を基準にAJAXロード数の参照時、上位10個に該当するAJAX件数を提供します。
- ウィジェットチャートにカーソルを合わせると、ツールチップの形でAJAX件数情報を表示されます。

- ウィジェットの右上にある ![詳細を見るアイコン](/img/ico-detail-mode.svg)ボタンを選択し、時間帯を設定してAJAXレスポンス時間を表示することができます。 
- データ検索基準を ***AJAX Host*** または ***全体*** AJAXに変更するにはウィジェット右上の ***AJAX Path*** ボタンを選択します。
- ***AJAX Path***、***AJAX Host*** を基準にAJAXロード数の参照時、上位10個に該当するAJAXレスポンス時間を提供します。

## 状態別Ajax

<ImgLang img='rum-ajax-status.png' desc='状態別Ajax' />

時間帯のAJAXに関連したHTTPステータスコードを確認することができます。

<ImgLang img='rum-ajax-status-count.png' desc='状態別Ajax' />

AJAXに関連するHTTPステータスコードを参照する時間基準の累積件数を確認することができます。

AJAXに関するステータスは次のとおりです。 次の基準で、WebアプリケーションでAJAXが正常に動作するか確認してください。

- **1xx**(情報レスポンス): サーバーがリクエストを受信し、サーバーに接続されているクライアントは、タスクを続行する必要があることを意味します。

- **2xx**(成功): リクエストの受信に成功して、認識、処理されました。

- **3xx**(リダイレクション): リクエスト完了するには、追加の措置が必要です。

- **4xx**(クライアントエラー): リクエストの文法が間違っているか、リクエストを処理できません。

- **5xx**(サーバーエラー): サーバーが明らかに有効なリクエストを満たすことができませんでした。

- **Request Fail**(リクエスト失敗): サーバーからレスポンス値を受信できず、キャンセルされた場合です。 例えば、タイムアウトでフロントデスクのリクエストをキャンセルする場合、間違った方法でリクエストをする場合などがあります。

:::note

HTTPステータスコードの詳細については、[次の文書](../url/url-http-status)を参照します。

:::

## AJAXエラー件数

<ImgLang img='rum-ajax-error-count.png' desc='AJAXエラー件数' />

リアルタイムでAJAXに関連したエラー件数を確認することができます。

- ウィジェット右上の ![詳細を見るアイコン](/img/ico-detail-mode.svg)ボタンを選択し、タイムゾーンを設定してAJAXエラーのロード件数を表示することができます。 
- データ参照基準を ***AJAX Host*** または***ページグループ***に変更するにはウィジェットの右上にある ***AJAX Path*** ボタンを選択します。
- ***ページグループ***を基準に検索する場合、ウィジェットチャートにカーソルを合わせると、ツールチップを使用して特定のページグループから発生したAJAXエラーを特定できます。
