<Cmdname sid="side_pageLoadStatistics" className="uitext" />メニューは、ウェブサイト上のページごとのロード性能情報を提供します。 このメニューを通じて各ページがロードされる過程を詳細に追跡でき、性能が低下したページを識別し、改善されたページの成果を分析できます。 また、ウェブアプリケーションのユーザーエクスペリエンスを改善するための有用な情報を取得できます。

- ページURLパスごとのロード性能の統計を提供します。

- 平均値、最大値、最小値、百分位数ベースの統計を確認できます。

- ページロードプロセスの詳細時間のフィルターを使用して、性能を集中的に分析できます。

- Navigation Timing APIを通じてページロードの各段階別時間を収集し、統計を提供します。

:::note

<Cmdname sid="side_pageLoadStatistics" className="uitext" />メニューで収集するデータの詳細については、[次の文書](collect-data#pageloadtime)を参照してください。

:::

## 基本的な使い方

ページのURL経路を基準に**ページロード時間**と**DOMロード時間**、**ネットワーク遅延時間**、**リソースロード時間**など様々な性能メトリクスを確認できます。 これにより、各ページの性能状態を評価し、最適化可否を判断できます。

<ImgLang img="browser-pageload-statistics.png" desc="ページロード統計" />

1. <Cmdname sid="time" className="uitext" />で性能メトリクスを照会する日時を設定します。

2. <Cmdname sid="TTL08276" className="uitext" />でご希望の基準を選択します。

3. 画面右上の![検索アイコン](/img/ic-search-on-press.svg)ボタンを選択します。

照会した時間範囲の性能メトリクスを各セクションでグラフチャートと一覧を通じて確認できます。 各セクションのグラフチャートでマウスをオーバーすると、その時間帯のメトリクス情報を確認できます。

:::note

<Cmdname sid="TTL08276" className="uitext" />についての詳細は、[次の文書](#statcriteria)を参照してください。

:::

### 統計基準を選択する{#statcriteria}

<Cmdname sid="TTL08276" className="uitext" />は、ユーザーが分析しようとする統計メトリクスを次に選択した基準で確認できるオプションです。

- <Cmdname sid="merge_avg" className="uitext" />: 各ページの平均ロード時間を計算して統計を提供します。

- <Cmdname sid="merge_max" className="uitext" />: 各ページの最大ロード時間に基づいて統計を提供します。 つまり、ページロード時間が最も非効率的なページURLを把握できます。

- <Cmdname sid="merge_min" className="uitext" />: 各ページの最小ロード時間に基づいて統計を提供します。 これにより、最適な条件でロードされたページURLを確認できます。 または、性能の最適化目標を設定する時に役立ちます。

- <Cmdname sid="ITM08291" className="uitext" />: 各ページの百分位数ベースのロード時間の統計を提供します。

  - <Cmdname sid="TTL08288" className="uitext" replace="95" />: 最も遅い5%のページロード時間であり、ページロード時間が最も遅かった上位5%のユーザー経験を意味します。

  - <Cmdname sid="TTL08288" className="uitext" replace="90" />: 最も遅い10%ユーザーのページへのロード時間で、主に遅いユーザーグループの性能問題を特定する時に役立ちます。

  - <Cmdname sid="TTL08288" className="uitext" replace="75" />: ページのロード時間が下位75%より遅い上位25%のユーザーのロード時間をを表示します。

  - <Cmdname sid="TTL08288" className="uitext" replace="50" />: 中間値であり、全体ユーザーの半分はその時間より早くページをロードし、半分はより遅くロードしたことを意味します。
