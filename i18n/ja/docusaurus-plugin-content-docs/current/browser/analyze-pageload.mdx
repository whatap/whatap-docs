**[ページ読み込み](before-starting#load-page)**イベントは、ウェブ アプリケーションを訪問したエンド ユーザーがページの読み込みを完了したときに発生するイベントです。 **ページ読み込み**イベントをヒートマップチャート形式で提供し、各イベントに関する詳細情報をタイムライン形式で確認することができます。 これにより、ウェブページの読み込みに障害を発生させる要素を把握できます。 次の情報を含みます。

-   **ページロード** イベント区間別の詳細時間情報

-   ロードしたリソースの詳細

-   ブラウザエラーデータ

## ページロードイベント一覧確認{#pageload-events-list}

単一のページごとに読み込みイベントを分析できます。 ページ読み込みの性能を分析するために、ヒートマップと様々な指標を提供します。

![ページロードヒートマップ](/img/browser-analysis-pageload-hitmap.png)

![指示線1](/img/number-01.png) **_ページ読み込みヒートマップ_**ウィジェットのチャート領域をドラッグしてください。 ドラッグした領域のページロードイベント一覧を画面下の ![指示線 2](/img/number-02.png) **_ページ一覧_**に読み込み、**[ページグループ](before-starting#page-group)**別に分類して、画面の右側の ![指示線 4](/img/number-04.png) **_ページグループ一覧_**に読み込みます。 ![指示線3](/img/number-03.png) **_ページURL検索_**で希望するページの一覧を検索できます。

### ![](/img/number-01.png)ページロードヒートマップ{#pageload-hitmap}

**_ページロードヒートマップ_** ウィジェットは、時間に応じた **ページロード**レスポンス時間を分布図チャートとして表現します。 

-   横軸は**ページロード**イベントの終了時間です。

-   縦軸は**ページロード**イベントの経過時間です。

-   <span class='lowfg'>空色</span> → <span class='normal'>青色</span> → <span class='highfg'>藍色</span> 順に正常<b>ページロード</b>イベントを表現します。

-   <span class='logwr'>黄色</span> → <span class='slow'>オレンジ色</span> → <span class='vslow'>赤色</span> 順にエラー発生<b>ページロード</b>イベントを表現します。 エラー<b>ページ読み込み</b>イベントは、ページの読み込み中にブラウザーエラーが発生した場合です。

### ![](/img/number-02.png)ページ一覧{#page-list}

**_ページ読み込みヒートマップ_**ウィジェットでドラッグした領域の**ページ読み込み**イベントが**_ページ一覧_**ウィジェットのテーブルに表示されます。 一覧で各項目の一番左の![詳細を見るアイコン](/img/ico-detail-view.svg)を選択すると**_ページ読み込み詳細_**ウィンドウが表示されます。

**_ページ一覧_** ウィジェットのテーブルには、次の基本情報が含まれます。

-   **_開始時間_**：**ページロード** イベントの開始時間

-   **_経過時間_**：**ページロード** イベント完了までの所要時間

-   **_ページURL_**：エンドユーザーがアクセスしたページのURLパス

-   **_ブラウザー_**：エンドユーザーが使用したブラウザーの種類

-   **_デバイス_**：エンドユーザーが使用した機器の種類

-   **_オペレーティングシステム_**:エンドユーザーが使用した機器のオペレーティングシステムの種類

-   **_クライアントIP_**:エンドユーザーが使用した機器のIPアドレス

テーブルに他の情報カラムを追加したり、追加したカラムを非表示にすることもできます。 **_ページ一覧_**ウィジェットの右上の![カラムアイコン](/img/ico-column.svg)ボタンを選択してください。

:::note

ページ一覧のうち、ブラウザエラーデータを含むページロードイベントは赤色で表示されます。

![WEBブラウサエラーデータ](/img/browser-pageload-error-list.png)

:::

### ![](/img/number-03.png)ページ一覧のフィルタリング{#filtering-page-list}

ページ一覧を対象に検索ワードを入力し、一致する対象をフィルタリングできます。 照会基準は次のとおりです。 希望する項目を選択し、検索ワードを入力してエンターキーを入力するか![検索アイコン](/img/ico-search.svg)ボタンを選択してください。

-   **_ページURL_**

-   **_WEBブラウサ_**

-   **_デバイス_**

-   **_OS_**

-   **_クライアントIP_**

### ![](/img/number-04.png)ページグループ一覧{#page-group-list}

![ページグループの一覧](/img/browser-analysis-pageload-hitmap-pagegroup.png)

**_ページ読み込みヒートマップ_**ウィジェットでドラッグした領域の**ページ読み込み**イベントを**[ページ·グループ](before-starting#page-group)**別に確認できます。 一覧から希望する項目を選択すると、選択した**ページグループ**に該当する**ページロード**イベントのみ**_ページ一覧_**に表示されます。

## ページ読み込みイベント詳細情報{#pageload-details}

**_ページ一覧_**で各項目の一番左の![詳細を見るアイコン](/img/ico-detail-view.svg)を選択すると、**_ページ読み込み詳細_**ウィンドウが表示されます。 **ページ読み込み**の性能を分析するために様々な指標を提供し、性能に影響を与える要素を把握することができます。

![ページロード詳細](/img/rum-pageload-event-02.png)

### ![](/img/number-01.png)URLリスト{#url-list}

**_ページ一覧_**の項目をURL基準で確認できます。 **_URL一覧_**のソート基準は、経過時間です。 URL項目を選択すると**ページロード**イベントの詳細情報を![指示線 2](/img/number-02.png)右側の画面で照会することができます。

-   各項目の青色の棒グラフは、**ページロード**の経過時間です。
-   各項目の棒グラフの右側の時間は、**ページロード**の開始時間です。

### ![](/img/number-02.png)概要（ページロードイベント要約）{#pageload-events-summary}

**_概要_**タブでは、**ページ読み込み**のプロセス中、区間別にかかった時間をグラフチャートとして提供します。 これにより、**ページ読み込み**プロセス中、遅い区間を把握することができます。

-   **_ページ情報_**：エンドユーザーの接続環境、アクセスページ情報、**_ユーザーID_**、**_セッションID_** 情報を提供します。

-   **_ページロードタイミング情報_**：**ページロード**時の区間別所要時間を確認することができます。

    -   **_redirect_**：ブラウザーがリダイレクトにかかる時間

    -   **_cache_**：ブラウザーがキャッシュされたリソースを検索するのにかかる時間

    -   **_dns_**：ブラウザーがドメインを照会するのにかかる時間

    -   **_wait_**：ブラウザーが **ページロード**に必要なデータへのレスポンスを待つのにかかる時間

    -   **_download_**：ブラウザーがサーバーからリソースをダウンロードするのにかかる時間

    -   **_render_**：サーバーからダウンロードしたリソースを画面にレンダリングし **ページロード** イベントを完了するのにかかる時間

### リソース一覧{#resource-list}

ブラウザーがサーバーからダウンロードするリソースは、**ページ読み込み**の性能に大きな影響を与える可能性があります。 **_リソース一覧_**では読み込みの速度が遅いか、ファイルサイズが大きいリソースを迅速に把握することができます。

![リソース一覧](/img/rum-pageload-event-03.png)

各リソースの開始時間に基づいてタイムラインチャートを提供し、ロード速度が遅いリソースを把握できます。 各リソースのファイルサイズを参照し、パフォーマンスに影響を与える部分を改善することができます。 各リソースをクリックすると**_リソース詳細_**ウィンドウを通じて詳細時間情報を確認できます。

![リソース詳細](/img/rum-pageload-event-04.png)

### ブラウザーエラーを確認する

ブラウザエラーデータを含むページロードイベントの場合、赤色で表示されます。 **_リソース一覧_**でエラーが発生したステップを確認することができます。

![リソースリスト-エラーデータ](/img/browser-pageload-details-resource-error.png)

1.  **_リソース一覧_**でエラーが発生したステップを選択すると、**_エラースタック_**ウィンドウが表示されます。

    ![エラースタック](/img/browser-pageload-details-resource-error-stack.png)

2.  **_Upload_**ボタンを選択してください。

3.  該当コードスタックに対するソースマップファイルを選択してください。

難読化されたスタック情報とJavaScriptのソースマップ情報を比較して、実際にエラーが発生したコードを確認できます。
