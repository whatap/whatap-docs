WhaTap**Network Management System**は、Standalone方式で単独仮想マシン(VM)にインストールされます。 これにより、ネットワーク管理システムが独立して動作し、ネットワークのモニタリング及び管理に必要なすべての機能を安定的に提供します。

## システム要求仕様

次は、WhaTap**Network Management System**の安定した性能を保証するために推奨される最小仕様です。 

| 項目         | 仕様                                   |
| ---------- | ------------------------------------ |
| **CPU**    | 4 Core 以上 (推奨)                       |
| **Memory** | 最小8GB以上                              |
| **Disk**   | 最小30GB以上                             |
| **OS**     | Rocky Linux 9.4, Ubuntu Server 22.04 |

## 収集範囲

WhaTap**Network Management System**は、次のようなネットワークデータ収集方式に対応します。

- **SNMP Polling** 

  ネットワーク機器の状態と性能データを定期的に要請して収集します。 `SNMP v1`及び`v2c`プロトコルに対応します。 

- **SNMP Trap** 

  ネットワーク機器から非同期で送信されるイベント及び通知情報を収集します。

- **Syslog** 

  ネットワーク機器から生成されたログデータをリアルタイムで収集して分析します。

:::note 

**ICMPプロトコル**によるネットワーク接続状態や遅延時間(Pingレスポンスタイム)などの収集は今後開発予定です。

:::
