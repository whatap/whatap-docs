<Cmdname sid="side_deviceInventory" className="uitext" /> はネットワーク上の機器登録と監視設定を管理します。機器はマネージャが提供する Auto Discovery 機能で自動検出するか、必要に応じて手動で追加できます。登録された各機器は SNMP ベースの監視と Health Check を設定して、リアルタイム状態を確認できます。これによりネットワーク管理システムで監視対象デバイスを一元管理できます。

## 基本画面案内

<Cmdname sid="side_deviceInventory" className="uitext" /> では登録済みネットワーク機器の一覧をテーブル形式で確認できます。

上部メニューでは機器登録と管理に関連する主要機能を提供します。

* **マネージャ選択**: 接続された NMS Manager インスタンスを選択  
* ![追加アイコン](/img/ico-add.svg) **デバイス追加**: 新しいネットワーク機器を手動で追加  
* **Auto Discovery**: NMS Manager を通じてネットワーク機器を自動検出して追加  

| デバイス一覧項目 | 説明 |
| :-------------: | ------------------------------ |
| **Alias** | 機器エイリアス |
| **Host** | IP アドレス |
| **Hostname** | 機器のホスト名 |
| **Description** | 機器の説明 (モデル名、用途など) |
| **Manufacturer** | 製造元 (Cisco Systems, America など) |
| **Location** | 設置場所 (階数、ラック位置など) |
| **Contact** | 担当者情報 |
| **Host Type** | 機器タイプ (Switch, Router, Server など) |
| **Role** | 機器の役割 |
| **Serial No.** | シリアル番号 |
| **Firmware** | ファームウェアバージョン |
| **SNMP** | SNMP 有効状態 (Enabled/Disabled) |

### ![追加アイコン](/img/ico-add.svg) デバイス追加

新しいネットワーク機器を手動で登録できます。

1. <Cmdname sid="side_deviceInventory" className="uitext" /> 画面で ![追加アイコン](/img/ico-add.svg) <Cmdname sid="BTN09017" className="uitext" /> ボタンをクリック。  
1. <Cmdname sid="BTN09017" className="uitext" /> の **Device Info** タブで情報を入力。  

    * **Host** (`必須`): 機器の IP アドレス  
    * **Host Name**: 機器のホスト名  
    * **Manufacturer**: 製造元情報  
    * **Contact**: 担当者情報  
    * **Role**: ネットワーク上の役割  
    * **Alias**: 管理用エイリアス  
    * **Description**: 機器詳細説明  
    * **Location**: 物理設置場所  
    * **Host Type**: 機器タイプ  
    * **Serial No.**: シリアル番号  

1. **追加** ボタンをクリック。  

### デバイス詳細管理

1. 登録済み機器の ![編集アイコン](/img/ic-edit.svg) をクリック。  
1. 詳細情報画面の **Device Info, SNMP Polling, Health Check** タブで情報を確認または修正し、**管理** ボタンをクリック。  

#### Device Info タブ

機器の基本情報を設定。

#### SNMP Polling タブ

SNMP プロトコルを使って機器の性能と状態を定期的に収集する設定を管理。修正後、**保存** ボタンで反映。

* **Port** (`必須`): SNMP ポート (デフォルト: 161)  
* **Community** (`必須`): SNMP v1/v2c コミュニティ文字列 (デフォルト: public)  
* **Version** (`必須`): SNMP バージョン (v1, v2c, v3)  
* **User Name**: SNMPv3 ユーザー名  
* **Security Level**: セキュリティレベル (noAuthNoPriv, authNoPriv, authPriv)  
* **Authentication Protocol**: 認証プロトコル  
    * **Authentication Key**: 認証キー  
* **Privacy Protocol**: 暗号化プロトコル  
    * **Privacy Key**: 暗号化キー  

##### OID 選択と管理

1. **Edit OID** ボタンをクリックし、OID リストから収集する MIB 情報を選択。  

    | OID リスト項目 | 説明 |
    | :---: | --- |
    | **OID** | SNMP OID 値 |
    | **MIB Information** | MIB 名称と情報 |
    | **Manufacturer** | 製造元 |
    | **Category** | 指標カテゴリ |
    | **Description** | 詳細説明 |

1. **OK** ボタンをクリック。  

##### 収集間隔設定

* **Custom Polling Interval**: データ収集間隔を設定 (秒単位、デフォルト: 60)  
* **Activate**: SNMP Polling の有効/無効  

#### Health Check タブ

ネットワーク機器の可用性をリアルタイムで監視。

| Health Check 項目 | 説明 |
| :--: | --- |
| **Activate** | ヘルスチェック有効状態 |
| **Target Host** | 監視対象 IP アドレスまたは FQDN |
| **Protocol** | ヘルスチェックプロトコル (ICMP など) |
| **Memo** | メモや説明 |
| **Actions** | 監視対象の編集 (![編集アイコン](/img/ico-edit.svg))、削除 (![削除アイコン](/img/ico-trash-red.svg)) |

##### ヘルスチェック対象の追加

1. **Health Check** タブで ![追加アイコン](/img/ic-s-plus.svg) **追加** ボタンをクリック。  
1. **ヘルスチェック対象追加** ポップアップで情報を入力。  

    * **Target Host**: IP アドレスまたは FQDN を入力  
    * **Protocol**: ICMP を選択 (ドロップダウン)  
    * **Memo**: 任意のメモ入力  
    * **Activate**: 即時有効化の有無を選択  

1. **OK** ボタンをクリックして保存。
