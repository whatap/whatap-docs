**デバイスダッシュボード**は、モニタリング中のネットワークデバイスの状態及び性能情報をリアルタイムで確認できるダッシュボードです。 **デバイスダッシュボード**を通じて、デバイスのインターフェース状態、トラフィック現況、Syslogイベント及びSNMP Trapメッセージなど、様々な運用データを視覚的にモニタリングできます。

## 基本画面ガイド

<ImgLang img="nms-device-dashboard.png" desc="デバイスダッシュボード" />

:::note 

**デバイスダッシュボード**は、リアルタイムモニタリング専用画面で、過去の特定時点を選択して照会する機能を提供しません。

:::

### <Cmdname sid="TTL09011" className="uitext" />

デバイスインターフェースの全体個数及び使用現況を直観的に確認でき、ネットワークトラフィックを処理する活性化されたインターフェース個数を素早く把握できます。 

- 登録されているすべてのデバイスの使用中のインターフェイス数(**Connected**) / 全体インターフェース数(**Total Interfaces**)を比率で表示します。

- 照会を希望するデバイスを選択すると、当該デバイスのインターフェース詳細情報が確認できる**インターフェース状態**メニューに移動します。

- 照会を希望するデバイスの上にマウスオーバーすると、そのデバイスの主要な状態情報をポップオーバー形式で簡単に確認できます。 

### <Cmdname sid="TTL09007" className="uitext" /> / <Cmdname sid="TTL09008" className="uitext" />

ネットワークトラフィックとパケット処理量(PPS, Packets Per Second)に基づいて、ネットワーク負荷の高いインターフェイスを確認できます。 

- ここ24時間で最大ThroughputまたはPackets Per Second (PPS)値を記録したインターフェイスとその時間をデバイスごとに表示します。

- トラフィックの高いインターフェイスは、最大値に基づいて降順でソートして表示します。

### <Cmdname sid="TTL08967" className="uitext" />

デバイスで本日発生したSyslogイベントを分析し、深刻度によって分類して円形チャート形式で確認できます。 ログの発生頻度と重大度に応じて、ネットワーク障害の原因をすばやく特定できます。

### <Cmdname sid="TTL08976" className="uitext" />

SNMP Trapメッセージの発生頻度に基づいて、ネットワークイベントが集中しているデバイスを特定できます。 ネットワークオペレータは、イベントの発生回数が急増したデバイスを集中的にチェックすることで、問題を早期に解決できます。

- 最近24時間SNMP Trapメッセージが最も多く発生したデバイスを降順にソートしてTop 10を表示します。

- SNMP Trapメッセージの発生頻度を使用して、ネットワークイベントと障害発生パターンを分析できます。

### <Cmdname sid="realtime_log_event" className="uitext" />

デバイスとネットワーク インフラストラクチャで発生するイベントログをリアルタイムでモニタリングできます。

- プロジェクト内で発生するログをライブ テールのようにリアルタイムで表示します。

- ログをリアルタイムで分析することで、システム障害や異常の兆候をすばやく検出できます。
