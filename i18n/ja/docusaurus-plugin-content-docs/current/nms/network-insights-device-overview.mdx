<Cmdname sid="device_overview" className="uitext" /> は [共通オプション](#common-options) と [ウィジェット](#widget) で構成されたダッシュボードで、プロジェクト内のネットワークデバイスの状態や性能指標を一目で確認できます。管理者はインターフェース状態、トラフィック処理状況、ICMP HealthCheck、Syslog、Trap イベント、リアルタイムログなどの主要運用データをウィジェット形式で確認できます。

## 共通オプション {#common-options}

Overview 画面全体に共通適用される機能で、データ参照範囲、マネージャ選択、ウィジェット配置保存などを設定できます。

* **プリセット選択**: 保存されたウィジェット配置を呼び出し、同じレイアウトで使用

* **プリセット保存**: 現在の画面構成を保存し、後で再利用可能

* **ウィジェット管理**: ウィジェットの追加、削除、配置変更、サイズ調整が可能

    * **ウィジェット削除**: 削除するウィジェットを右クリックし、表示される `削除` ボタンで削除

    * **ウィジェット追加**: ウィジェットのない空白部分を右クリックし、追加したいウィジェットを選択

## ウィジェット {#widget}

### デバイス状態

全インターフェース数と有効化数を確認できます。デバイスにマウスオーバーすると状態情報が表示され、クリックするとインターフェース詳細画面に移動します。

* 例: `8 / 22` → 全 22 個中 8 個が有効

:::note
インターフェース情報は SNMP の ifTable(1.3.6.1.2.1.2.2) と ifXTable(1.3.6.1.2.1.31.1.1) データを収集する必要があります。機器によっては未対応または差異があります。
:::

### ICMP HealthCheck

ICMP HealthCheck ウィジェットはネットワークジッタと最大往復時間 (Max RTT) をグラフで表示します。状態が異常な場合、対象 IP アドレスを <span class='vslow'>赤色</span>で表示します。モニタリング結果は直近 10 分単位で確認できます。

1. **デバイスインベントリ** に移動し、設定するデバイスの ![編集アイコン](/img/ic-edit.svg) アイコンをクリックします。  

1. デバイス設定画面の **Health Check** タブで設定できます。

### 本日の Syslog

本日発生した Syslog イベントを重大度別に円グラフで表示し、Emergency から Debug までのレベル別に分類します。ログの発生頻度と重大度分布を直感的に確認できます。

### Trap 発生量 TOP 10

直近 24 時間で Trap メッセージが最も多く発生したデバイスを表示し、イベント発生頻度順に降順で並び替えます。これにより、イベントが集中している機器をすばやく把握できます。

### 最大 Throughput インターフェース

直近 24 時間で最大 Throughput を記録したインターフェースを表示し、トラフィックが最も高い順に降順で表示します。

* 例: `41.09 Mbps (FastEthernet8, 2025-08-27 16:50:36)`

### 最大 PPS インターフェース

直近 24 時間で最大 PPS を記録したインターフェースを表示します。

* 例: `5.56K PPS (FastEthernet8, 2025-08-27 16:50:36)`

### リアルタイムログイベント

プロジェクト内で発生するログをリアルタイムテーブルで確認でき、時間、ホスト、ログ内容が表示されます。新しいログは Live tail 形式ですぐに反映されます。
