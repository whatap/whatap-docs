ホーム画面 > プロジェクト選択 > ***管理*** > ***メンテナンス計画***

<ImgLang img='maintenance.png' desc='メンテナンス計画' />

メンテナンス計画を登録して、登録したメンテナンス時間に受信設定した通知を一時停止できます。 定期的な設定で定期的なメンテナンス計画を設定します。

- <Cmdname sid="BTN07004" className="uitext" />ボタンを選択すると、新規メンテナンス計画画面に移動します。

- 既存のメンテナンス計画と新規メンテナンス計画は互換性がありません。 

  - 既存のメンテナンス計画メニューで登録したメンテナンス計画は、既存のメンテナンス計画メニューに表示されます。

  - 新規メンテナンス計画メニューで登録したメンテナンス計画は、新規メンテナンス計画メニューで表示されます。

- 新規メンテナンス計画の説明は、[次の文書](#maintenance-v2)を参照してください。

:::note

**権限**

メンテナンス計画を追加して修正・削除するには、プロジェクトに対する**修正**権限または**通知設定**権限が必要です。 メンバー権限の詳細については、[次の文書](../project/project-structure#project-auth)を参照してください

:::

## 基本メンテナンス計画{#maintenance-v1}

### メンテナン計画を登録する

1. 画面の右上にある<Cmdname sid="add_plan" className="uitext" />ボタンを選択します。

2. ***メンテナンス計画の追加***ウィンドウが表示されたら***作成者***、***メンテナンス履歴***の順に入力します。

   <ImgLang img='maintenance-add.png' desc='メインテナンス計画追加' />

3. <Cmdname sid="repeat" className="uitext" />項目で定期的な繰り返し間隔を選択します。

   - ***1回限り***：設定した時間内に1回のみメンテナンス計画の通知がアクティブになります。 繰り返し設定が必要ない場合は選択します。
   - **毎日**：毎日繰り返すメンテナンス計画です。
   - **毎週**：毎週選択した曜日ごとに繰り返すメンテナンス計画です。
   - ***毎月***：毎月選択した日ごとに繰り返すメンテナンス計画です。
   - **毎年**：毎年選択した月、日ごとに繰り返すメンテナンス計画です。

4. 開始時間と終了時間を設定してください。

5. 設定をすべて完了した後、***確認***ボタンを選択します。

メンテナンス計画一覧で追加した項目を確認できます。 

メンテナンス計画を登録した後、プロジェクト内のメニューから1時間以内に完了する予定または進行中のメンテナンス計画がある場合、画面上にバナー形式で表示します。

<ImgLang img='maintenance-header.png' desc='メンテナンス計画ヘッダーバナー' />

- 進行中のメンテナンス計画はオレンジ色で表示され、1時間以内に実行される予定のメンテナンス計画は灰色の背景で表示されます。

  <ImgLang img='progress-banner.png' desc='進行中' />

  <ImgLang img='before-banner.png' desc='進行予定' />

- 上部にあるメンテナンス計画バナーを選択すると、右側に***メンテナンス計画***一覧が表示されます。 1時間以内に進行予定または進行中のメンテナン計画の一覧を確認することができます。

- メンテナンス計画の一覧で１つを選択すると、***メンテナンス計画の確認***ウィンドウが表示され、詳細情報が確認できます。 そのウィンドウからメンテナンス計画を修正できません。

:::note

サーバープロジェクトのメンテナンス計画<Cmdname sid="repeat" className="uitext" />機能は、既存のメンテナンス計画メニューでは使用できません。 

:::

:::note 

**メンテナンス計画を活用した警告通知の非活性化**<ScrollToLink anchor='scrollTo-v1-event-temp-pause' />

メンテナンス計画の設定時間の間に**通知**を一時的に**非活性化**できます。 定期メンテナンスまたは予定された作業中に管理者が作業に集中できるよう不要な通知を防止するための機能です。 メンテナンス計画の設定時間の間に警告通知が一時的に中断され、管理者が作業を終えた後、メンテナンス計画の終了時点で通知が再び活性化されます。

特定の時間帯に警告通知を無効にしたい場合、***メンテナンス計画***メニューから繰り返しおよび時間を設定できます。 例えば、毎週水曜日の午前1時から午前4時までに通知を受けたくない場合は、次の例の画像を参照してください。 ***繰り返し***、***曜日***、***時間***設定を通じてその時間帯に反映されるように、***メンテナンス計画***に追加すると警告通知が非活性化されます。

<ImgLang img='maintenance-v1-pause-alert.png' desc='メンテナンスプランによる通知の無効化 v1' />

:::

### メンテナンス計画を修正する

1. ***メンテナンス計画***一覧で修正する項目の右側にある![編集アイコン](/img/ico-edit.svg)ボタンを選択します。 

2. ***修正***ウィンドウが表示されたら、必要な項目を修正してください。 

3. 修正を完了したら***確認***ボタンを選択します。

### メンテナンス計画を無効化する

***メンテナンス計画***一覧で、リストの右端にあるアクティブなトグル ボタンを選択して無効にします。 そのメンテナンス計画の項目は機能しません。

### メンテナンス計画を削除する

1. ***メンテナンス計画***一覧から削除する項目の左端のチェックボックスを選択します。 重複することを選択できます。

2. 画面右上にある***削除***ボタンが無効化されます。 

3. ***削除***ボタンを選択すると、チェックしたメンテナンス計画を削除できます。

:::note

- すべてのメンテナンス計画を選択するには、一覧の左上にあるチェックボックスを選択します。

- 個別の削除確認メッセージは表示されません。 削除する前に注意してください。

:::

### ダッシュボードにメンテナンス計画ウィジェットを追加する

***Flexボード***からウィジェットテンプレートとして***メンテナンス計画***ウィジェットを提供します。 

<ImgLang img='maintenance-widget.png' desc='メンテナンス計画ウィジェット' />

1. ホーム画面 > プロジェクト選択 > ***ダッシュボード*** > ***Flexボードメニュー***に移動して、***ダッシュボード一覧***からウィジェットを追加するダッシュボードを選択します。

2. ***修正モード***を選択します。

3. ***ウィジェットテンプレート***一覧から***メンテナンス計画***を選択します。

4. ***メンテナンス計画*** ウィジェットを必要な場所に配置します。

:::note

- ***メンテナンス計画***ウィジェットで一覧のいずれかを選択すると、詳細情報を確認できるウィンドウが表示されます。 そのウィンドウでは、メンテナンス計画を修正することはできません。

- ***メンテナンス計画***ウィジェットの右上にある![その他のアイコン](/img/ico-more.svg) > ![ショートカットアイコン](/img/ico-shortcut.svg)ボタンを選択すると、そのプロジェクトの***管理*** > ***メンテナンス計画*** メニューに移動します。

:::

### メンテナン計画を登録する

1. 画面の右上にある<Cmdname sid="add_plan" className="uitext" />ボタンを選択します。

2. ***メンテナンス計画の追加***ウィンドウが表示されたら***作成者***、***メンテナンス履歴***の順に入力します。

   <ImgLang img='maintenance-add-v2.png' desc='メインテナンス計画の追加' />

3. ***対象エージェント***を選択します。

   - ***全体エージェントの自動選択***活性化

     - メンテナンス計画の設定後に追加されたエージェントが自動的にメンテナンス計画に自動的に含まれます。

   - ***全体エージェント自動選択***無効化

     - プロジェクトにインストールされている活性化または非活性化状態のすべてのエージェントの一覧(エージェント名、IP)が表示されます。 

     - ***対象エージェント***一覧から選択したエージェントは、保守プランの対象となります。

     - エージェントは最大300個まで選択できます。

4. <Cmdname sid="repeat" className="uitext" />項目で定期的な繰り返し間隔を選択します。

   - ***1回限り***：設定した時間内に1回のみメンテナンス計画の通知がアクティブになります。 繰り返し設定が必要ない場合は選択します。

   - **毎週**：毎週選択した曜日ごとに繰り返すメンテナンス計画です。
   - ***毎月***：毎月選択した日ごとに繰り返すメンテナンス計画です。
   - **毎年**：毎年選択した月、日ごとに繰り返すメンテナンス計画です。

5. ***日付及び時間***項目で開始時間と終了時間を設定してください。

   - １回限りのメンテナンス計画を登録する場合、早い時間選択オプションで***30分***、***1時間***、***2時間***、***6時間***の中から目的の時間を選択して下さい。

6. 設定をすべて完了した後、***確認***ボタンを選択します。

メンテナンス計画一覧で追加した項目を確認できます。 

メンテナンス計画を登録した後、プロジェクト内のメニューから1時間以内に完了する予定または進行中のメンテナンス計画がある場合、画面上にバナー形式で表示します。

:::note

- クバネティスプロジェクト、URLプロジェクトは、***対象エージェント***選択を使用できません。 

- クバネティスプロジェクト、URLプロジェクトは、メンテナンス計画の設定後に追加でインストールされるエージェントが自動的にメンテナンス計画に含まれます。
