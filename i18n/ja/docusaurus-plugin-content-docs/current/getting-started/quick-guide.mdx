### ![](/img/number-04.png)エージェントインストール{#install}

<!-- 공통 안내로 적용되지 않는 경우가 많아서 상품군 링크 제공? -->

**プロジェクトアクセスキー**を発行された場合は、インストール案内セクションでWhaTapエージェントのインストールを進めることができます。 

**_管理_** > **_エージェントインストール_**メニューを通じてインストール案内セクションにアクセスすることもできます。

1.  対応環境を確認してください。 

2.  **_管理_** > **_エージェントインストール_**メニューの**_WhaTapエージェントダウンロード_**タブを選択してください。案内に従って直接エージェントをダウンロードするか、パッケージのインストールを進めてください。

    :::note 

    **エージェントダウンロード**

    -   環境により**_Download_**ボタンを選択するか、コマンドを使用してサーバーから直接ダウンロードします。
    -   環境に応じてリポジトリを登録し、パッケージをインストールします。

    :::

3.  **プロジェクトアクセスキー**と**WhaTapサーバーIP**を入力し、該当設定スクリプトを実行してください。  

4.  エージェントを再起動してください。 

これからWhaTapモニタリングサービスが始まります。 

ダウンロードおよび設定プロセスの詳細については、製品別のインストール案内文書を参考にしてください。左側から製品別メニューを選択するか、次のショートカットを確認してください。

### ![](/img/number-05.png)ダッシュボード照会{#dashboard}

エージェントインストール後、データがWhaTapサーバーに送信されると、**_ダッシュボード_**メニューでプロジェクトの全体現況をリアルタイムで照会することができます。WhaTapモニタリングサービスは、収集されたすべてのデータを統合ダッシュボードとして構成でき、ユーザーによって自由に編集できるパーソナライズダッシュボードを提供します。

-   アプリケーションは、**_アプリケーションダッシュボード_**と**_トランザクションマップ_**、**_アクティブトランザクション_**、**_アプリケーショントポロジー_**そして**_Flexボード_**の各ウィジェットで収集された性能指標を確認できます。
    {@video: https://whatap-landing-resource.s3.ap-northeast-2.amazonaws.com/app-activeTransection.mp4}
-   サーバは、**_リソースボード_**と**_コンパウンドアイ_**そして**_Flexボード_**の各ウィジェットで収集された性能指標を確認できます。
    {@video: https://whatap-landing-resource.s3.ap-northeast-2.amazonaws.com/server-compound.mp4}
-   データベースは、**_インスタンス一覧_**と**_インスタンスモニタリング_**、**_マルチインスタンスモニタリング_**、**_スロークエリ_**そして**_Flexボード_**の各ウィジェットで収集された性能指標を確認できます。
    {@video: https://whatap-landing-resource.s3.ap-northeast-2.amazonaws.com/db-realtimeMonitoring.mp4}
-   Kubernetesは、**_コンテナマップ_**の各ウィジェットを使用して、ノード、コンテナ、アプリケーション、ログ、イベント情報を動的に確認できます。
    {@video: https://whatap-landing-resource.s3.ap-northeast-2.amazonaws.com/k8s-targetgroup.mp4}

:::note
사용자 정의형 통합 대시보드인 Flex 보드 생성 및 설정에 대한 자세한 설명은 [다음 문서](../java/flex-board)를 참조하세요.

### ![](/img/number-06.png)外部データ収集{#collect}

<!-- 텔레그라프, 포커스 -->

TelegrafとFocusなどを活用してWhaTapサービス収集データを拡張できます。詳細については、次の文書[Telegraf](../telegraf/introduction.mdx)、[Focus](../focus/introduction.mdx)を参考にしてください。

-   WhaTapプラグインをインストールしてTelegrafで収集したデータをWhaTap収集サーバーに伝えることができます。
-   WhaTap Focusをインストールして、ユーザーが直接開発したチェックスクリプトまたはデータベースクエリを作動させ、収集された値などをユーザーのWhaTapプロジェクトにアップロードできます。

### ![](/img/ic-number-07.svg)通知設定{#notification}

WhaTapモニタリングサービスは、収集されるすべての指標に対してしきい値ベースの通知設定が可能です。**_警告通知_**メニューでイベントと受信設定及び過去の記録を確認することができます。個人別に通知受信の設定が可能なだけでなく、メール、SMS、メッセンジャー、App Pushなど多様な通知受信チャンネルを提供します。

### ![](/img/ic-number-08.svg)レポート照会{#report}

レポートは、サービス利用状況と障害の発生記録を報告する文書で、モニタリングデータ分析はサービスの改善方向を決める指標になります。WhaTapは、**_レポート_**メニューでレポートを作成し、業務自動化をサポートします。個々のプロジェクト単位のレポートとマルチプロジェクト単位の統合レポートを提供します。

{@video: https://whatap-landing-resource.s3.ap-northeast-2.amazonaws.com/prd/server/server_05_report.mp4}

:::note
レポートの詳細については、[次の文書](../java/report-intro)を参考にしてください。製品別レポートの案内については、製品別の技術文書を参考にしてください。
:::
