ANR 統計メニューは、Android アプリで発生した ANR (Application Not Responding) イベントを分析・モニタリングするためのダッシュボードです。  
ANR はアプリがユーザー入力に 5 秒以上応答しないときに発生する重大な性能問題です。  
上部エリアでは共通の時間選択とフィルタリング機能を提供します。

- **時間範囲**: 上部の時間選択で直近 5 分から最大 7 日まで表示可能  
- **フィルタ**: ANR 発生画面、デバイス、アプリバージョンなどでフィルタリング  
- **性能指標**: ANR 発生時の経過時間、頻度分析  

## 主な構成

### ANR 推移 (ラインチャート)

- **表示**: 時間帯別の ANR 発生件数推移  
- **Y軸**: ANR 発生件数  
- **X軸**: 時間  
- **インタラクション**  
  - マウスオーバー → 該当時点の件数表示  
  - ドラッグ → 特定区間の拡大表示  
  - クリック → 該当時点の ANR 詳細情報を確認  

### 画面別ANR経過時間トップ10 (棒グラフ)

- 平均応答遅延が長い上位 10 画面を表示  
- **表示**: 画面別平均 ANR 継続時間 (ms)  
- **活用**: 最も深刻な ANR が発生する画面を特定  
- **範囲**: デフォルト 5 秒以上、最大計測可能時間まで  

### 画面別ANR件数トップ10 (棒グラフ)

- ANR 発生頻度が高い上位 10 画面を表示  
- **表示**: 画面別総 ANR 発生回数  
- **活用**: 発生頻度の高い画面の優先度把握  

### 画面別ANRリスト

| カラム | 説明 |
| --- | --- |
| 詳細 | ANR 詳細分析ページへ移動 |
| 画面名 | ANR が発生した Activity 名 |
| ANR タイプ | Input dispatching timeout、Service timeout などの種類 |
| 発生時間 | ANR が発生した時刻 |
| 経過時間 | ANR が継続した時間 (ms) |
| アプリバージョン | 発生したアプリバージョン |
| デバイス | 発生したデバイスモデル |
| OS バージョン | Android OS バージョン |
| メインスレッド状態 | ANR 発生時のメインスレッド状態 |

- **機能**  
  - カラム別ソート (発生時間、経過時間など)  
  - テキスト検索 (画面名、デバイスなど)  
  - カラム設定で表示項目を選択  
  - CSV ダウンロード対応  

## 利用ガイド

- **時間範囲設定**: 上部時間選択で期間を設定後、更新ボタンで最新データを取得  
- **ANR フィルタリング**: 画面名検索、またはアプリバージョン、デバイス、OS、ANR タイプで条件指定  
- **性能指標分析**: 平均・最大・最小経過時間の中から分析基準を選択  
- **チャートインタラクション**  
  - 推移チャートクリック → 該当時点の ANR 一覧表示  
  - Top 10 チャートクリック → 該当画面の ANR のみをフィルタリング  
  - ドラッグ → 区間選択して詳細分析  
- **詳細分析**: 一覧から詳細ボタンクリック → スタックトレース、メインスレッド状態、待機リソース分析  
- **データエクスポート**: 右上の CSV ダウンロードボタンで現在条件に合うデータを保存  

:::caution

- ANR は Android 専用の指標で、iOS では収集されません。  
- ANR 発生時、システムがアプリを自動終了する場合があります。  
- メインスレッドで長時間の処理を実行しないよう注意が必要です。  

:::
