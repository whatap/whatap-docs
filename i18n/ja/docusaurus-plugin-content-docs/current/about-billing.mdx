## 決済アカウント管理{#faq-billing-account}

> WhaTapサービスの決済アカウントの管理方法について知りたいです。

WhaTapモニタリングでの決済情報登録、決済権限譲渡及びプロジェクト有料転換など、決済アカウントの管理方法については、次の概括案内を参考にしてください。 

:::note

トライアル利用期間中、およびトライアル利用期間が終了した時点で、サービス利用を続けたい場合は決済情報を登録し、該当プロジェクトを有料として転換します。 

:::

### ![1](/img/number-01.png) 決済情報の登録 {#regisgerpay}

決済情報を登録する前に、まず**決済**権限を確認してください。 基本的に、プロジェクト作成アカウントに**決済**権限が付与されます。

1. [WhaTapモニタリング](https://service.whatap.io)にログインします。 

2. ホーム画面の右上のプロファイルアイコンを選択してください。

3. ドロップダウンメニューが表示されたら<Cmdname sid="my_usage" className="uitext" />を選択します。

   <ImgLang img='billing-select-my-usage.png' desc='利用状況' />

4. <Cmdname sid="payment_information" className="uitext" />メニューから<Cmdname sid="register_billing_info" className="uitext" />ボタンをクリックします。

5. 詳細項目を順番に設定します。 <span class="vslow">\*</span>表示は必須項目です。

6. <Cmdname sid="payment_method" className="uitext" />項目でご希望の決済手段を選択します。

7. <Cmdname sid="register_billing_info" className="uitext" />画面が表示されたら、必須情報を入力して<Cmdname sid="apply" className="uitext" />ボタンを選択します。

8. 請求書を受信する担当者情報を順番に入力し<Cmdname sid="save" className="uitext" />ボタンをクリックします。

:::note 

- 決済情報登録の詳細については、[次の文書](/management/billing)を参照してください。

- 対応する決済通貨は<UI>KRW</UI>、<UI>JPY</UI>、<UI>USD</UI>です。 変更した通貨によって<Cmdname sid="payment_method" className="uitext" />が変更されます。

- 複数のメールで請求書を受信する場合は、<Cmdname sid="invoice_cc" className="uitext" />項目にコンマ(,)で区切って複数のメールを入力します。 例：a@whatap.io,b@whatap.io,c@whatap.io

  <ImgLang img='best-p/about-billing-member-recipient.png' desc='請求先設定' />

:::

### ![2](/img/number-02.png) 決済権限の変更 {#transferpayauth}

**決済**権限は組織、グループ、プロジェクト単位で付与でき**1人**だけ持つことができます。 例えば、AアカウントからBアカウントに決済権限を譲渡する場合、Aアカウントはこれ以上決済権限を保有しなくなります。 また、**決済**の権限を移譲するアカウントは、決済情報を登録した状態でEメール認証手続きを通じて承認を受ける方式で移譲できます。 

#### 決済権限を受け取るアカウントの決済情報登録

1. [WhaTapモニタリング](https://service.whatap.io)から**決済**権限を受け取るアカウントとしてログインします。

2. ![指示線1](/img/number-01.png)の過程を通じて、該当アカウントの決済情報を登録してください。

#### メンバー招待

1. [WhaTapモニタリング](https://service.whatap.io)で、**決済**権限と**修正**権限を持ったアカウントとしてログインしてください。

2. ホーム画面から、決済権限を譲渡する組織やグループ、またはプロジェクトを選択します。 

3. 選択した単位のメンバー管理メニューに移動します。 単位別メンバー管理メニューの移動の詳細については[次の文書](/about-billing#o-member-management)を参照してください。 

4. 画面右上の<Cmdname sid="invite_member" className="uitext" />ボタンを選択します。

5. メール入力欄に決済権限を受け取るアカウントのメールアドレスを入力し、<Cmdname sid="add" className="uitext" />ボタンをクリックしてください。

6. <Cmdname sid="member_invite_list" className="uitext" replace=" ({length})" />に該当アカウントが登録されたことを確認してから、<Cmdname sid="complete" className="uitext" />ボタンをクリックしてください。

:::note 

**組織メンバー管理メニューの移動** <ScrollToLink anchor='o-member-management' />

1. サイドメニュー上段の組織メニューの横の![右方向アイコン](/img/right-arrow.svg)ボタンを選択します。 

2. 組織リストパネル内の組織名の横の![設定アイコン](/img/ico-setting.svg) ボタンを選択します。 

3. <Cmdname sid="side_organizationManagement" className="uitext" />画面で<Cmdname sid="side_organizationMembers" className="uitext" />メニューに移動します。

**グループメンバー管理メニューの移動** 

1. サイドメニュー上段の<Cmdname sid="BTN07332" className="uitext" />メニューを選択します。 

2. グループ及びプロジェクトリストパネルにおいて、グループ名の横の![右方向アイコン](/img/right-arrow.svg)ボタンを選択します。 

3. <Cmdname sid="side_groupMember" className="uitext" />メニューに移動します。

**プロジェクトメンバー管理メニューの移動**

<Cmdname sid="management" className="uitext" />メニュー下位の<Cmdname sid="project_member" className="uitext" />メニューに移動します。

:::

#### 権限の修正

{/* 이미지 추가 필요 */}

<ImgLang img='best-p/about-billing-member-role.png' desc='権限修正sc' />

1. 選択した単位のメンバーリストから決済権限を移譲するアカウントの右側に<LinkImage img="char-a.svg" desc="지시선 a" /> <LinkImage img="ico-edit.svg" desc="편집 아이콘" />ボタンを選択します。

2. <Cmdname sid="edit_member_role" className="uitext" /> 画面が表示されたら権限のうち、<LinkImage img="char-b.svg" desc="指示線 b" /> <Cmdname sid="billing" className="uitext" />権限を許可してください。

3. <Cmdname sid="billing" className="uitext" />権限トグルを活性化した後<Cmdname sid="edit_member_role" className="uitext" />画面で<LinkImage img="char-c.svg" desc="지시선 c" /> <Cmdname sid="apply" className="uitext" />ボタンを選択します。

4. 決済確認画面が表示されたら<Cmdname sid="apply" className="uitext" />ボタンを選択します。

5. <Cmdname sid="TTL09415" className="uitext" />画面で下位グループまたは下位プロジェクトに同じ権限の付与有無を設定した後<Cmdname sid="apply" className="uitext" />ボタンを選択します。

   <ImgLang img='faq/about-billing-member-role-check.png' desc='결제 권한 변경' />

6. 当該アカウントに決済権限の変更を承認するためのEメール送信案内を確認した後<Cmdname sid="apply" className="uitext" />ボタンを選択します。

7. 決済権限変更対象アカウント**Eメール**通じて**承認**時、決済権限が最終的に変更されます。 

:::note

- 決済権限を持ったメンバーの場合、<Cmdname sid="project_member" className="uitext" />一覧で名前の右側に<code class='pay'>決済</code>タグが表示されます。 

- 他のメンバーに決済権限を付与する場合は、Eメール認証による承認手続きが必要です。 しかし、メンバー本人に直接決済権限を付与する場合、該当承認手続きが省略されます。

- <Cmdname sid="edit_member_role" className="uitext" />画面は、プロジェクトメンバー権限の修正か、または組織およびグループメンバー権限の修正かによって異なります。 メンバー権限の詳細については、<Link to="/project/project-structure#project-auth">次の文書</Link>を参照してください。

:::

### ![3](/img/number-03.png) プロジェクト有料転換 {#subscripeproject}

決済情報の登録を完了すると、<Cmdname sid="side_projectSubscription" className="uitext" />メニューに移動します。 該当アカウントが保有しているすべての<Cmdname sid="project_list" className="uitext" />を次のように確認できます。 

<ImgLang img='best-p/about-billing-subscription.png' desc='プロジェクト有料転換' />

- 有料状態ではないプロジェクトすべてを一括で有料転換する場合、右上の<Cmdname sid="BTN06013" className="uitext" />ボタンを選択します。

- 複数のプロジェクトを有料転換する場合は、<Cmdname sid="project_list" className="uitext" />の各チェックボックスを選択し、右上の<Cmdname sid="BTN06013" className="uitext" />ボタンを選択します。

- プロジェクト別の有料転換の場合、<Cmdname sid="project_list" className="uitext" />のステータス カラムから<Cmdname sid="subscribe" className="uitext" />ボタンを選択します。

- <Cmdname sid="project_list" className="uitext" />の各プロジェクト項目から<Cmdname sid="subscribe" className="uitext" />ボタンを選択すると個別に有料転換できます。

:::note

<Cmdname sid="side_projectSubscription" className="uitext" />メニューの詳細については、<Link to="management/billing#project-subscription">次の文書</Link>を参照してください。

:::
