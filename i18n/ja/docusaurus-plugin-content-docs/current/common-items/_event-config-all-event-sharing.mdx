## すべてのイベント設定を共有する {#share-event-setting}

基本イベントを含む他のすべてのイベント設定をJSON形式のファイルにダウンロードし、同じタイプの製品間で設定を共有できます。 これにより、繰り返しイベントの設定作業を簡素化できます。

### エクスポート {#export-event-setting}

画面の上の<Cmdname sid="BTN07762" className="uitext" />ボタンを選択します。 すべてのイベント設定を含むJSONファイルがダウンロードされます。 JSONファイル名は、_integrated-event-`YYYY`-`MM`-`DD`.json_の形式です。

JSONデータの構造は次のとおりです。

```json title="integrated-event"
{
  "productType": "{productType}",
  "platform": "{platform}",
  "events": {
    "HITMAP": {
      ...
    },
    "LOG_REALTIME": [
      {...}
    ],
    "METRICS": [
      {...}
    ],
    "COMPOSITE_LOG": [
      {...}
    ],
    "BASIC": [
      {...}
    ],
    "COMPOSITE_METRICS": [
      {...}
    ],
    "ANOMALY": [
      {...}
    ]
  }
}
```

`events`属性の下位属性は、次のイベント設定に該当します。

JSONデータの構造と`events`属性の下位属性は製品(`platform`)または製品タイプ(`productType`)によって異なる場合があります。

:::

:::caution

JSONデータの属性のうち`productType`または`platform`属性の値を任意に修正すると、他のプロジェクトでイベントが動作しない場合があります。

:::

### インポート {#import-event-setting}

1. 画面の上の<Cmdname sid="BTN07761" className="uitext" />ボタンを選択します。

2. JSON編集画面が開くと、画面の下の<Cmdname sid="load" className="uitext" />ボタンを選択します。

   <ImgLang img='event-config-v2-json-batch-edit.png' desc='JSON一括修正' />

3. <Cmdname sid="BTN07762" className="uitext" />機能でダウンロードしたJSONファイルを選択します。

4. <Cmdname sid="overwrite" className="uitext" />ボタンを選択します。

5. 確認メッセージが表示されたら<Cmdname sid="overwrite" className="uitext" />ボタンを選択します。

:::note

- 上書きすると、以前のイベント設定は回復できません。

- `productType`または`platform`属性の値を任意に修正すると、他のプロジェクトでイベントが動作しない場合があります。

- JSON編集画面には、JSON検証(Validation)機能が組み込まれています。 形式や値が正しくない場合は、編集画面の下にエラーメッセージが表示され、<Cmdname sid="overwrite" className="uitext" />ボタンは非活性化されます。

  <ImgLang img='event-config-v2-all-events-json-validate.png' />

:::
