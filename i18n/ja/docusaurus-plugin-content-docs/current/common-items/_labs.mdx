**サイトマップから実験室メニューにアクセスする**

サイドメニューに ![実験室アイコン](/img/ic-laboratory.svg) <Cmdname sid="side_lab" className="uitext" /> が表示されていない場合は、![サイトマップアイコン](/img/ic-sitemap.svg) <Cmdname sid="sitemap" className="uitext" /> からアクセスできます。

1. 画面左下の ![サイトマップアイコン](/img/ic-sitemap.svg) <Cmdname sid="sitemap" className="uitext" /> アイコンをクリックしてください。 

2. <Cmdname sid="sitemap" className="uitext" /> ウィンドウの右下にある <Cmdname sid="side_lab" className="uitext" /> セクションで、目的のメニューを選択してください。

:::

#### ビジネスダッシュボード

ビジネスダッシュボードは、アプリケーションのビジネス単位（App Context）ごとのパフォーマンスをモニタリングできるダッシュボードです。  
URLパターンやパスに基づいて定義された各ビジネス項目について、リアルタイムおよび過去のパフォーマンスデータを確認・分析できます。  
TPS、応答時間、エラー率などの主要なパフォーマンス指標をビジネス単位で確認することで、特定の業務やサービスのパフォーマンス状況を迅速に把握できます。  
詳しくは、[次のドキュメント](dashboard-business)をご参照ください。

:::note

**対応エージェントバージョン**

- **Java**：2.2.40以上  
- **PHP**：2.11.0以上  
- **Python**：1.8.5以上  
- **.NET**：2.4.3以上  

:::

</InDoc>

#### <Cmdname sid="side_collectionMap" className="uitext" />

  JVMに指定されたクラス（Class）が使用するCollection（例：`java.util.Map`）の情報を提供します。 この機能を使用するには、次のエージェントオプションを追加します。

```ini title="whatap.conf"
collection_nonstatic_holders=io.home.test.baseapp.app.post.web.PostResource
```

  :::note

  **対応エージェントバージョン**

  [Javaエージェント2.1.2バージョン](../release-notes/java/java-2_1_2#collectionmonitor) 以降から対応します。

  :::

</InDoc>

#### <Cmdname sid="side_remoteAccessStatus" className="uitext" />

  アプリケーションエージェントの設定により、各モニタリング対象ごとの外部(remote) 呼び出しに対する成功/失敗の有無を記録したログを視覚化して提供します。

  この機能を利用するには、以下のオプションをエージェント設定ファイル（*whatap.conf*）に追加するか、<Cmdname sid="side_management" className="uitext" /> &gt; <Cmdname sid="side_agentSetting" className="uitext" /> メニューで追加してください。

```ini title="whatap.conf"
logsink_rt_enabled=true
logsink_enabled=true
```

</InDoc>
