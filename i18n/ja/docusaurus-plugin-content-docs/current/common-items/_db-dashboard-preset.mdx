## プリセットを設定する{#preset}

ダッシュボードからユーザーが設定したウィジェットの設定とレイアウト状態を保存して読み込むことができます。 ウィジェットのサイズを調整し、希望する場所に配置して新しいプリセットを作成できます。

![Preset](/img/db-common-multidb-preset.png)

- ***Default***: ***RACモニタリング***画面の主要DB指標とXOS指標で構成されたプリセットです。 RACデータベースで実行中のインスタンスに基づいて指標を確認できます。

:::note

- プリセットに保存できるダッシュボードの設定は次のとおりです。

  - ウィジェットのサイズ、位置

  - ユーザー定義ウィジェットに定義された設定

  - <UI>Session Table</UI>ウィジェットに定義されたテーブルの幅とフィルター、選択したカラム値

- 基本プリセット(<Cmdname sid="kube_default" className="uitext" />)は変更できません。

- データベース環境によって、一部のプリセットはサポートされない場合があります。

- <Cmdname sid="edit_project" className="uitext" />権限のあるメンバーのみがプリセットを保存または変更できます。 メンバー権限の詳細については、[次の文書](../project/project-structure#member-auth)を参照してください。

:::

### 新しいプリセットを作成する{#newpreset}

1. ダッシュボードから、希望する形にウィジェットを配置してみてください。 サイズを変更して、頻繁に確認するウィジェットのみを配置することもできます。

2. 画面の右上に![保存アイコン](/img/ic-save.svg)ボタンを選択します。

3. 新しいプリセット名を入力します。

   <InDoc product="oracle,oracle-pro"><ImgLang img='db-dashboard-preset-save.png' desc='Preset' /></InDoc>

   エージェント選択履歴を一緒に保存するには、<Cmdname sid="ITM07345" className="uitext" />を選択します。

4. <Cmdname sid="save" className="uitext" />ボタンを選択します。 プリセット一覧から新しく保存したプリセットを確認できます。

:::note

- 新しく作成したプリセットに変更が加えられた場合は、再度プリセットを保存してください。 ![保存アイコン](/img/ic-save.svg)ボタンを選択し、同じ名前でプリセットを保存します。 既存のプリセットに変更を上書きします。

- ダッシュボードの変更を保存せずに別のメニューに移動すると、変更は保存されません。

- プリセットはプロジェクト単位で保存され、他のユーザーと共有できます。

:::

### プリセットを削除する{#removepreset}

使用していないプリセットは、プリセット一覧から削除できます。 プリセット一覧から削除したい項目の右側に![削除アイコン](/img/ico-remove.svg)ボタンをクリックします。

:::note

<UI>Default</UI>プリセットは削除できません。

:::
