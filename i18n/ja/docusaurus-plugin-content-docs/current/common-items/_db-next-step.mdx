## 次の段階 {#nextstep}

<div className="nextstep">

- **インストールの点検**

  プロジェクトの作成やエージェントのインストール、基本エージェントのオプションまですべて適用した場合は、[次の文書](after-install-agent)で点検事項を確認します。

- **インストール問題の解決**

  エージェントをインストールする際に発生する可能性がある問題と、これを解決するための具体的なガイドを提供します。 詳細については、[次の文書](troubleshooting)を参照してください。

- **エージェント設定**

  <InDoc product="oracle-pro,mssql">

  エージェント設定(*whatap.conf*)ファイルにオプションを適用して、モニタリングのための様々な機能を提供します。 詳細については、[次の文書](agent-settings)を参照してください。

  </InDoc>

  <Xclude product="oracle-pro,mssql">

  エージェント設定(*whatap.conf*)ファイルにオプションを適用して、モニタリングのための様々な機能を提供します。 詳細については、[次の文書](agent-dbx-settings)を参照してください。

  データベースのサーバーリソースを追加で監視するには、追加のエージェント(XOS)に追加オプションを設定します。 詳細については、[次の文書](agent-xos-settings)を参照してください。

  </Xclude>

  <InDoc product="cubrid">

  XCUBエージェント設定の詳細については[次の文書](agent-xcub-settings)を参照してください。

  </InDoc>

<InDoc product="postgresql,mysql,oracle,mssql">

- **クラウド設定**

  クラウドサービスが提供するモニタリングメトリクスを、データベースプロジェクトのダッシュボードで追加で収集してモニタリングするには、[次の文書](cloudsettings)を参照してください。

</InDoc>

- **モニタリングの開始**

  すべての設定を完了したら、エージェントがデータベースサーバーのメトリクスに対する情報収集を開始します。 まず、<Cmdname sid="side_instanceList" className="uitext" />メニューでモニタリングデータが収集されていることを確認します。 <Cmdname sid="side_instanceList" className="uitext" />の詳細については、[次の文書](instance-list)を参照してください。

</div>
