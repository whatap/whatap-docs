## 項目別イベント設定

次のイベント項目を設定できます。

-   **_サーバーCPU_**、**_サーバーメモリ_**、**_サーバーディスク_**<br/>
    各項目の使用量が設定した**_条件（%）_**に設定した基準値に達すると通知を送信します。

-   **_アクティブトランザクション_**<br/>
    アクティブトランザクションの発生件数が**_条件（件数）_**に設定した基準値を超えた状態を継続すると通知を送信します。発生件数のデフォルト値は100個です。

-   **_トランザクションエラー_**<br/>
    正常に実行されていないトランザクションを意味します。エラー件数が**_条件（件数）_**に設定した基準値を超えた状態を継続すると通知を送信します。収集周期は5秒です。

-   **_トランザクションレスポンス時間_**<br/>
    トランザクションが終了した時間で確認します。例えば、**10**秒の**_レスポンス時間_**基準を超えるトランザクションの**_件数_**が**10**個を超え、この状態が**5**秒間**_持続_**すると通知します。

-   **_ヒートマップ横線_**<br/>
    ヒートマップのレスポンス時間分布で、**_レスポンス時間_**が**5**秒以上のトランザクションが同一レスポンス時間の区間に連続<sup>\*</sup> **5** **_件数_**が発生すると通知します。

    :::note

    -   \* X軸（5秒）単位の分布が連続した線で示される場合（途中で切れがある場合はこの限りではありません）
    -   提供が中断された通知設定機能です。**_ヒートマップパターン_**通知の利用をお勧めします。
        :::

-   **_ヒートマップ縦線_**<br/>
    ヒートマップのレスポンス時間分布からX軸単一区間（5秒）に縦線<sup>\*</sup>が表示される現象が該当時間（1回）中5秒間**_持続_**すると通知します。

    :::note

    -   \* 縦線現象：**_最小時間（秒）_**2.5秒から80秒を表現する全体100区間のうち、3**30~50**個の区間がマーキングされた場合は<span class='slow'>警告通知</span>が発生し、同じルールで区間が![指示線2](/img/number-02.png)**50**個以上マーキングされた場合は<span class='vslow'>危険通知</span>が発生します。
    -   提供が中断された通知設定機能です。**_ヒートマップパターン_**通知の利用をお勧めします。
        :::
