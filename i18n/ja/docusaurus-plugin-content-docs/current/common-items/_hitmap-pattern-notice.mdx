アプリケーション実行の分布パターンをマシンラーニングを介して自動認識し、警告通知を送信します。複雑な条件設定なしでイシュー状況を迅速かつ正確に認知することができます。数多くのヒートマップパターンが学習されており、リアルタイムで収集されるデータから学習されたパターンと類似したパターンが発生すると警告通知を受けることができます。

## 基本オプション

-   **_検知対象外時間_**：設定した時間未満のパターンは無視します。

-   **_ミュート_**：過剰なアラート通知の発生を防ぐオプションです。最初のアラート通知から選択した時間にアラート通知は送信しません。また、**_イベント履歴_**メニューに記録されません。

-   **_保存_**：設定した内容を保存して反映します。

## ヒートマップパターン

次のヒートマップパターンが発生した場合は、警告通知を送信するように設定できます。各項目のトグルボタンを選択して、警告通知をオンまたはオフにすることができます。

-   **_ヒートマップの横ラインパターン_**：複数のトランザクションが一定時間内に終了するパターンです。リソースを獲得したり、外部HTTPS Callをしたりするときにタイムアウトまたは遅延が発生します。

    ![横ラインパターン](/img/blog_49_04.png)

-   **_ヒートマップ縦ラインパターン_**：呼び出し時点が異なりますが、同一時点でトランザクションが終了するパターンです。トランザクションが使用する共通のリソースに一時的なボトルネックが発生します。

    ![縦ラインパターン](/img/blog_49_05.png)

-   **_ヒートマップのフライングパターン_**：特定リソースやログのような共通リソース不足現象で、間隔をあけて波打つような現象のパターンが発生します。

    ![フライングパターン](/img/blog_49_06.png)

-   **_ヒートマップ過負荷パターン_**：全体または一部の応答に一時的な問題が発生すると、一時的にトランザクションが密集するパターンが発生します。

    ![過負荷パターン](/img/blog_49_07.png)

-   **_ヒートマップの暴走パターン_**：過度なトランザクションのリクエストや負荷が発生すると、応答時間が全体的に増加するパターンが発生します。

    ![暴走パターン](/img/blog_49_08.png)
