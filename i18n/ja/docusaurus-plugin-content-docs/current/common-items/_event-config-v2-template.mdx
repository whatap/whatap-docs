## 基本イベントテンプレート {#basic-template}

作成したプロジェクトの製品に応じて、基本イベントテンプレートを提供します。 イベント一覧の一番左に<Cmdname sid="TTL06099" className="uitext" />ボタンを選択して、目的のイベントを活性化することができます。

<InDoc product="java,php,python,nodejs,dotnet,golang">

<ImgLang img='event-config-v2-apm.png' desc='イベントリスト' />

提供するイベントテンプレートは次のとおりです。 <Cmdname sid="ITM07578" className="uitext" />項目に設定された既定値は、ユーザーが希望する値に変更できます。

- **Active Transaction**

  カテゴリ：[`app_counter`](metrics-app#app_counter)

  アクティブトランザクション発生件数(`active_tx_count`)が100個を超過した状態になると警告(<span class="slow">Warning</span>)レベルの通知を送信します。

- **Agent Data Timeout**

  カテゴリ: [`agent_status_summary`](metrics-app#agent_status_summary)

  エージェントから30秒間データを収集できない場合（now-`lastActTime` >= 30000）、警告レベル（<span class="slow">Warning</span>）のアラートを送信します。
  :::note

  このテンプレートイベントは、ネットワークが切断された場合でも発生する可能性があります。

  :::

- **Apdex 100 by kind**

  カテゴリ：[`app_counter`](metrics-app#app_counter)

  エージェント種類別アプリケーション性能メトリクスのパーセンテージ(`apdex100`)が95%未満だと警告(<span class="slow">Warning</span>)レベルの通知を、90%未満であれば危険(<span class="vslow">Critical</span>)レベルの通知を送信します。

- **Apdex 100 by project**

  カテゴリ：[`app_counter`](metrics-app#app_counter)

  プロジェクトの全体エージェントのアプリケーション性能マトリクスのパーセンテージ(`apdex100`)が95%未満だと警告(<span class="slow">Warning</span>)レベルの通知を、90%未満であれば危険(<span class="vslow">Critical</span>)レベルの通知を送信します。

- **Server CPU**

  カテゴリ：[`app_host_resource`](metrics-app#app_host_resource)

  ホストのCPU使用率(`cpu`)が70%を超えると警告(<span class="slow">Warning</span>)レベルの通知を、90%を超えると危険(<span class="vslow">Critical</span>)レベルの通知を送信します。

- **Server Memory**

  カテゴリ：[`app_host_resource`](metrics-app#app_host_resource)

  ホストのメモリ使用率(`mem`)が70%を超えると警告(<span class="slow">Warning</span>)レベルの通知を、90%を超えると危険(<span class="vslow">Critical</span>)レベルの通知を送信します。

- **Transaction Error Count**

  カテゴリ：[`app_counter`](metrics-app#app_counter)

  トランザクションエラー件数(`tx_error`)が10個を超えると警告(<span class="slow">Warning</span>)レベルの通知を送信します。

- **Transaction Response Time**

  カテゴリ：[`app_counter`](metrics-app#app_counter)

  トランザクションの平均リスポンスタイム(`resp_time`)が10回連続、2,000ms(2秒)を超えた状態で続く場合は警告(<span class="slow">Warning</span>)レベルの通知を送信します。

:::note

基本イベント テンプレートに適用されているフィールドの詳細については、[次の文書](metrics-app)を参考にしてください。

:::

</InDoc>

提供するイベントテンプレートは次のとおりです。 <Cmdname sid="ITM07578" className="uitext" />項目に設定された既定値は、ユーザーが希望する値に変更できます。

- **Agent Data Timeout**

  カテゴリ: [`agent_status_summary`](metrics-server#agent_status_summary)

  エージェントから30秒間データを収集できない場合（now-`lastActTime` >= 30000）、警告レベル（<span class="slow">Warning</span>）のアラートを送信します。
  
  :::note

  このテンプレートイベントは、ネットワークが切断された場合でも発生する可能性があります。

  :::

- **CPU**

  カテゴリ：[`server_base`](metrics-server#server_base)

  CPU使用率(`cpu`)が70%を超えると警告(<span class="slow">Warning</span>)レベルの通知を、90%を超えると危険(<span class="vslow">Critical</span>)レベルの通知を送信します。

- **CPU Steal**

  カテゴリ：[`server_base`](metrics-server#server_base)

  [CPU steal使用率](../glossary#cpusteal)(`cpu_steal`)が5を超えると警告(<span class="slow">Warning</span>)レベルの通知を、10を超えると危険(<span class="vslow">Critical</span>)レベルの通知を送信します。

- **Disk**

  カテゴリ：[`server_disk`](metrics-server#server_disk)

  ファイルシステム使用率(`usedPercent`)が90%を超えると警告(<span class="slow">Warning</span>)レベルの通知を、95%を超えると危険(<span class="vslow">Critical</span>)レベルの通知を送信します。

- **Disk I/O**

  カテゴリ：[`server_disk`](metrics-server#server_disk)

  [Disk I/O使用率](../glossary#diskio)(`ioPercent`)が90%を超えると警告(<span class="slow">Warning</span>)レベルの通知を、95%を超えると危険(<span class="vslow">Critical</span>)レベルの通知を送信します。

- **Disk Inode**

  カテゴリ：[`server_disk`](metrics-server#server_disk)

  Disk Inode使用率(`usedInodePercent`)が90%を超えると警告(<span class="slow">Warning</span>)レベルの通知を、95%を超えると危険(<span class="vslow">Critical</span>)レベルの通知を送信します。

- **Memory**

  カテゴリ：[`server_base`](metrics-server#server_base)

  メモリ使用率(`memory_pused`)が70%を超えると警告(<span class="slow">Warning</span>)レベルの通知を、90%を超えると危険(<span class="vslow">Critical</span>)レベルの通知を送信します。

- **Network BPS**

  カテゴリ：[`server_network`](metrics-server#server_network)

  ネットワークインバウンドトラフィック(`trafficIn`)とアウトバウンドトラフィック(`trafficOut`)の合計が150,000,000byte(150MB)を超えると警告(<span class="slow">Warning</span>)レベルの通知を、200,000,000byte(200MB)を超えると危険(<span class="vslow">Critical</span>)レベルの通知を送信します。

- **Network IOPS**

  カテゴリ：[`server_network`](metrics-server#server_network)

  ネットワークインバウンドパケット(`packetIn`)とアウトバウンドパケット(`packetOut`)の合計が45000byteを超えると警告(<span class="slow">Warning</span>)レベルの通知を、50000byteを超えると危険(<span class="vslow">Critical</span>)レベルの通知を送信します。

- **Port Alive Check**

  カテゴリ: `server_tcpport`

  モニタリング中のネットワークデーモンの受信ポートが無効になった場合、警告(<span class="slow">Warning</span>) レベルの通知を送信します。

  :::note

  このテンプレートを使用するには、エージェント設定でポートをモニタリングするためのオプションを追加する必要があります。 モニタリング対象サーバーで次のコマンドを実行してください。

  <Tabs groupId="operating-systems">
  <TabItem value="lx" label="Linux Shell">

  ```bash title="SH"
  export TARGET_IP=127.0.0.1 # Monitoring target IP address
  export TARGET_PORT=80 # Monitoring target port number
  echo "tcp.check.$TARGET_PORT=tcp://$TARGET_IP:$TARGET_PORT" |sudo tee -a /usr/whatap/infra/conf/whatap.conf
  ```

  </TabItem>
  <TabItem value="wd" label="Windows Powershell">

  ```bash title="SH"
  $TARGET_IP="127.0.0.1" # Monitoring target IP address
  $TARGET_PORT="80" # Monitoring target port number
  Add-Content -Path "C:\Program Files\WhatapInfra\whatap.conf" -Value "tcp.check.$TARGET_PORT=tcp://$TARGET_IP:$TARGET_PORT"
  ```

  </TabItem>
  </Tabs>

  :::

- **Process CPU**

  カテゴリ：[`server_process`](metrics-server#server_process)

  CPU全体使用率(`cpu`)が90%を超えると警告(<span class="slow">Warning</span>)レベルの通知を、95%を超えると危険(<span class="vslow">Critical</span>)レベルの通知を送信します。

- **Process Count**

  カテゴリ：[`server_process`](metrics-server#server_process)

  プロセス数(`count`)が2つ以上ある場合は警告(<span class="slow">Warning</span>)レベルの通知を、1つ未満の場合は危険(<span class="vslow">Critical</span>)レベルの通知を送信します。

- **Process Memory**

  カテゴリ：[`server_process`](metrics-server#server_process)

  プロセスメモリ使用率(`memory`)が90%を超えると警告(<span class="slow">Warning</span>)レベルの通知を、95%を超えると危険(<span class="vslow">Critical</span>)レベルの通知を送信します。

- **Server Reboot Check**

  カテゴリ: [`agent_status_summary`](metrics-server#agent_status_summary)

  サーバーが再起動(`isRebooted` \= `true`)になると、危険(<span class="vslow">Critical</span>)レベルの通知を送信します。

  :::note

  少なくとも10分以上実行されたサーバーに対してのみ、Rebootイベントが動作します。 (`uptime` \> 600)

  :::

- **Swap Memory**

  カテゴリ：[`server_base`](metrics-server#server_base)

  スワップメモリ使用率(`memory_swappused`)が70%を超えると警告(<span class="slow">Warning</span>)レベルの通知を、90%を超えると危険(<span class="vslow">Critical</span>)レベルの通知を送信します。

:::note

基本イベント テンプレートに適用されているフィールドの詳細については、[次の文書](metrics-server)を参考にしてください。

:::

- **Active connections**

  カテゴリ: [`agent_status_summary`](metrics-data-list#agent_status_summary)

  :::note

  このテンプレートイベントは、ネットワークが切断された場合でも発生する可能性があります。

  :::

- **CPU Utilization**

  カテゴリ：[`db_mongodb_counter`](metrics-data-list)

  CPU使用率(`cpu(xos)`)が80%を超えると警告(<span class="slow">Warning</span>)レベルの通知を、90%を超えると危険(<span class="vslow">Critical</span>)レベルの通知を送信します。

  カテゴリ：[`db_mongodb_counter`](metrics-data-list)

  データベース接続が15回以上失敗(`connect_error`)であれば、危険(<span class="vslow">Critical</span>)レベルの通知を送信します。

- **Global Lock Queue count**

- **Memory Utilization**

  カテゴリ：[`db_mongodb_counter`](metrics-data-list)

  メモリ使用量(`mem(xos)`)が80%を超えると警告(<span class="slow">Warning</span>)レベルの通知を、95%を超えると危険(<span class="vslow">Critical</span>)レベルの通知を送信します。

  :::note

  このテンプレートは、XOSエージェントを設定して実行した場合にのみ使用できます。 XOSエージェント設定の詳細については、[次の文書](agent-xos-settings)を参照してください。
