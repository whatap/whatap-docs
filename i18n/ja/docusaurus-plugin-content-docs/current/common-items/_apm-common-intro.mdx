WhaTapアプリケーションのモニタリングは、障害を再現することなく、リアルタイムでモニタリングできます。

### 主な機能

- **リアルタイムなトランザクションモニタリング**

  WhaTapは、現在進行中のトランザクションの実行状況をリアルタイムで表示します。すべてのアプリケーションの問題は、進行中のトランザクションの増加で現れます。増加した瞬間に実行中のSQL、外部呼び出し、Methodをすぐに確認できることで問題を早く認識して分析することができます。

- **トランザクション性能の詳細分析**

  WhaTapは、すべてのトランザクションの実行履歴を収集して分析できます。トランザクションの性能は、SQLやHTTPコールだけでなく、メソッドレベルまで追跡できます。 

  <InDoc product='java,python,dotnetㄴ'>

  特に、**アクティブスタック**機能により、煩わしいMethodトレース設定なしでMethodレベルの分析が可能です。**アクティブスタック**はWhaTapの特許技術であり、既存のサービスでは発見できなかった隠れた問題の原因を特定し、改善ポイントを提示します。

  :::note

  アクティブスタック機能は、Java、Python、.NET の環境でのみ対応しています。

  :::

  </InDoc>

- **MSA呼び出しパターン分析**

  MSA構造の複雑なアプリケーション間の呼び出し関係を、トランザクションURLを基準に統計的に可視化して提供します。時点ごとに変化する呼び出し関係や呼び出し比率の変化を統計的に把握できます。

</Xclude>

- **AIを活用した応答時間分布パターン分析**

  WhaTapは、特定のパターンをマシンラーニングで自動的に認識して通知アラートを生成し、システム運用者が気づいていない問題もお知らせします。（オンプレミス型は別途ご相談ください）<br/>
  出願番号：10-2020-0037381

<Xclude product='dotnet'>

- **マルチプロジェクトのトランザクション連携分析**

  MSA環境においてトランザクションの連携追跡機能を提供します。全体のトランザクションを追跡し、遅延が発生しているアプリケーションを特定できます。

</Xclude>

<InDoc product='java,python'>

- **スタック統計の分析**

  統計分析では、リアルタイムのモニタリングやトレース情報で見つけることが困難なチューニングポイントを見つけることができます。 マクロ的な問題の推移もグラフと表を通じて改善対象を選択したり、特定の時間帯の応答遅延現象を確認することができます。

</InDoc>

- **事後分析**(**Cube**)

  時間を軸にしてさまざまな指標を一度に確認できます。応答時間、エラー件数、スループットなどが高い時間帯を特定することで、同時間帯における問題要素を特定したり、サービスの状況を正確に把握することができます。

<InDoc product='java'>

- **サービス輻輳時の負荷制御**

  システムが処理できない過剰なリクエストが発生した場合、一部のリクエストを拒否したり、不正なサービスリクエストを遮断することで、安定したサービス運用を支援します。

- **非同期トランザクショントレース**

  WhaTap独自のBCI(Byte Code Instrumentation) 技術で、非同期トランザクションの性能を効果的に追跡します。WebFlux、Reactor、Hystrixなどを使用するアプリケーションのトランザクショントレース情報を収集します。

</InDoc>
