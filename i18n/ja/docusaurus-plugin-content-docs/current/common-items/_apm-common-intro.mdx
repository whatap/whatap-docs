WhaTapアプリケーションのモニタリングは、障害を再現することなく、リアルタイムでモニタリングできます。

{@video: https://www.whatap.io/video/main-product/monitoring_step_0.mp4}

### 主な機能

-   **リアルタイムなトランザクションモニタリング**

    WhaTapは、現在進行中のトランザクションの実行状況をリアルタイムで表示します。すべてのアプリケーションの問題は、進行中のトランザクションの増加で現れます。増加した瞬間に実行中のSQL、外部呼び出し、Methodをすぐに確認できることで問題を早く認識して分析することができます。

-   **トランザクション性能の詳細分析**

    WhaTapは、すべてのトランザクションの実行履歴を収集して分析できます。トランザクション性能は、SQL、Http Callだけでなく、Methodレベルまで追跡できます。 

    <InDoc product='java,python'>

    特に、**アクティブスタック**機能で煩わしいMethodトレース設定がなくても、Methodレベル分析が可能です。 **アクティブスタック**は、WhaTapの特許技術で、既存の他のサービスで発見できず隠された領域の問題原因を見つけだし、改善点を提供します。

    :::note

    アクティブスタック機能は、Java、Python環境のみ対応しています。

    :::

    </InDoc>

-   **MSA呼び出しパターン分析**

    MSA構造の複雑なアプリケーション間の呼び出し関係をトランザクションURL基準で呼び出し率を統計的に視覚化して提供します。視点ごとに変化する呼び出し関係や呼び出し比重の変化を統計的に眺めることができます。 視点ごとに変化する呼び出し関係や呼び出し率の変化を統計的に見ることができます。

</Xclude>

-   **AIを活用した応答時間分布パターン分析**

    WhaTapは、特定のパターンをマシンラーニングを通じて自動的に認識して通知アラートを生成し、システムオペレーターが認識していない問題をお知らせします。(インストール型はお問い合わせください)<br/>
    出願番号：10-2020-0037381

<Xclude product='dotnet'>

-   **マルチプロジェクトのトランザクション連携分析**

    MSA環境でトランザクション連携追跡機能を提供します。トランザクション全体を追跡して、どのアプリケーションで遅延が発生するかを把握できます。

</Xclude>

<InDoc product='java,python'>

-   **スタック統計の分析**

    統計分析では、リアルタイムのモニタリングやトレース情報で見つけることが困難なチューニングポイントを見つけることができます。 マクロ的な問題の推移もグラフと表を通じて改善対象を選択したり、特定の時間帯の応答遅延現象を確認することができます。

</InDoc>

-   **事後分析(Cube)**

    時間を軸として設定し、様々な指標を一度に確認することができます。応答時間、エラー件数およびスループットが高い時間帯のみを特定できるため、同時時間帯の問題要素を特定したり、サービス状況を正確に表示できます。

<InDoc product='java'>

-   **サービス輻輳時の負荷制御**

    システムが処理できない過度なリクエストが発生した場合は、一部のリクエストを拒否したり、誤ったリクエストをブロックしたりできます。サービスをより安定的に運用することができます。

-   **非同期トランザクショントレース**

    WhaTap独自のBCI(Byte Code Instrumentation) 技術で、非同期トランザクションの性能を効果的に追跡します。WebFlux、Reactor、Hystrixなどを使用するアプリケーションのトランザクショントレース情報を収集します。

</InDoc>
