エージェントが収集したアクティブセッション内のSQLを実行回数、実行時間、待機時間などを基準に統計を算出し、表示期間中のTop SQL一覧を提供します。平均実行時間とピーク実行時間は、頻繁に実行されるSQL性能を分析できる根拠を提供します。

<InDoc product="postgresql">

頻繁に性能分析のためによく分析するカテゴリ(DB, Application, USER, HOST) 別にTop SQLを検索することができます。主な参照基準は次のとおりです。

</InDoc>

<InDoc product="db2">

パフォーマンス測定でよくで使う分析カテゴリ(USER、HOST、APPLICATION)ごとにTop SQLを照会できます。主な照会基準は次のとおりです。

</InDoc>

<InDoc product="mysql">

パフォーマンス測定でよくで使う分析カテゴリ(DB、USER、HOST)ごとにTop SQLを照会できます。主な照会基準は次のとおりです。

</InDoc>

<InDoc product="oracle-pro,oracle">

パフォーマンス測定でよくで使う分析カテゴリ(DB、USERNAME、MACHINE、PROGRAM)ごとにTop SQLを照会できます。主な照会基準は次のとおりです。

</InDoc>

- **_execute count_**: SQLの実行回数

- **_elapse sum (sec)_**: SQLの実行時間(秒) 合計

- **_elapse avg (sec)_**: SQLの平均実行時間(秒)で**elapse sum**/**execute count**の結果値

- **_elapse max (sec)_**: SQLの最大実行時間(秒)

<Xclude product="mysql,mssql,mssql-v1,tibero">

- **_elapse wait (sec)_**: SQLの待機時間合計(秒)

</Xclude>

<InDoc product="altibase">

- **_data page read_**: データページを物理的に読み取った回数

- **_data page gets_**: データページを読み取るためにアクセスした回数

- **_rebuild count_**: 再構成作業の回数

- **_memory table access count_**: メモリテーブルにアクセスした回数

- **_execute success count_**: 正常に実行されたコマンドの回数

- **_user_**: データベースにリンクされているユーザー名

- **_sql hash_**: クエリのハッシュ値

</InDoc>

<Xclude product="mssql,mssql-v1,tibero,cubrid,db2">

- **_instance_**: インスタンス名

</Xclude>

<InDoc product="oracle,oracle-pro">

- **_physical reads_**: 該当SQLがディスクからデータを読み込んだ回数

- **_session logical reads_**: 該当SQLがメモリ内でデータを読み込んだ回数

</InDoc>

<InDoc product="oracle-pro">

- **_cpu time_**: 該当SQLの実行中に使用されたCPU時間

- **_redo size_**: 該当SQLが発生させたトランザクションログ(redo log) のサイズ

</InDoc>

:::info

<Cmdname sid="side_dbSqlStat" className="uitext" />機能は、リテラル値を除いたSQLを基準に統計を算出します。

:::
