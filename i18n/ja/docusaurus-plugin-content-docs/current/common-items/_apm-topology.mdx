## トポロジーとは？

モニタリング対象サーバーから収集されたリアルタイムの情報により、アプリケーションサーバー間の相関関係を容易に理解できます。

次の動画をご覧になって、トポロジーの詳細を確認してください。

{@youtube: 10sYNSZNZEk}

:::note

一般的なトポロジ定義では、リンク情報を持たないNodeは含まれません。モニタリングの観点からのトポロジには、分離したNodeの情報も必要です。

:::

## トポロジーの基本要素

トポロジー画面で表示される情報は、ノードとリンク、それぞれに含まれる追加情報に分割されます。

<ImgLang img="topology-element.png" desc="トポロジーノードのエラー情報" />

-   ![指示線1](/img/number-01.png) **Node** 

    プロジェクト、アプリケーションのグループ、アプリケーション、データベース、HTTPリクエスト対象の外部ホスト、外部モジュール(アプリケーションのリクエスト元)を意味します。

    :::note

    **Node**からの呼び出し情報がある場合、**Node**外側にエラーの重みを意味します。

    :::

-   ![指示線2](/img/number-02.png) **リンク**

    アプリケーションまたはアプリケーション グループ間の呼び出し情報、データベース呼び出し情報、外部ホスト呼び出し情報、外部モジュールからの呼び出し情報を意味します。

    -   **_平均応答時間_**：5秒間のNode間でトランザクションまたは、呼び出しの応答時間の合計を合計数で割った値

    -   **_件数_** ：5秒間のNode間のトランザクションまたは、呼び出しの(エラー件数を含む) 合計数

    -   **_エラー_**：5秒間のNode間でのトランザクションまたは、呼び出し履歴の総エラー件数

    -   **_アクティブTX_**：Node間のトランザクションまたは呼び出しの現在の進行状況の件数

    -   **_パッチの時間_**：5秒間のDBの呼び出し履歴のレコード取得時間の合計

    -   **_パッチの件数_**：5秒間のDB呼び出し履歴のレコード取得件数の合計
