## トポロジーとは

モニタリング対象サーバーから収集されたリアルタイムの情報により、アプリケーションサーバー間の相関関係を容易に理解できます。

次の動画を通じてトポロジーについて確認してください。

{@youtube: 10sYNSZNZEk}

:::note
一般的なトポロジ定義では、リンク情報を持たないNodeは含まれません。モニタリングの観点からのトポロジには、孤立したNodeの情報も必要です。
:::

## トポロジーの基本要素

トポロジー画面で表現する情報は、ノードとリンク、それぞれが含む付加情報に区分します。

![トポロジノードのエラー情報](/img/topology-element.png)

-   ![指示線1](/img/number-01.png) **Node** 

    プロジェクト、アプリケーション グループ、アプリケーション、データベース、HTTP呼び出し対象の外部ホスト、外部モジュール（アプリケーションの呼び出し元）を表現します。

    :::note
    **Node**から呼び出し情報が存在する場合、**Node**外郭にエラー比重を表現します。
    :::

-   ![指示線2](/img/number-02.png) **リンク**
    アプリケーションまたはアプリケーション グループ間の呼び出し情報、データベース呼び出し情報、外部ホスト呼び出し情報、外部モジュールからの呼び出し情報を表現します。
    -   **_平均応答時間_**：5秒間Node間でトランザクションまたは呼び出し履歴の応答時間の合計を総件数で割った情報
    -   **_件数_** ：5秒間Node間でのトランザクションまたは呼び出し履歴の（エラー件数を含む）総件数
    -   **\*エラー**：5秒間Node間でのトランザクションまたは呼び出し履歴のうち、総エラー件数
    -   **_アクティブTX_**：Node間でのトランザクションまたは呼び出し中の現在の進行状況の件数
    -   **_パッチの時間_**：5秒間DBの呼び出し履歴のレコード取得時間の合計
    -   **_パッチ件数_**：5秒間DB呼び出し履歴のレコード取得件数の合計
