ホーム画面 > プロジェクト選択 > **_ダッシュボード_** > **_アプリケーションダッシュボード_**

WhaTapモニタリングサービスのホーム画面でプロジェクトを選択し、**_プロジェクトメニューの_**下部の**_ダッシュボード_** > **_アプリケーションダッシュボード_**を選択します。

<ImgLang img='apm-dashboard.png' desc='アプリケーションダッシュボード'/>

エージェントから収集した情報は、**_ダッシュボード_**でリアルタイムで確認できます。CPU、Memoryを除く一般的なチャートの場合、安定したデータは青色の系列で表現され、問題として識別される要素は赤色の系列で表示されるため、現状を簡単に認識できます。

-   ![リアルタイム通知icon](/img/ico-noti.svg)**リアルタイム通知**：最近発生したイベントを確認できます。切り替えメニューをクリックしてブラウザ通知をオンまたはオフにすることができます。

## ![](/img/number-02.png)エージェント領域

現在のプロジェクトに関連付けられているエージェントを選択してフィルタリングできます。選択したエージェントによってウィジェット領域の指標が変更されます。

-   アクティブなエージェントの場合は青色で表示し、無効なエージェントはグレーで表示します。

-   右側の![down-arrow](/img/down-arrow.svg)ボタンをクリックすると、エージェントをサーバー別、種類別にフィルタリングしたり、検索することができます。

## ![](/img/number-03.png)ウィジェット(Widget) 領域

ウィジェットに表示されているアイコンの機能は次のとおりです。

-   ![データマージのアイコン](/img/ico-data-merge.svg) **_データマージ_**：ウィジェット項目のすべてのエージェントデータをマージし、グラフとして表示します。

-   ![個別アイコン](/img/ico-each-data.svg) **_個別_**：各エージェントごとに分けてグラフで表示します。ただし、プロジェクトにアプリケーションの数が多すぎると効果がありません。

-   ![詳細アイコン](/img/ico-detail.svg) **_詳細_**：ウィジェット項目のデータを詳細に表示し、モーダルウィンドウが表示されます。 **_マトリックスをもっと見る_**を選択すると、**_分析_** > **_メトリクスチャート_**メニューへ移動し、ユーザーが目的とする時間帯別のデータを参照することができます。

-   ![チャート設定アイコン](/img/ico-setting.svg) **_チャート設定_**：チャートに表示する要素の設定を変更できます。変更をチャートに適用するには、**_Save_**ボタンをクリックします。

    -   **_HorizontalLine Setting_**：チャートでユーザーが目的の数値をしきい値として設定することができます。しきい値のラベル、数値、色を設定でき、追加することもできます。

    -   **_Y tick static max setting_**：Y軸に表示する最大数値を設定できます。ユーザーが目的とする数値を入力するには、**_Custom_**を選択し、数値を入力します。

:::note

**_All APP._**アイコンが表示されているウィジェットは、アプリケーション全体に対応するデータです。

:::

### スピードメーター

![スピードメーター](/img/speedmeter.png)

スピードメーターで左から右に渡すドロップ型のアイコンは、トランザクションを意味します。アクティブトランザクションの状態を速度(<span class='normal'>Normal</span>, <span class='slow'>Slow</span>, <span class='vslow'>Very Slow</span>)で分類し、右上に数値を表示します。中央の領域は進行中のトランザクションを表し、右側の領域は終了したトランザクションです。

### エージェント要約

<ImgLang img='agent-dashboard-summary.png' desc='エージェント要約' />

登録したエージェントの実行状態と個数を表示します。

### **_アクティブステータス_**{#active-status}

<ImgLang img='apm-dashboard-active-status.png' desc='アクティブステータス' />

進行中のトランザクションは実行間隔で分類され、イコライザーとして表示されます。プロジェクト内のすべてのエージェントは、5秒ごとにアクティブなトランザクション数を収集すると同時に進捗状況を収集します。アクティブなステータスを**_METHOD_**、**_SQL_**、**_HTPC_**、**_DBC_**、**_SOCKET_**に分類します。

イコライザーの色は重要度を意味します。**_DBC_**と**_SOCKET_**の、1つまたは、2つが継続して現れるまで問題として認識する必要があります。一方で、**_SQL_**と**_HTTPC_**は個数が比較的に大きい場合は分析が必要で、**_METHOD_**は一般的な状況であるため青色です。![right-arrow](/img/right-arrow.svg)ボタンをクリックすると、**_ダッシュボード_** > **_アクティブトランザクション_**メニューに移動します。

-   **_METHOD_**：メソッドが実行されている状態です。

-   **_SQL_**：SQLが実行されている状態です。

-   **_HTTPC_**：外部APIを呼び出し中の状態です。

-   **_DBC_**：トランザクションがConnection Poolから新しいConnectionを取得(get)しようとしている状態です。

-   **_SOCKET_**：外部からTCP Socketを接続している状態です。

:::note

**_DBC_**または**_SOCKET_**の数が1以上続く場合は、問題があると考えられます。

:::

### **_アクティブTX_**{#active-trs}

<ImgLang img='apm-dashboard-active-tx.png' desc='アクティブトランザクション' />

**_アクティブトランザクション_**は、進行中のトランザクションを意味します。**_アークイコライザー_**チャートとして表現されます。エージェント数でアークが分割されます。5秒ごとにサーバによって処理されているリクエストの数を表現して、各リクエストが処理されている時間を確認できます。5秒間隔で検知されたリクエストは、リスクがあるかどうかを識別するために、次のように色で分類します。

-   赤色(<span class='vslow'>Very Slow</span>)：8秒以上 

-   オレンジ色(<span class='slow'>Slow</span>)：3秒~8秒

-   青色(<span class='normal'>Normal</span>)： 0秒~3秒

青色が多い状況は問題ありません。これらの一部が紫色や赤色に変わるかどうか推移を見守る必要があります。トラブルが発生すると、進行中のトランザクション数が増加し、赤色の割合が高くなります。一方で、レスポンスが非常に速いシステムの場合、処理するトランザクション数(TPS)が多くても、進行中のトランザクション数は低い場合があります。

-   画面の左上には、進行中のトランザクションのほとんどのエージェント名が表示します。

-   画面右上のアイコンを選択してビューを切り替えることができます。![right-arrow](/img/right-arrow.svg) ボタンをクリックすると、**_ダッシュボード_** > **_アクティブトランザクション_**メニューに移動します。

    -   ![非アクティブエージェントを含むアイコン](/img/ico-disable-agent.svg)**_非アクティブエージェントを含む_**：非アクティブ化されたエージェントを含みます。

    -   ![アークイコライザーアイコン](/img/ico-arc-equalizer.svg) **_アークイコライザー_**：アークイコライザーでグラフを表示します。

    -   ![縦イコライザーアイコン](/img/ico-vertical-graph.svg) **_縦イコライザー_**：縦イコライザーでグラフを表示します。

-   真ん中の数字は、選択したエージェントのアクティブトランザクション数の合計です。

-   アクティブトランザクションの周りにある2本の棒は、処理量(TPS)に基づいて3段階の速度でグラフの周りを回転します。

:::note

トランザクションの詳細については、[次の文書](track-transactions-intro)を参照してください。

:::

### **_ヒットマップ_**

**_ヒットマップ_**チャートは、終了したトランザクションの応答時間を分布図で表現します。横軸はトランザクションの終了時間です。縦軸は実行時間です。実行時間の長いトランザクションは、分布図の上部に表示されます。**_ヒットマップ_**内の領域をドラッグすると、詳細なトランザクション情報を確認できます**_トレース分析_**ウィンドウが表示されます。**_トレース分析_**についての詳細は[次の文書]](dashboard-hitmap-trace)を参照してください。

:::note

-   エラーが発生せず、実行時間が500ms以下のトランザクションの詳細は、URLあたり5分に1回のみ収集されます。

-   TPSや平均応答時間などの統計情報には影響しません。

-   エージェント設定により、ポリシーを調整できます。各エージェントの「設定」に関する文書でトランザクションの詳細を参照してください。

:::

**_ヒットマップ_**でドットカラーの色の濃さの密度を表現します。

-   正常TR：空色 → 青色 → 藍色

-   エラーTR：黄色 → 橙色 → 赤色

<ImgLang img='hitmap.png' desc='ヒットマップ' />

-   画面の右上に位置する数字はチャート内の<span class='normal'>トランザクション数と</span>/<span class='slow'>エラー数</span>を意味します。

-   **_Y軸自動_**ボタンをタップすると、実行時間を示すために、Y軸が自動的に調整されます。

-   **_エラー_**ボタンを選択すると、エラートランザクションのみ表示します。

-   ![上向きアイコン](/img/up-arrow.svg)または![下向きアイコン](/img/down-arrow.svg)ボタンを選択して、5秒~80秒でチャートを拡大／縮小することができます。

### **_TPS_**

Tansaction Per Second(TPS)は、毎秒処理されるトランザクション数を意味し、サービス性能指標の基準となります。5秒ごとに処理されたトランザクション数を1秒あたりの数に換算してチャートで表現します。直前10分間のTPSを表示します。

![TPS](/img/apm-dashboard-tps.png)

### **_本日TPS_**

1日のTPSを時間帯別に表示します。昨日の同時間帯のTPS(グレー)と比較して確認できます。![right-arrow](/img/right-arrow.svg)ボタンをクリックすると、**_統計/レポート_** > **_日付別アプリケーションの状態_**メニューに移動します。

### **_本日のユーザー_**

1日のユーザー数を時間帯別に表示します。昨日の同時間帯のユーザー数(グレー)と比較して確認できます。![right-arrow](/img/right-arrow.svg)ボタンをクリックすると、**_統計/レポート_** > **_日付別アプリケーションの状態_**メニューから移動します。

### **_平均応答時間_**

アプリケーションサーバーがユーザーにリクエスト結果を返すのに掛かる時間をリアルタイムでモニタリングします。**_平均応答時間_**は、チューニング指標として意味があります。WhaTapのモニタリングサービスは、5秒間隔でトランザクションの平均応答時間を計算します。

### **_システムCPU_**

アプリケーションサーバーのCPU使用量です。リアルタイムでCPU使用量の変化推移を把握することができます。

### **_ヒープメモリ_**{#heap-memory}

各サーバーごとに使用可能な最大メモリと現在のメモリを表示して、危険な数値にあるサーバーを確認できます。時間の経過によるメモリ使用量の変化をリアルタイムで確認できます。

メモリ折れ線チャートは通常、波打ちを続けます。アプリケーションサーバーがリクエストを処理するためにメモリを使用するとメモリ折れ線チャートは増加します。GC(Garbage Collection)によってメモリをクリーンアップすると減少します。**_ヒープメモリー_**指標は平均値を使用して表示します。

:::note

-   JavaとNode.jsは、ヒープメモリの合計量と使用量を意味します。

-   PHPおよびPython, .Netはプロセスのメモリの使用量を意味します。

:::

### **_同時接続ユーザー_**

リアルタイムのブラウザのユーザー数を表示します。5秒ごとに直前の5分以内にトランザクションを行ったユーザーをカウントして表示します。ユーザーはブラウザのIPに基づいてカウントします。エージェント設定でユーザーを識別するためにIPを使用して、クッキーを使用することができます。

:::note

-   通常、接続ユーザーはネットワークに接続するユーザーを意味します。しかし、ウェブシステムは未接続のネットワークを使用するため異なります。接続されているという意味ではなく、最近のリクエストを送信したユーザーが測定の対象になる可能性があります。

-   リアルタイム(同時接続) ユーザーとは、同じ時間帯にシステムを使用しているユーザーを意味します。リアルタイム(同時接続) ユーザーを計算または測定する方式はさまざまです。

-   ユーザー数は、単に合計されるのではなく、**HyperLog**によって計算されます。**HyperLog**は、非常に少ないメモリ内の要素の数を推定する確率的なデータ構造を意味します。

-   **_アプリケーションダッシュボード_**を分析する方法の詳細は、[次の文書](../best-practice-guides/about-apm-dashboard)を参照してください。

:::
