## イベント履歴を照会する{#eventview}

次のオプションを順番に設定した後、![検索アイコン](/img/ico-btn-search.png)ボタンを選択します。設定したオプションを基準にイベント記録を照会できます。

- <Cmdname sid="time_selector" className="uitext" />: 照会しようとしているイベントの時間範囲を設定できます。<Cmdname sid="event_time" className="uitext" />を基準に照会します。

  :::note

  <Cmdname sid="time_selector" className="uitext" />オプションの利用方法の詳細については、[次の文書](usage-ui#timeselector)を参照してください。

  :::

- <Cmdname sid="filter" className="uitext" />: <Cmdname sid="event_title" className="uitext" />または<Cmdname sid="message" className="uitext" />内容に基づいて入力した文字列を含むイベントの一覧をフィルタリングできます。

<Xclude product="browser,url">

- <Cmdname sid="agent" className="uitext" />: 特定のエージェントで発生したイベントのみを選択してフィルタリングできます。

</Xclude>

## 処理履歴を登録する{#alertacknowledge}

処理履歴を登録すると、イベント処理状況のアップデートと履歴管理が簡単にできます。

:::caution

処理履歴を登録すると、当該イベントの<Cmdname sid="TTL06149" className="uitext" />は中止となります。<Cmdname sid="TTL06149" className="uitext" />についての詳細は、[次の文書](set-receive-event#escalation)を参照してください。

:::

1. 処理履歴を登録するイベントで<Cmdname sid="sms_alert_acknowledge" className="uitext" />ボタンを選択します。

2. <Cmdname sid="sms_alert_acknowledge_add" className="uitext" />画面が表示されたら、処理履歴を入力します。

   <ImgLang img="event-history-add-acknowledge.png" desc="処理内容追加" />

3. <Cmdname sid="registration" className="uitext" />ボタンを選択します。

登録した処理履歴は、<Cmdname sid="TTL08669" className="uitext" />に一覧として表示されます。作成時間と作成者、詳細を確認できます。

<ImgLang img="event-history-add-acknowledge-history.png" desc="処理履歴詳細" />

イベント一覧で追加した処理履歴の個数を確認できます。

<ImgLang img="event-history-add-acknowledge-added.png" desc="処理内容" />

:::note

処理履歴は最大255 byteまで入力できます。

:::

## 付加機能を利用する{#additionaloption}

- <Cmdname sid="BTN06441" className="uitext" />: イベント一覧のカラム項目を追加と削除、順序を変更できます。

- <Cmdname sid="exclude_resolved_alerts" className="uitext" />：イベント記録一覧で**進行中のイベント**のみ照会できます。

- <Cmdname sid="event_setting" className="uitext" />: <Cmdname sid="event_statistic" className="uitext" /> &gt; <Cmdname sid="event_setting" className="uitext" />メニューに移動します。
