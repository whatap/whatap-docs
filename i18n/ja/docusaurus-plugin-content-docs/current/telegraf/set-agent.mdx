WhaTapエージェントは、エージェントごとに必要な設定を_whatap.conf_ファイルにに書き込みます。 エージェントは、環境変数を使用して設定ファイルを見つけて読み込みます。 _whatap.conf_ファイルを使用して設定できるオプションは次のとおりです。

## エージェント名の識別

WhaTapは、モニタリング情報収集対象のインフラサーバーの識別のために、基本的にはサーバーから収集した情報を活用します。 

{@include: ../common-items/_agent-name-warning.mdx}

-   **object_name** <span class='type'>String</span>

    デフォルト値`{type}`-`{hostname}`-`{ip2}`-`{ip3}`-`{docker}`

    アプリケーションを識別するエージェント名(ONAME) システムです。ONAMEに基づいてOIDが生成されます。

    | 名称         | 説明                                                        |
    | ---------- | --------------------------------------------------------- |
    | {type}     | app_nameに設定された値を使用します。 デフォルト値はTelegrafです。                 |
    | {ip#}      | IPを分割するときは、#番目の桁を使用します。                                   |
    | {hostname} | サーバーのホスト名を使用します。                                          |
    | {docker}   | DockerコンテナIDを使用します。                                       |

-   **app_name** <span class='type'>String</span>

    アプリケーションを識別するエージェント名(ONAME) 体系で使用されるアプリケーション名です。`object_name`の`{type}`に対応する値です。

    :::note

    whatap-telegrafのサービスを再起動した後に適用されます。

    :::

## エージェントの通信設定

-   **license** <span class='type'>String</span>

    エージェントのインストール時にサーバーから割り当てられたプロジェクトアクセスキーを設定します。 プロジェクトアクセスキーには、エージェントが属するプロジェクトとセキュリティ通信のためのパスワードキーが含まれています。

-   **whatap.server.host** <span class='type'>ip_address</span>

    デフォルト値`127.0.0.1,127.0.1`

    エージェントが収集したデータを送信するサーバーを指定します。収集サーバーを二重化して2つ以上のIPを持つ場合、コンマ（,）で区切って指定することができます。指定されたIPには、収集サーバーのproxyデーモンがリスニング状態でサービスされる必要があります。

-   **whatap.server.port** <span class='type'>tcp_port</span>

    デフォルト値`6600`

    収集サーバーのポート番号を指定します。 ポートは1つしか指定できないので、`whatap_server_host`に指定されている収集サーバーは同じポート番号を使用する必要があります。

-   **tcp_so_timeout** <span class='type'>Millisecond</span>

    デフォルト値`60000`

    収集サーバーと通信するTCPセッションのSocket Timeout値を指定します。

-   **tcp_connection_timeout** <span class='type'>Millisecond</span>

    デフォルト値`5000`

    収集サーバーと通信するTCPセッションのConnection Timeout値を指定します。

-   **net_send_max_bytes** <span class='type'>Byte</span>

    デフォルト値`5242880`

    収集サーバーにデータを送信するときに、一度に送信される最大サイズを指定します。

-   **net_send_buffer_size** <span class='type'>Byte</span>

    デフォルト値 `1024`

    データ送信を行う必要がある最大バイトのサイズです。 

## Telegraf

-   **telegraf_tcp_port** <span class='type'>tcp_port</span>

    デフォルト値`6600`

    Telegraf socker_writer output pluginが接続するTCPのポート番号です。

    :::note

    whatap-telegrafのサービスを再起動した後に適用されます。

    :::

-   **telegraf_delta_fields** <span class='type'>String</span>

    デフォルト値 `1024`

    累積値を持つフィールドの増加値を計算し、別のフィールドを追加します。 name.fields形式として指定し、複数はコンマ（,）で区切ります。

    e.g. nginx.accepts,nginx.requests nginxのaccepts, requests値の増加値をacceptsDelta, requestsDeltaフィールド名で追加して渡します。

## ログ

-   **log_keep_days** <span class='type'>Day</span>

    デフォルト値`7`

    ログファイルの保存期間を設定します。
