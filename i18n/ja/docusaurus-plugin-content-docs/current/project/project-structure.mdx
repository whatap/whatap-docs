WhaTapモニタリングサービスでは、プロジェクト階層構造（Grouping）を活用して複数のプロジェクトを便利に管理できます。

## 階層構造

<ImgLang img='project-management-3depth.png' desc='프로젝트 계층 구조' className='noneborder'/>

### プロジェクトとは

WhaTapのモニタリングプロジェクトは、モニタリング単位を意味します。プロジェクト単位でモニタリング単位が区分されるとともに、テナント権限も管理されます。プロジェクトは最も基本的な階層です。

### グループとは

複数のプロジェクトを管理する際にグループを活用すると役立ちます。2つ以上のプロジェクトになると管理するのが面倒になります。それぞれのプロジェクトごとにユーザーを追加または権限を付与するにはどうすればよいでしょうか？ この場合、2つのプロジェクトを1つのグループにまとめ、グループとしてユーザーを管理できます。1人のユーザーは、1つのグループで複数のプロジェクトを所有できます。

プロジェクトユーザーはグループユーザーを継承します。グループに属するユーザーはグループ内のすべてのプロジェクトをモニタリングできます。特定プロジェクトに限って権限が付与されたユーザーなら、グループの有無に関係なく、そのプロジェクトだけをモニタリングできます。

### 組織とは

組織は、複数のグループをまとめる上位グループです。ほとんどの状況では、プロジェクトとグループだけで十分です。

:::note 

**どんな顧客が有効に活用できますか？**

運用・管理サービスを専門とする企業であれば、複数の会社がグループとして存在します。グループやプロジェクトの数が非常に多いかもしれません。この場合、グループという概念だけでは十分ではありません。グループの統合管理できる更なる概念が必要です。組織という概念を活用してみてください。

専門的な運用管理サービス企業は、複数のサブグループを1つの組織にグループ化することで管理できます。各グループメンバーに権限を付与してみてください。権限を付与されたメンバーは、個別にグループを管理できます。

:::

## メンバー権限{#member-auth}

上位グループが存在する場合は、基本的に上位グループのメンバー権限を継承します。例えば、組織A、グループB、プロジェクトCがあるとします。 この場合、別途設定をしない場合、グループBは組織Aのメンバー権限を継承し、プロジェクトCはグループBのメンバー権限を継承します。

<ImgLang img='project-member-auth.png' desc='멤버 권한' className='noneborder'/>

:::note

プログラミングでいう「継承」は、一般的な継承概念とは意味が異なります。例えば、組織Aのユーザ1の権限をグループBのユーザ1に継承すると仮定すると、その後もグループBのユーザ1は該当権限を維持することになります。

:::

## メンバー権限体系{#project-auth}

ユーザーに各プロジェクト、グループごとに異なる権限を他のメンバーに柔軟に付与できます。以下は、各権限に関する基本的な説明です。

<div class='c5-3-5-avg'>

| 権限体系              | 説明                                          |
| ----------------- | ------------------------------------------- |
| ビュー               | プロジェクト照会                                    |
| 修正                | 組織、グループ、プロジェクトの移動・修正・削除                     |
| ログ読取り             | ログ照会、ログ関連機能へのアクセス                           |
| ログエディット           | ログ設定編集                                      |
| プロジェクトのFlexボードの修正 | プロジェクトの**_Flexボード_**編集                      |
| メンバー管理            | メンバー権限の修正、メンバーの追加・除外                        |
| アラート通知受信          | アラート通知受信                                    |
| アラート設定            | アラート受信設定                                    |
| 決済                | 金額請求                                        |

</div>

### 照会・分析

組織、グループ、プロジェクトにメンバーを招待する場合、そのメンバーに対して基本的に付与する権限です。**_ダッシュボード_**およびその他のメニューページを見ることができます。

### 修正

-   プロジェクトの機能を修正できます。例えば、**_ダッシュボード_**の追加・修正・削除などができます。 

-   グループやプロジェクトの移動・修正・削除ができます。

### メンバー管理

-   組織、グループ、プロジェクトごとに所属するメンバーの追加・削除・権限の編集ができます。

-   必ず1人以上の**メンバー管理**権限を持つメンバーが必要です。

### アラート通知受信

-   **_警告通知_** > **_イベント受信設定_**メニューで通知受信の設定を制御できます。 

-   **通知受信**権限を設定しない場合は、自分のアカウントに対する受信設定に限って可能です。

### アラート設定

**_警告通知_** > **_イベント受信設定_**メニューで通知の追加・削除・修正ができます。その他すべての機能を制御できます。

### ログ読取り

-   ログを照会することができます。組織、グループ、プロジェクトにメンバーを招待する場合、そのメンバーに対して基本的に付与する権限です。

-   **ログ照会**権限を持つメンバーに限ってログ関連機能にアクセスできます。ログ関連機能は次のとおりです。

    -   **_ログ_**メニューと下位メニュー

    -   **_警告通知_** > **_イベント設定_** > **_対数_** タブ

    -   トランザクション詳細ウィンドウの**_トランザクションログ_**タブ

    -   コンテナマップメニューで**要約を見る**、**詳細を見る**の**_ログ_**タブ

### ログエディット

**_ログ_** > **_ログ設定_**メニューで設定を編集する権限です。**ログ編集**権限がある場合**_ログ設定_** メニューを利用できます。

:::note

**ログ編集**権限を持っていても**アクセスキー**を発行されていないか、ログモニタリングが非活性化になっている場合は、**_ログ設定_**メニューは利用できません。

:::

### プロジェクトのFlexボードの修正

プロジェクトの**_Flexボード_**メニューでダッシュボードを修正できる権限です。プロジェクト内の**_Flex ボード_**メニューでは、**_修正モード_**および**_管理者_**モード、**_フィルター_**機能にアクセスできます。

:::note

Flexボードの詳細については、[次の文書](../java/flex-board)を参考にしてください。

:::

### 決済

-   組織、グループ、プロジェクト別に**決済**権限は1人が持ちます。決済権限を持つメンバーは、必須で1人存在する必要があります。

-   **決済**権限を持つメンバーの横に**_決済_**と表示されます。

-   もしAが持っていた**決済**権限をBに譲渡すると、Aはこれ以上**決済**権限がありません。

-   **決済**権限を譲渡されるメンバーは、決済情報を登録した状態である必要があります。決済情報は**_利用内訳_**メニューで登録することができます。詳しくは[次の文書](../management/billing)を参考にしてください。

-   組織、グループ、プロジェクトにそれぞれ決済担当者が存在する場合、プロジェクト、グループ、組織の順に決済権限が与えられます。グループ、プロジェクトに決済権限者がいない場合は、上位グループまたは上位組織の決済権限者に決済が請求されます。

## メンバー権限修正{#modify-auth}

### プロジェクトメンバー権限の修正

現在選択している組織やグループ、プロジェクトのメンバー一覧で所属メンバーのユーザー権限を修正できます。 

<ImgLang img='member-modify-authority-0508.png' desc='멤버 권한 수정' />

右側のトグルボタンを選択して権限を設定できます。設定を完了した後、**_適用_**ボタンを選択してください。

### 組織またはグループメンバー権限の修正

組織またはグループの下位に属するグループまたはプロジェクトに対してメンバー権限を設定できます。

<ImgLang img='modify-organize-member-authority.png' desc='멤버 권한 수정' />

**_組織のグループ別のユーザー権限_**または**_グループのプロジェクト別のユーザー権限_**セクションの下で権限を修正しようとする項目の! [編集アイコン](/img/ico-edit.svg)ボタンを選択してください。**_メンバー権限の修正_**ウィンドウが表示されます。権限に対して設定できるオプションは次のとおりです。

-   **_継承（許可）_**：上位組織またはグループの権限を継承します。

-   **_許可_**：現在選択しているグループまたはプロジェクトの権限を使用できるよう設定します。

-   **_非許可_**：現在選択しているグループまたはプロジェクトの権限を使用できないよう設定します。

:::note

メンバーの追加・削除・権限修正の詳細については、次の文書を確認してください。
