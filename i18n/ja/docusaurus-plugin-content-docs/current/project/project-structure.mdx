WhaTapモニタリングサービスでは、プロジェクト階層構造(Grouping)を使用して複数のプロジェクトを便利に管理できます。

## 階層構造

<ImgLang img='project-management-3depth.png' desc='プロジェクト階層' className='noneborder'/>

### プロジェクトとは?

WhaTapのモニタリングプロジェクトは、モニタリング単位を意味します。プロジェクト単位でモニタリング単位が区分に加えて、テナントのアクセス権限も管理されます。プロジェクトは最も基本的な階層です。

### グループとは

複数のプロジェクトを管理する場合は、グループがあると便利です。2つ以上のプロジェクトになると管理が複雑になります。各プロジェクトごとにユーザーを追加または権限を付与するにはどうすればよいですか？ この場合、2つのプロジェクトを1つのグループにまとめ、グループとしてユーザーを管理できます。1人のユーザーは、1つのグループで複数のプロジェクトを所有できます。

プロジェクトユーザーはグループユーザーを継承します。グループに所属するユーザーは、グループ内のすべてのプロジェクトをモニタリングできます。特定プロジェクトに対して権限が付与されたユーザーがグループの有無に関係なく、そのプロジェクトのみをモニタリングできます。

### 組織とは

組織は、複数のグループをまとめる上位グループです。ほとんどの状況では、プロジェクトとグループだけで十分です。

:::note 

**どんな顧客が有効に利用できますか？**

運用管理サービスを専門とする企業であれば、複数の会社がグループとして存在します。グループやプロジェクトの数が非常に多くなる可能性があります。この場合、グループの概念だけでは不十分です。グループを統合管理できるようにする概念が必要です。組織という概念を使用します。

専門的な運用管理サービス企業は、複数のサブグループを1つの組織にグループ化することで管理できます。各グループメンバーに権限を付与してみてください。権限を付与されたメンバーは、個別にグループを管理できます。

:::

## メンバー権限{#member-auth}

上位グループが存在する場合は、基本的に上位グループのメンバー権限を継承します。例えば、組織A、グループB、プロジェクトCがあるとします。 この場合、別の設定を行わない場合、グループBは組織Aのメンバー権限を継承し、プロジェクトCはグループBのメンバー権限を継承します。

<ImgLang img='project-member-auth.png' desc='メンバー権限'className='noneborder'/>

:::note

プログラミングでいう「継承」は、一般的な継承の概念とは意味が異なります。例えば、組織Aのユーザ1の権限をグループBのユーザー1に継承すると仮定すれば、その後も組織Aのユーザー1はその権限を維持することになります。

:::

## メンバー権限体系{#project-auth}

ユーザーは、各プロジェクト、グループごとに異なる権限を異なるメンバーに柔軟に付与できます。次は、各権限に関する基本的な説明です。

<div class='c5-3-5-avg'>

| 権限体系              | 説明                                                      |
| ----------------- | ------------------------------------------------------- |
| ビュー               | プロジェクト参照                                                |
| 修正                | 組織、グループ、プロジェクトの移動、修正、削除                                 |
| ログ読取り             | ログ参照、ログ関連機能へのアクセス                                       |
| ログエディター           | ログ設定の編集                                                 |
| プロジェクトのFlexボードの修正 | プロジェクトの<Cmdname sid="flexboard" className="uitext" />編集 |
| メンバー管理            | メンバー権限の修正、メンバーの追加と除外                                    |
| アラート通知受信          | アラート通知受信                                                |
| アラート設定            | アラート受信設定                                                |
| 課金                | 金額請求                                                    |

</div>

### ビュー

組織、グループ、プロジェクトにメンバーを招待する場合、これらのメンバーに基本的に付与する権限です。<Cmdname sid="dashboard" className="uitext" />およびその他のメニューページを表示することができます。

### 修正

- プロジェクトの機能を修正できます。例えば、<Cmdname sid="dashboard" className="uitext" />の追加、修正、削除などができます。 

- グループやプロジェクトの移動、修正、削除ができます。

### メンバー管理

- 組織、グループ、プロジェクトごとに所属するメンバーの追加、削除、権限の編集ができます。

- 必ず1人以上の**メンバー管理**権限を持つメンバーが必要です。

### アラート通知受信

- <Cmdname sid="event_statistic" className="uitext" /> > <Cmdname sid="event_receive_configuration" className="uitext" />メニューから通知受信設定を制御できます。

- **通知受信**権限を設定しない場合は、アカウントにの受信設定のみを行うことができます。

### アラート設定

<Cmdname sid="event_statistic" className="uitext" /> > <Cmdname sid="event_receive_configuration" className="uitext" />メニューからアラート通知の追加、削除、修正ができます。それ以外は、すべての機能を制御できます。

### ログ読取り

- ログを検索することができます。組織、グループ、プロジェクトにメンバーを招待する場合、そのメンバーに基本的に付与する権限です。

- **ログ参照**権限を持つメンバーのみが、ログ関連の機能にアクセスできます。ログ関連の機能は次のとおりです。

  - <Cmdname sid="log" className="uitext" />メニューとサブメニュー

  - <Cmdname sid="event_statistic" className="uitext" /> > <Cmdname sid="event_setting" className="uitext" /> > <Cmdname sid="log" className="uitext" /> タブ

  - トランザクション詳細ウィンドウの<Cmdname sid="transaction_log" className="uitext" />タブ

  - コンテナマップメニューから**概要の表示**、**詳細ビュー**の<Cmdname sid="log" className="uitext" />タブ

### ログエディター

<Cmdname sid="log" className="uitext" /> > <Cmdname sid="log_setting" className="uitext" />メニューで設定を編集する権限です。<b>ログ編集</b>権限がある場合は、<Cmdname sid="log_setting" className="uitext" />メニューを使用できます。

:::note

**ログ編集**権限を持っていても、**アクセスキー**が発行されていない場合やログモニタリングが無効になっている場合は、<Cmdname sid="log_setting" className="uitext" />メニューを使用できません。

:::

### プロジェクトのFlexボードの修正

プロジェクトの<Cmdname sid="flexboard" className="uitext" />メニューでダッシュボードを修正できる権限です。プロジェクト内の<Cmdname sid="flexboard" className="uitext" />メニューから<Cmdname sid="edit_mode" className="uitext" />および<Cmdname sid="admin" className="uitext" />モード、<Cmdname sid="filter" className="uitext" />機能にアクセスできます。

:::note

Flexボードの詳細については、[次の文書](../java/flex-board)を参照してください。

:::

### 課金

- 組織、グループ、プロジェクト別に**課金**権限を1人が持ちます。課金権限を持つメンバーは、1人必要です。

- **課金**権限を持つメンバーの横に<Cmdname sid="billing" className="uitext" />と表示されます。

- Aが持っていた**課金**権限をBに移行すると、Aは**課金**権限がありません。

- **課金**権限を移行するメンバーは、決済情報を登録した状態である必要があります。決済情報は<Cmdname sid="my_usage" className="uitext" />メニューから登録することができます。詳しくは[次の文書](../management/billing)を参照してください。

- 組織、グループ、プロジェクトにそれぞれ課金担当者が存在する場合、プロジェクト、グループ、組織の順に課金権限が与えられます。グループ、プロジェクトに課金権限者がいない場合は、上位グループまたは上位組織の課金権限者に請求されます。

## メンバー権限を修正{#modify-auth}

### プロジェクトメンバー権限の修正

現在選択している組織やグループ、プロジェクトのメンバー一覧で所属メンバーのユーザー権限を修正できます。 

<ImgLang img='member-modify-authority-0508.png' desc='メンバーの権限を変更する' />

右側のトグルボタンを選択して、アクセス権限を設定できます。設定を完了した後、<Cmdname sid="apply" className="uitext" />ボタンを選択してください。

### 組織またはグループメンバー権限の修正

組織またはグループの下位に属するグループまたはプロジェクトに対してメンバーのアクセス権限を設定できます。

<ImgLang img='modify-organize-member-authority.png' desc='メンバーの権限を修正する' />

<Cmdname sid="user_role_per_group_of_organization" className="uitext" />または<Cmdname sid="user_role_per_project_of_group" className="uitext" />セクションの下で権限を修正しようとする項目の<LinkImage img="ico-edit.svg" desc="편집 아이콘" />ボタンを選択してください。<Cmdname sid="edit_member_role" className="uitext" />ウィンドウが表示されます。権限を設定できるオプションは次のとおりです。

- <Cmdname sid="inheritance" className="uitext" /><span class="uitext">(</span><Cmdname sid="allow" className="uitext" /><span class="uitext">)</span>：上位組織またはグループの権限を継承します。

- <Cmdname sid="allowed" className="uitext" />：現在選択しているグループまたはプロジェクトの権限を使用できるよう設定します。

- <Cmdname sid="deny" className="uitext" />：現在選択しているグループまたはプロジェクトの権限を使用できないよう設定します。

:::note

メンバーの追加または、削除と権限修正の詳細については、次の文書を確認してください。
