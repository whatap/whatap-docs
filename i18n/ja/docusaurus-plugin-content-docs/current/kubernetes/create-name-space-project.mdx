次のような場合は、クラスタープロジェクトからネームスペースプロジェクトから切り離しする理由です。

- １つのクラスター(Master) プロジェクトに多数のリソースがあり、Kubernetes Namespaceで切り離すことができます。

- Kubernetes Namespaceごとに部署や担当者が異なるため、特定のNamespaceのみをモニタリングする必要があります。 

クラスタープロジェクトの各Kubernetes Namespaceを別のWhaTapネームスペースに分離する場合は、***管理*** > ***ネームスペース管理***メニューを使用してプロジェクトを作成します。

<ImgLang img='whatap_kubernetes_namespace_project_mgt.png' desc='ネームスペース管理' />

:::note

***管理*** > ***ネームスペース管理***メニューは、クラスタープロジェクトを選択した場合にのみ表示されます。ネームスペースプロジェクトは、メニューに入ることができません。

:::

***ネームスペース一覧***に***+ プロジェクト追加***ボタンが表示する場合、そのボタンをクリックすると、WhaTapネームスペースプロジェクトを作成できます。ネームスペース名を入力し、***確認***ボタンを選択します。

<ImgLang img='whatap_kubernetes_create_namespace_project_popup.png' desc='ネームスペースプロジェクトの作成' />

ネームスペースプロジェクトを作成すると、新しい個別のプロジェクトに分割されるため、***ネームスペース一覧***のクラスタープロジェクトに表示されなくなります。

クラスタープロジェクトからネームスペースプロジェクトに分離され追加されたNamespaceは、次のようなステータス情報が表示されます。

<ImgLang img='whatap_kubernetes_namespace_project_mgt_complete.png' desc='ネームスペースプロジェクトの作成完了' />

クラスタープロジェクトの下位にネームスペースプロジェクトを作成した場合、クラスタープロジェクトを含むプロジェクトリストの表示やプロジェクト間の移動が可能です。***ダッシュボード*** > ***コンテナマップ***メニューの画面上のプロジェクトリストボックスを選択します。

<ImgLang img='whatap_kubernetes_container_map_namespace_project_list.png' desc='コンテナマップメニューのプロジェクトリスト' />
