WhaTapエージェントのインストールが完了しましたら、[WhaTapモニタリングサービス](https://service.whatap.io)にログインします。WhaTapモニタリングサービスは、円滑なサービス提供と安定した運用、異常検知モニタリングおよび把握のための情報を提供します。

Kubernetesには、マイクロサービスを効率的に運用できる様々な機能が含まれています。コンピューティング リソースを効率的に使用するためのスケジューリング機能とアプリケーションアップデートのためのデプロイ機能、コンテナ実行に必要なストレージとネットワーク管理など、サービス運用に必要なすべての機能を提供します。

モニタリングサービスは、ユーザーがKubernetesを運用するすべてのプロセスに役立てたいと考えています。リアルタイムで問題状況をモニタリングして障害状況を知らせ、問題解決に役立つモニタリングの基本的な機能だけでなく、リソースを分配しインフラコストを削減する運用業務時、またはKubernetesに統合された複数のマイクロサービスとそれに関連する業務部門間の協業ツールとしてモニタリングサービスが活用できます。

Kubernetesには多くの用途があるため、運用環境やビジネス状況に応じてユーザーのモニタリングのニーズも多様です。そのために、WhaTapのモニタリングサービスは、アプリケーショントランザクション
やリソース使用量、ログやイベントなど、Kubernetesが管理または生成するすべての情報を収集および制御する機能を提供します。

## リアルタイムモニタリング

- マルチクラウド、ハイブリッドクラウド上のKubernetes統合モニタリング

- コンテナマップ - ノード（ホスト）、コンテナ、アプリケーション、ログ、イベント情報を提供する動的ダッシュボード

  - Namespace、Deployment、ReplicaSet、PodなどのKubernetesの構成要素を個別/グループ化して表示

  - 個別対象 / ユーザー指定グループ別の情報をリアルタイムで確認
    - コンテナおよびコアリソースのメトリクス別リアルタイム状況を3段階の色で区分(Normal、 Warning、 Critical)
    - コンテナに割り当てられたCPU Limit、Request及び実使用量（core）
    - コンテナに割り当てられたMemory Limit、Request及び実使用量（MiB, GiB）
    - コンテナが使用中のDisk I/O、Network I/O情報
    - ヒットマップベースのアプリケーションモニタリング
    - Pod / コンテナ内で発生する標準出力/Stderr標準エラー情報
    - コンテナ / アプリケーションログを期間別検索
    - コンテナ/アプリケーションに対するユーザー指定のアラート
    - Kubernetesクラスター内イベント情報

- リソースモニタリング - コンテナおよびノードの詳細なモニタリング

  - コンテナ
    - すべてのコンテナリストと使用率の確認
    - 個別コンテナのCPU、Memory、Disk(Volume)、Network使用率
    - コンテナに接続されたボリューム情報

  - ノード
    - すべてのノードリストと使用率の確認
    - 個別ノード(ホスト)のCPU、Memory、Disk、Network使用率

- アプリケーションモニタリング - Kubernetes上でコンテナ化されたアプリケーションモニタリング
  - 対応アプリケーション：Java、Node.js、Python
  - フィルタリングを使用した特定アプリケーションのモニタリング
  - 進行中のトランザクション情報、終了したトランザクション情報のモニタリング
  - アプリケーショングループおよび個別アプリケーションのTPSおよび応答時間の推移
  - 5分間の集計ユーザー数の推移

- Kubernetesマスターのメタ情報を確認
  - ETCDに保存されているマスターメタ情報はWhaTapコンソールで確認できます。
  - 主要な Kubernetes コンポーネントの一覧と各コンポーネントのいくつかの詳細な仕様 (YAML) 情報

## 性能分析

- Kubernetes Podピーク使用量および初期化時間

- 期間別メモリ不足発生コンテナ情報とメモリチャート

- アプリケーション(APM)と同一機能

  - メトリクスチャート

  - MSA分析

  - スタック

## イベント通知

- メール、SMS、SNS(Telegram、Slack) 送信

- イベント期間としきい値設定によるイベント発生

- 無音設定時間登録による同一イベント通知の抑制機能

- CPU、Memory、Disk通知機能

- リアルタイムトランザクション数のしきい値設定による通知

- トランザクションエラー数のしきい値設定による通知

- 応答時間のしきい値のトランザクション数による通知

- 応答時間分布の水平・垂直分布パターンによる通知

## 統計/レポート

- 日次/週次レポート：リアルタイムユーザー、TPS、応答時間、CPU推移など

- 月次レポート
  - リアルタイムユーザー、TPS、レスポンス時間、CPU推移など
  - 毎日の訪問者数の推移、毎月の訪問者数、顧客ロイヤルティ指標

- アプリケーション単位の日次レポート

- アプリケーションピーク性能比較レポート：前日・前週のピークタイムとの比較

- Pod別の週次CPU、メモリレポート
