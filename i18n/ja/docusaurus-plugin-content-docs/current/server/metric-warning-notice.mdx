## 複合メトリクスのイベントテンプレート{#composite-metrics-template}

複合メトリクスイベントを追加する場合は、**_+_**<Cmdname sid="create_new_composite_mxql_event" className="uitext" />以外のテンプレートを選択して使用できます。CPU、 Disk I/O、Network I/O指標が設定したしきい値基準に基づいて通知を発生させるテンプレートを提供します。 

<ImgLang img='server-composite-metrics-template.png' desc='サーバー複合メトリクスのイベントテンプレート' />

### CPU

:::note 

CPU関連指標は、次のとおりです。`cpu`、`user_cpu`、`system_cpu`

:::

#### **_CPU % is too high_**

直前１分間で最も高く測定されたCPU関連指標がしきい値設定値(`${_rule_}`)の条件(`>`, `<`, `=`)と一致すると通知が発生します。この通知はエージェント(oname)ごとに発生します。<Cmdname sid="message" className="uitext" />例は次のとおりです。

> CPU utilization rate of the `${oname}` in the last minute > `${_rule_}` %

#### **_CPU User % is too high_**

過去１分間で最も高く測定された`user_cpu`指標がしきい値の設定値(`${_rule_}`)の条件(`>`, `<`, `=`)と一致すると通知が発生します。これらの通知はエージェント(oname)ごとに発生します。<Cmdname sid="message" className="uitext" />例は次のとおりです。

> CPU User utilization rate of the `${oname}` in the last minute > `${_rule_} `%

#### **_The number of agents with high CPU SYS % is too large_**

`system_cpu`指標が70%以上のサーバの個数がしきい値設定値(`${_rule_}`)の条件(`>`, `<`, `=`)に一致すると通知が発生します。これらの通知はエージェント(oname) ごとに発生します。<Cmdname sid="message" className="uitext" />例は次のとおりです。 

> The number of agents with a CPU SYS of 70% or more in the last minute > `${_rule_}`

### Disk

#### **_The Disk I/O is too high_**

直前1分間で最も高く測定されたDisk I/Oがしきい値設定値(`${_rule_}`)の条件(`>`, `<`, `=`)に一致すると通知が発生します。これらの通知はエージェント(oname)ごとに発生します。<Cmdname sid="message" className="uitext" />例は次のとおりです。

> In the last minute, `${oname}`'s Disk I/O > `${_rule_}` %

#### **_The Disk Used % is too high_**

直前1分間で最も高く測定されたDisk User I/Oがしきい値設定値(`${_rule_}`)の条件(`>`, `<`, `=`)に一致すると通知が発生します。これらの通知はエージェント(oname)ごとに発生します。<Cmdname sid="message" className="uitext" />例は次のとおりです。

> In the last minute,`${oname}`'s Disk Used > `${_rule_}` %
