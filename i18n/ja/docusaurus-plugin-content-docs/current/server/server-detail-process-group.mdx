## プロセスグループとは？

プロセスグループは、インスタンスに対して同じアクションを実行する複数のプロセスのグループです。個別のプロセスではなく、グループ単位でモニタリングまたは通知を受け取る場合は、プロセスグループを使用してください。

プロセスグループのメニューでは、既存の各プロセスのリソース使用量の合計を一目で確認できます。個々のプロセスのリソース使用量を計算する必要はありません。

プロセスグループのリソース使用量の合計が特定のしきい値を超えたときに通知されるように指定できます。
通知の設定に関する詳細は、 [次の文書](warning-notice#process-event)を参照してください。

## プロセスグループの追加

:::note

プロジェクト**修正**権限または**通知設定**権限が必要です。 権限の詳細については、[次の文書](../project/project-structure#project-auth)を参照してください。

:::

1. ホーム画面&gt; プロジェクト選択&gt; <Cmdname sid="side_serverList" className="uitext" /> &gt; <Cmdname sid="side_serverDetail" className="uitext" />に移動した後、プロセス一覧で<Cmdname sid="groupby" className="uitext" />ボタンをクリックします。

   <ImgLang img='server-detail-process.png' desc='サーバー詳細プロセス一覧'/>

2. プロセスグループ設定ウィンドウが表示されたら、グループに含めるプロセス名パターンを**_パターン_**に、前後のアスタリスク(\*)を含めて入力します。

   > 例）whatap → \*whatap\*

   <ImgLang img='server-detail-process-group.png' desc='サーバー詳細プロセス一覧グループ化'/>

   入力後、右側の ![虫眼鏡アイコン](/img/ic-search.svg)ボタンを選択します。そのパターンに該当するプロセスの一覧を照会できます。プロセス名は、画面に表示されるプロセス名を基準とします。

3. **_Pattern_**<Cmdname sid="alias" className="uitext" />にプロセスグループ名を入力し、右側の<Cmdname sid="apply" className="uitext" />ボタンを選択します。グループ化は**設定したプロジェクト**の**すべてのサーバー**で実行されます。

## プロセスグループの例

- **プロセスグループ化適用**

  次の画面は、**_Pattern_**に\*tomcat\*を入力し、**_Pattern_**<Cmdname sid="alias" className="uitext" />をtest_groupとして適用した例です。

  <ImgLang img='server-detail-process-group-ex.png' desc='サーバー詳細プロセス一覧グループ化例'/>

- **プロセスグループ化以前のプロセス一覧**  

  ![グループ化以前のプロセス一覧](/img/server-process-list.png)

  **_パターン_**に合わせてグループ化するプロセス一覧は次のとおりです。

  > - 例）javatomcat_left/bin/test.jarcom.virtual.App 
  > - 例）javatomcat_right/bin/test.jarcom.virtual.App

- **プロセスグループ化登録確認**

  新しいプロセスグループtest_groupが、次のようにプロセス一覧に登録されていることが確認できます。

  ![プロセスグループ化登録確認](https://img.whatap.io/media/images/test_group.png)

:::note

- グループ化の適用後、プロセスグループで各プロセスに対応するデータの計算された数値を確認できます。 

- グループ化以前のデータを確認する場合、プロセス別にデータを確認する必要があります。

:::
