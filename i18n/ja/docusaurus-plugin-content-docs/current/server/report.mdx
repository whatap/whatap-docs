ホーム画面 &gt; プロジェクト選択 &gt; <Cmdname sid="side_report" className="uitext" />

画面の上に<Cmdname sid="report" className="uitext" />で![down-arrow](/img/down-arrow.svg)ボタンをクリックすると、日刊、週刊、月刊に分類された各種分析レポートを確認できます。希望するレポート様式を選択し、照会したい日付と時間、フィルターオプションを設定してください。![검색 아이콘](/img/ico-btn-search.png)ボタンを選択すると、選択したレポートを確認できます。

## 日次レポート：サーバー {#S001_ServerDaily_TagReport}

<Cmdname sid="S001_ServerDaily_TagReport" className="uitext" report />は、1日あたりのサーバーリソース使用量を確認できるレポートです。 開始日、開始時刻、終了時刻を設定して照会できます。

### 概要 {#overview}

<ImgLang img='server_daily_report_01.png' desc='日次サーバーレポート概要'/>

プロジェクト名とホスト数、発生した通知件数を概要を確認できます。 ホスト数と通知件数が前日と比較してどの程度変化したかを確認できます。

- <Cmdname sid="host_count" className="b600" report/>：照会期間中のサーバーの合計数です。
- <Cmdname sid="alert_count" className="b600" report/>：照会期間中に発生したアラートの総件数です。

### サーバーリスト {#server_list}

<ImgLang img='server_daily_report_02.png' desc='日次サーバーレポートサーバーリスト'/>

サーバーごとに毎日の主要メトリクスを確認できます。 CPU最大値(CPU Max)とメモリ最大値(MEM Max)の場合、それらの数値が発生した時間で同時に表示します。

アラートの数は、7日前、1日前、当日で比較できます。

:::note

CPUとメモリの平均値と最大値が90%以上の場合、<span class='loger'>赤</span>で表示されます。

:::

### アラート履歴 {#alert_history}

<ImgLang img='server_daily_report_03.png' desc='アラートリスト'/>

照会期間中に発生したアラート履歴について、発生時間、通知レベル、発生したサーバー名、アラートの原因を確認できます。
各アラートの詳細な情報は<Cmdname sid="side_event" className="uitext" /> &gt; <Cmdname sid="side_eventHistory" className="uitext" />メニューから確認できます。該当プロジェクトにアラート履歴がない場合は、「<Cmdname sid="no_data" className="uitext" report />」メッセージが表示されます。

:::note

通知のレベルに応じて <span class='loger'>Critical</span>は <span class='loger'>赤</span>、<span class='wer'>Warning</span>は <span class='wer'>オレンジ</span>、 <span class='mstop'>Info</span>は <span class='mstop'>グレー</span>で表示されます。

:::

## 週次レポート：サーバー {#S100_ServerWeeklyReport}

<Cmdname sid="S100_ServerWeeklyReport" className="uitext" report/>は、一週間のサーバーのCPUとメモリ使用量を確認できるレポートです。開始日を設定でき、当日の7日前まで設定できます。開始日から7日間のデータを照会します。

### 全体リソースの使用状況一覧 {#total_resource_used_list}

#### <Cmdname sid="_cpu" report/>

<ImgLang img='server_weekly_report_cpu.png' desc='週間全体リソースの使用率一覧CPU'/>

参照期間中、各サーバーのCPU使用率を日別に表示します。表示される項目は、CPU平均値と最大値95%、最大値があり、各数値はパーセンテージで表示されます。

:::note 

**最大95%**

上位5%のデータを除く最大値を意味します。 サーバー再起動などの理由による一時的に高い数値を除外するためです。

:::

#### <Cmdname sid="_mem" report/>

<ImgLang img='server_weekly_report_memory.png' desc='週間全体リソースの使用率一覧メモリー'/>

検索期間中の各サーバーのメモリ使用量を日別ごとに表示します。表示される項目には、メモリ使用量の平均値と最大値があり、各数値はパーセンテージで表示されます。

#### <Cmdname sid="_disk" report/>

<ImgLang img='server_weekly_report_disk.png' desc='週間全体リソースの使用率一覧ディスク'/>

検索期間中に各サーバーにマウントされたファイルシステムごとのメトリクスです。
マウントする場所と合計ディスク数、使用率を検索され、各数値はパーセンテージで表示されます。

:::note

- 数値が90％以上の場合、<span class='loger'>赤</span>で表示されます。 
- グループ名は、`okindName`と表示されます。`okindName`が設定されていない場合は、空白で表示されることがあります。

:::

## 月次レポート：サーバー {#S200_ServerMonthlyReport}

<Cmdname sid="S200_ServerMonthlyReport" className="uitext" report />は、1か月間のサーバーリソースの使用量と情報を確認できるレポートです。開始日を設定でき、当日の30日前まで設定できます。

### 概要 {#overview}

<ImgLang img='server_monthly_report_01.png' desc='月次サーバーレポート概要'/>

1 か月間のサーバーの合計リソース使用量を集計して履歴を表示することができます。

- <Cmdname sid="total_host" className="b600" report/>：照会期間中のサーバーの合計数です。
- <Cmdname sid="total_core" className="b600" report/>：照会期間中に使用されたコアの合計数です。
- <Cmdname sid="total_disk" className="b600" report/>：照会期間中にマウントされたファイルシステムの合計サイズです。
- <Cmdname sid="total_alarms" className="b600" report/>：照会期間中に発生したアラートの総件数です。

### サーバー情報 {#server_info}

<ImgLang img='server_monthly_report_02.png' desc='サーバー情報'/>

照会する期間中に稼働していたサーバーのIPアドレス、OS情報、CPU情報およびメモリ使用量を表示します。

### アラート詳細 {#alarms_list}

<ImgLang img='server_monthly_report_03.png' desc='アラートリスト'/>

検索期間中のすべてのサーバーの通知の合計数を確認でき、サーバーごとに発生したアラートを週単位で表示できます。

### サーバー使用リスト {#server_usage_history}

<ImgLang img='server_monthly_report_04.png' desc='サーバー使用一覧'/>

サーバーごとのCPU平均、最大値、メモリ平均、最大値を日単位で一目で確認できます。

:::note

数値が90％以上の場合、<span class='loger'>赤</span>で表示されます。 

:::
