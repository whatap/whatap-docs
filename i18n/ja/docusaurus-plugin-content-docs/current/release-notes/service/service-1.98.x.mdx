## Service-1.98.0

2023年3月29日

<div class='indentTab'>

### `共通`

#### UX/UI

ユーザーの満足度を向上させるためには、次のようにユーザーインターフェースのデザインを改善します。

- <Status>Change</Status> <UI>利用内訳</UI> > <UI>請求書プレビュー</UI> メニューのUI改善

  - プロジェクト、製品別にグループ化された請求書のプレビュー情報の提供
  - グループ化されたデータの関連商品情報及び、課金の集計量、請求額の合計などの確認が可能

- <Status>Change</Status> <UI>利用履歴</UI> > <UI>プロジェクト有料プラン変更</UI> メニューのUI改善

- <Status>Change</Status> 画面の上部の「メニューパス」を「メニュー名」として表示するように変更

  - 変更前：上位パス > メニュー名
  - 変更後：メニュー名

- <Status>Change</Status> 画面の右上のWhaTap技術文書のリンクボタンの追加し、チャンネルトークで技術文書リンクを削除します。

- <Status>Fixed</Status> サービスロゴの横のブラウザ時間の不一致通知アイコンの位置を修正

- <Status>Fixed</Status> プロジェクトがないグループを選択すると、プロジェクト作成ページに移動する問題を修正

- <Status>Deprecate</Status> <UI>統合ダッシュボード</UI>、<UI>統合メトリクスボード</UI> メニューは、2023年5月1日以降、左サイドメニューから削除、<UI>サイトマップ</UI>の移動に関する説明を表示

#### Flexボード

- <Status>Change</Status> ダッシュボードと一覧を画面に読み込む時に、読み込みインジケーターを適用

- <Status>Feature</Status> Seriesウィジェットタイプのチャート設定で凡例表示を追加

- <Status>Feature</Status> Seriesウィジェットタイプに累積チャートタイプを追加

- <Status>Feature</Status> Seriesウィジェットタイプのチャート設定でチャート色の基準オプションを追加

  - 従来：タグを基準にチャートの色を表現
  - 追加：フィールドを基準にチャートの色を表示するオプション

- <Status>Feature</Status> DB製品群から次のウィジェットテンプレートを追加

  - **DB共通**
    - CPU % (xos) ウィジェット
    - Memory % (xos) ウィジェット

  - **MySQL**
    - スロークエリ

  - **PostgreSQL**
    - スロークエリ
    - PostgreSQL Wait Event件数ウィジェット
    - PostgreSQL Client Ip件数ウィジェット
    - PostgreSQL統計テーブル
    - PostgreSQLアクティブセッション件数
    - PostgreSQLアクティブセッションテーブル

#### MXQL

<p><Status>Fixed</Status> MXQLクエリのフィールド名に特殊文字が含まれている場合、MXQLデータを呼び出し時に発生するバグを修正するための、MXQLクエリ生成ロジックを修正</p>

#### ログ

- <Status>Change</Status> <UI>ログ</UI> メニューからの機能をすばやく理解するためにメニュー名を変更

  - <b>ライブTail</b> → <UI>ライブテール</UI>
  - <b>検索機</b> → <UI>ログトレンド</UI>
  - <b>検索機</b> → <UI>ログ検索</UI>

- <Status>Change</Status> <UI>ログ検索</UI> メニューからクエリ内容を <UI>Timestamp</UI> を基準にソートする機能(<UI>新しい順</UI>/<UI>古い順</UI>) に変更

- <Status>Change</Status> <UI>ログトレンド</UI> メニューから検索キーを最大2つまで設定できるように修正 (検索値は制限なし)

- <Status>Change</Status> <UI>ログトレンド</UI> メニューから参照時間が変更された時に、選択したフィルターの結果がない場合は、上記のフィルターを解除できるよう改善

#### アラート通知

- <Status>Feature</Status> <UI>イベント設定</UI> メニューから <UI>エージェント重複通知</UI> オプションを追加

  > モニタリング中のエージェントで、同じIDを設定したエージェントが2つ以上ある場合、メトリクスを重複して収集することによって発生するアラート通知です。 エージェント名が重複して設定されていないかエージェント設定で関連オプションを確認してください。ブルー/グリーンのようなデプロイ状況でエージェントが重複してアラート通知が発生した場合、この機能を使用することをお勧めします。 同じ名前のエージェントが同時に2つ以上起動した場合、選択した時間中は通知が発生しません。

- <Status>Feature</Status> <UI>イベント受信設定</UI> メニューから <UI>受信タグの未設定通知を受け取る</UI> オプションを追加

  > イベント受信タグが設定されていないアラート通知を受信しない場合は、そのオプションを選択します。

- <Status>Fixed</Status> <UI>イベント受信設定</UI> メニューから登録したモバイル機器の数が表示されないバグを修正

- <Status>Fixed</Status> <UI>イベント設定</UI> メニューからログイベントを追加するときに <UI>イベント発生の一時停止</UI> 項目の説明文を修正

#### 管理

- <Status>Feature</Status> ユーザー権限で <b>ログ参照</b> アクセス権限の追加

- <Status>Feature</Status> <UI>プロジェクト管理</UI> メニューからアラート通知の言語を選択するオプションを追加

- <Status>Fixed</Status> メンバー招待ウィンドウで <b>メンバー管理</b> 権限をオフできないバグを修正
