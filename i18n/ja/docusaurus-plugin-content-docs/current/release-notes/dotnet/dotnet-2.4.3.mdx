2025年06月25日

- <Status priority='1'>New</Status> マトリクス照会で `app_context_stat` カテゴリを追加

  - **app_context_enabled** <span class='type'>Boolean</span> 

    - 既定値`false`

    - `true` に設定すると `app_context_stat` の収集機能がすべて有効になります。

  - **app_context_parser**

    - `default`, `prefix`, `match` のいずれかを使用

    - `default`: デフォルトパーサー（最初のパスセグメントを抽出）

    - `prefix`: 接頭辞マッチングパーサー（最も柔軟）

    - `match`: 完全一致マッチングパーサー（完全一致のみ）

  - **app_context_parser_reset** <span class='type'>Number</span> 

    - 既定値`0`

    - パーサーの内部キャッシュと設定を強制的にリセットするカウンター

  - **app_context_path_depth** <span class='type'>Number</span> 

    - 既定値`1`

    - URLから抽出するパスセグメントの深さを指定（defaultパーサーでのみ使用）

  - **app_context_path_set** <span class='type'>String</span> 

    - モニタリング対象のパスとその名前をマッピングして設定します。複数設定する場合は`,`（カンマ）で区切ります。

    - 例: `name1@url1,name2@url2,url3,name4@url4`

- <Status priority='1'>New</Status> URLを正規化した場合、元のURLをstepにも表示するオプションを追加

  - **step_origin_url_enabled** <span class='type'>Boolean</span> 

    - 既定値`false`

- <Status priority='0'>Fixed</Status> URLを正規化した場合、レコードサマリーに元のURLが表示されなかった現象を修正
