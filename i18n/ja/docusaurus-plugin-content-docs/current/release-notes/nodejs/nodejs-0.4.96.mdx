2024年05月30日

- <Status>Feature</Status>サーバでクライアントにデータを送信する際、データ収集のためのwebsocketパッケージモニタリングを追加

- <Status>Feature</Status>エージェント設定オプションを追加

  - **ignore_nextjs_build_file_enabled** <span class='type'>Boolean</span>

    既定値 `true`

    *Next.js*ビルドファイルに対するモニタリング可否を設定します。 このオプション値を`true`に設定すると、*Next.js*ビルドファイルをモニタリングしません。

  - **ignore_nextjs_build_file_path** <span class='type'>String</span>

    既定値`/next/`

    モニタリングで除外する*Next.js*ビルドファイルの開始パスを指定できます。 `ignore_nextjs_build_file_enabled`値が`true`の場合に適用されます。 コンマ(,)を区切り文字として複数のパスを設定できます。

    ```ini title="whatap.conf"
    ignore_nextjs_build_file_path=/next/abc/,/next/def/
    ```

- <Status>Changed</Status> `trace_sampling_enabled`オプションのデフォルト値を`true`に変更

- <Status>Fixed</Status> HTTPモニタリングでsocketエラーに関する統計データの収集を無視するように修正

- <Status>Fixed</Status>トランザクションの結果が<b>Lost Connection</b>場合、URLが表示されない問題を修正
