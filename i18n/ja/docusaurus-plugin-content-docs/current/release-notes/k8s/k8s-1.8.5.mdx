2025年02月17日

- <Status priority='1'>New</Status> 追加機能により、Kubernetesカスタムリソース関連メトリクス収集に対応

  - 基本機能ではなく、追加機能の適用を通じてwhatap-master-agentのコンテナに`collect_custom_resources` オプションを追加

    **collect_custom_resources** <span class='type'>bool</span>

    既定値`false`

    `true`設定する時に<Cmdname sid="side_metricsSearch" className="uitext" />メニューで`kube_cr`及び`kube_crd`カテゴリの確認が可能です。 収集されたデータは<Cmdname sid="side_objectManifest" className="uitext" />及び**タグカウント**で確認できます。 

    :::note 

    - `kube_crd`: CustomResourceDefinition(CRD)メトリクスを収集

    - `kube_cr`: CRDで定義されたCustom Resource(CR)メトリクスを収集

    :::

- <Status priority='1'>New</Status> `container_gpu`カテゴリに`DCGM_FI_DEV_POWER_USAGE`データを追加してGPUモニタリングを改善

- <Status priority='1'>New</Status>ログに`#whatapSysLog`カテゴリを追加してノードエージェントコンテナを実行する時にユーザー環境ランタイム情報を伝達

- <Status priority='0'>Fixed</Status> `kube_pod_volume`関連バグを修正

  - クラスタープロジェクト、ネームスペースを問わず全体データが収集されるバグを修正

  - 同期の問題により一部のデータが欠落している現象を修正

- <Status priority='0'>Fixed</Status>コンテナのイメージ名が正確にパーシングされない現象を修正

:::info

Agent download: [kube_mon_1.8.5.tar](https://repo.whatap.io/docker/kube_mon_1.8.5.tar)

:::
