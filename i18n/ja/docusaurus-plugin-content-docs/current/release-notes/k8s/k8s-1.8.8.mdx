> **Java Agent v2.2.61 を含む**

2025年9月4日

- <Status priority='1'>New</Status> Kubernetes 環境で APM をインストールする際のコンテナマップにアプリケーション情報が表示されるよう改善  

    コンテナマップにアプリケーション情報が表示されない場合、APM がインストールされたコンテナマニフェスト (*YAML*) の環境変数に、コンテナ内エージェントの絶対パスを指定するオプションを追加します。  

    ##### 言語別オプション

    - **Java**

        `WHATAP_JAVA_AGENT_PATH`

        :::note
        Java APM を使用していても、最小限のイメージを使用しておりコンテナ内で Java コマンドを実行できない場合があります。その場合、`WHATAP_EXECUTABLE_JAVA_PATH` キーを使用してコンテナ環境変数に Java 実行ファイルのフルパスを登録してください。
        :::

        ```yaml title="Java example"
        apiVersion: apps/v1
        kind: Deployment
        metadata:
          name: my-java-app
        spec:
          replicas: 1
          selector:
            matchLabels:
              app: my-java-app
          template:
            metadata:
              labels:
                app: my-java-app
            spec:
              containers:
                - name: my-java-container
                  image: your-java-app-image:latest
                  env:
                    # Java APM jar のパス
                    - name: WHATAP_JAVA_AGENT_PATH
                      value: /whatap/whatap.agent-2.2.35.jar
                    # 最小イメージで java 実行ファイルのフルパスを指定
                    - name: WHATAP_EXECUTABLE_JAVA_PATH
                      value: /layers/paketo-buildpacks_adoptium/jre/bin/java
        ```

    - **Python**

        `WHATAP_PYTHON_AGENT_PATH`

    - **PHP**

        `WHATAP_PHP_AGENT_PATH`

    - **Go**

        `WHATAP_GO_AGENT_PATH`

    - **Node.js**

        `WHATAP_NODEJS_AGENT_PATH`

    ##### コンテナ設定オプション

    * `WHATAP_CONTAINER_CONF_PATH` <span class='type'>String</span>  

        コンテナ情報を記録するための `container.conf` ファイルの生成場所を指定します。  
        このファイルにはコンテナID値が保存されます。
