# http callエラー通知で無視したstatus code

2022-03-24

### 新機能

#### Http Callエラー通知機能の追加

Http呼び出し時にエラーが発生すると通知を受けられるように機能を追加しました。

`exception_alert_enabled`オプションを`true`に設定して機能の活性化ができ、`exception_alert_stack_size`オプションで指定した値に合わせてエラースタックを通知メッセージとして受け取ることができます。

一度通知が発生すると`exception_alert_silent_time`オプションで指定した期間中にはエラー通知を非活性化することができます。

```ini title='whatap.conf'
# デフォルト値
exception_alert_enabled=false
exception_alert_buffer_size=5
exception_alert_stack_size=50

# 5分（30000ms）
exception_alert_silent_time=300000
```

### 新機能

#### フェデレーション サービスの状態追跡機能を追加

target IPまたはサービスの接続状態をログモニタリングのliveTailで確認できるように、連携サービス状態の追跡機能を追加しました。

`logsink_rt_enabled`オプションを`true`に設定して機能を活性化でき、#RemoteTraceカテゴリでログを収集します。

-   正常なレスポンスの場合のレベルはOK、エラーの場合のレベルはエラーです。

-   通常のレスポンスの場合、`logsink_rt_ok_interval`設定の間隔でログは記録されます。デフォルト値は30秒です。

-   エラーレスポンスの場合、`logsink_rt_error_interval`設定の間隔でログは記録されます。デフォルト値は5秒です。

```ini title='whatap.conf'
# デフォルト値
logsink_rt_enabled=false

logsink_rt_error_interval=5000
logsink_rt_ok_interval=30000
```

## 2021-12-17

### アップデート

#### weaving設定追加

プラグインを追跡できるようにweaving設定を追加しました。 既存の`weaving_reserved`設定と同じ機能で、`weaving`または`weaving_reserved`設定がすべて動作します。

#### エージェント環境変数の除外値の追加

環境変数の照会機能で特定のキー値を入力する場合、照会項目から除外できるように機能を追加しました。 区切り文字は「,」で大文字と小文字を区分しません。`agent_env_system_enabled`、`agent_env_jvm_enabled`の設定と関連付けて使用することができます。

```ini title='whatap.conf'
agent_env_system_enabled=false
agent_env_jvm_enabled=true

agent_env_ignore_keys=user.timezone,user.name,user.language
```
