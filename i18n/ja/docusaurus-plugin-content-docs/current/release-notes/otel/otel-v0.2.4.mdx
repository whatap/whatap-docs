2025年04月30日

- <Status priority='1'>Feature</Status> HTTP Statusエラー表示

  - HTTP Status codeが 400 以上の場合、エラーとして表記する設定

    - Key: **transaction_status_error_enable** 
    - Type: <span class='type'>bool</span> 
    - Default: `false`
    - 収集周期: 10秒

- <Status priority='1'>Feature</Status> HTTP Status Code統計収集

  - 全体URLのStatus Code別件数収集のための設定

    - Key: **tx_status_meter_enabled** 
    - Type: <span class='type'>bool</span> 
    - Default: `true`
    - 収集周期: 10秒

  - URL別Status Code件数収集のための設定 

    - Key: **stat_txstatus_enabled** 
    - Type: <span class='type'>bool</span> 
    - Default: `true`
    - 収集周期: 5分

- <Status priority='1'>Feature</Status> 分散追跡用HTTP callの仮想span情報を設定して分析追跡

  - HTTP call span情報がない場合、仮想spanを活性化

    - Key: **otel_virtual_httpc_enabled** 
    - Type: <span class='type'>bool</span> 
    - Default: `false`

  - HTTP call span情報がない場合、名前を設定 

    - Key: **otel_virtual_httpc_name** 
    - Type: <span class='type'>string</span> 
    - Default: `external http call`

- <Status priority='1'>Feature</Status> Gateway情報収集

  - ゲートウェイ情報収集の活性化

    - Key: **otel_gateway_enabled** 
    - Type: <span class='type'>bool</span> 
    - Default: `false`
    - 収集項目及び周期  
      - 全体URL単位トランザクション·ゲートウェイ·API呼び出しの経過時間の収集周期:10秒  
      - URL·Status別トランザクション·ゲートウェイ·API呼び出しの経過時間の収集周期: 5分

  - 指定された属性(attribute)の値でゲートウェイ経過時間を設定

    - Key: **otel_attr_tx_gw_time_keys** 
    - Type: <span class='type'>string</span> 
    - Default: `duration.kong`

  - 指定された属性(attribute)の値でAPIの呼び出し経過時間を設定 

    - Key: **otel_attr_tx_api_time_keys** 
    - Type: <span class='type'>string</span> 
    - Default: `duration.proxy`

  - 指定された属性(attribute)の値で応答時間を設定  

    - Key: **otel_replace_tx_resp_time_enabled** 
    - Type: <span class='type'>bool</span> 
    - Default: `false`  
    - Key: **otel_attr_replace_tx_resp_time_keys** 
    - Type: <span class='type'>string</span> 
    - Default: `duration.kong`

- <Status priority='0'>Fixed</Status> HTTP Method無視設定エラーを修正

  - `PATCH`、`OPTIONS`、`HEAD`、`TRACE`メソッドはトランザクションとして収集しない
