## 様々なプラットフォームに対応する統合モニタリング

WhaTapモニタリングサービスの基本目的はリソース（人的、物的）の共有です。 既存のモニタリング製品は、モニタリング対象別に別途の収集サーバー、ウェブ画面を構築する必要があります。 しかし、WhaTapのモニタリングサービスは、収集サーバーは共有するものの、各サービスの独立性は保証する構造です。 統合管理を行うには、モニタリングサーバーは柔軟な拡張構造でありながらマルチテナント機能を提供する必要があります。

リアルタイムでビジネス状況を視覚化することで、迅速に問題を把握できます。 トランザクションをリアルタイムでトレースすることで、再現なしで問題を追跡できます。 WhaTapモニタリングは、分散環境のITサービス性能をリアルタイムで視覚化し、問題認識と対応に対する顧客の時間を節約し、先制的な対応を可能にします。

## モニタリング範囲

アプリケーション、サーバー、データベース、Kubernetes、クラウド、ログ、URLモニタリングを通じてシステムモニタリングのほぼすべての部分をカバーします。

### アプリケーションモニタリング

{@video: https://www.whatap.io/video/main-product/monitoring_step_0.mp4}

[Java](java/introduction)、[PHP](php/introduction)、[Node.js](nodejs/introduction)、[Python](python/introduction)、[.NET](dotnet/introduction)、[Go](golang/introduction)などのアプリケーションのモニタリングに対応します。現在進行中のトランザクションの実行状況をリアルタイムで確認できます。 現在進行中のトランザクションの実行状況をリアルタイムで確認できます。 

クラウド、ハイブリッド、オンプレミスなど複雑で分散したビジネス環境のアプリケーションに対し、リアルタイムで性能低下問題をモニタリングし、再現なしで最も迅速に追跡して原因を見つけ出すことができます。 すべてのアプリケーションの問題は、進行中のトランザクションの増加によって発生します。 増加した瞬間に実行中のSQL、HTTP Call、Methodをすぐに確認できれば、問題を最も早く認知して分析することができます。 すべてのトランザクションの実行履歴を収集して分析できます。

### サーバーモニタリング

{@video: https://whatap-landing-resource.s3.ap-northeast-2.amazonaws.com/prd/application/monitoring_step_1.mp4}

[サーバーモニタリング](server/introduction)は、10,000台以上のサーバーでも、すべてのプロセスのCPU、メモリ、IO使用量などを秒単位で収集し、瞬間的に発生する過負荷および性能低下をリアルタイムで分析して通知することができます。 クラウド環境のアプリケーションで発生しうる脆弱性に対する自動検知と通知機能を提供し、毎月・毎週のように変更されるシステム規模を簡単に予測・管理するのに役立ちます。

### Kubernetesモニタリング

{@video: https://whatap-landing-resource.s3.ap-northeast-2.amazonaws.com/prd/k8s/01.webm}

[Kubernetesモニタリング](kubernetes/introduction)は、コンテナからアプリケーションまでの統合分析を提供します。 Pod、Worker Node、アプリケーションをモニタリングしてWorker Node別またはPod別のトランザクション性能の影響を分析します。 MSA環境のコンテナレベル、サービスレベル、トランザクションレベルの3段階の相関関係を追跡し、Pod、コンテナ、サービス間の呼び出し関係を視覚的に確認することができます。 オブザーバビリティのためのメトリクス-トレース-ログ情報を統合して提示します。 これにより、問題発生時の迅速な原因分析および措置が可能になります。

EKS、AKS、GKE、OKE、Openshift、Cocktail、Accordion、PaaS-TAを含む主要プラットフォームに対応します。

:::note

WhaTap Labsは、CNCF（Cloud Native Computing Foundation）とKCSP（Kubernetes Certified Service Provider）、クラウドサービスセキュリティ認証（CSAP）を取得しました。

:::

### データベースモニタリング

{@video: https://whatap-landing-resource.s3.ap-northeast-2.amazonaws.com/prd/db/DB_01_instanceMonitoring.mp4}

データベースモニタリングでは、データベースのアクティブセッション、実行中SQL、各種指標の統計および詳細情報をリアルタイムでモニタリングし、チューニング対象を選別して解決可否を確認することができます。 アプリケーション、サーバー、データベースモニタリングの連携ビューを通じてサービスに異常が発生した場合、直ちにDB状態を確認し、原因分析を行うことができます。 効率的なパフォーマンスデータ収集構造により、多数のデータベースモニタリング時にも迅速にデータを収集し、統合されたビューを提供することで分析が容易になります。 クラウド環境の様々なDBのパフォーマンスやトラブル状況を、面倒なインストールなしにひと目で確認でき、リアルタイムでモニタリングします。

### URLモニタリング

{@video: https://whatap-landing-resource.s3.ap-northeast-2.amazonaws.com/prd/application/monitoring_step_4.mp4}

[URLモニタリング](url/url-intro)は、実際のユーザーの立場からサイト利用に問題があるかを周期的に確認する管理者のモニタリング業務を肩代わりします。 HTTPステータスコードを通じてサイト利用の問題をリアルタイムでモニタリングし、地域によって所要される接続時間を確認することができます。 北米、ヨーロッパ、アジアなど世界6か国の主要都市でサービスの状態をチェックします。

### ブラウザーモニタリング

![ブラウザー案内の臨時画像](/img/browser-start.png)

[ブラウザーモニタリング](browser)は、ブラウザーアプリケーションのページ読み込み時間、サーバーレスポンス時間、リソース読み込み時間、ブラウザーエラーなど多様な性能指標を収集して分析します。 また、ブラウザーアプリケーションのボトルネックや性能上の問題を把握し、最適化することで、すべてのユーザーに対して一貫した最高の性能を提供できるようサポートします。

### クラウドモニタリング

[クラウドモニタリング](ref-cloud/cloud)は、クラウドプロバイダのAPIに接続し、すべての指標とクラウドイベントをリアルタイムで収集します。 収集したデータをダッシュボードを通じてリアルタイムで確認し、トラブル原因に対する直接分析ができます。

提供する製品は以下の通りです。 リンクをクリックして技術文書を確認できます。

-   [Amazon CloudWatch](amazon-cloudwatch/install-agent)
-   [Amazon ECS](amazon-ecs/introduction)
-   [Azure Monitor](azure/install-agent)
-   [Naver Cloud Monitoring](ncloud/install-agent)
-   [Oracle Cloud Monitor](oracle-cloud/install-agent)

### Logモニタリング

[Logモニタリング](log/introduction)では、クラウドを含む複合システムで発生する様々なログを簡単に統合管理・活用できます。 数百、数千台のサーバーで発生するログを送信して統合確認が可能です。 画面を通じて流れるログを照会することができます。 統計目的でログを分類し、特定の単位で発生するログ件数の推移を確認できます。 トラブル発生および解消時点と密接な関連を持つログデータを探索でき、様々な条件で大量のログを検索できます。

## WhaTapモニタリングの主な特徴

### リアルタイムモニタリングダッシュボード

![](/img/blog_35_3.png)

サーバーの様々な情報や現状をチャート型のダッシュボードを通じてリアルタイムで迅速に認識できます。 各指標の過ぎ去った時間のデータを確認して分析することができます。 進行中のトランザクションにより、リアルタイムで発生するボトルネックを確認し、問題を解決するのに役立ちます。

### 様々な通知発行チャネルに対応

<ImgLang img='overview-notification.png'/>

イベント条件を設定し、メール、SMS、メッセンジャー、アプリプッシュなど様々な経路で通知を提供します。 エージェントと収集サーバーの通信中断およびアプリケーションリソース使用率、アクティブトランザクション、エラー、レスポンス遅延、レスポンス分布異常に関する通知設定ができます。 また、マシンラーニングベースのAI通知設定を通じてレスポンス分布パターンを検知し、通知機能を提供します。

:::note

アラート通知に関連してモニタリングプラットフォーム別に対応されるイベントの種類を確認するには、[次の文書](support-env#support-notice)を参照してください。

:::

### 統合モニタリングが可能なマルチリージョンに対応

![](https://www.whatap.io/img/platform/solution_06.jpg)

追加のリージョンが発生するたびに収集サーバーのみをインストールし、1つのモニタリング画面で複数のリージョンのサーバーを統合モリタリングします。 WhaTapモニタリングの場合、IDCセンターの追加構築やクラウドプラットフォームの追加構築などをリージョン構築だけで収集サーバーを拡張できます。

### 収集サーバーの安定性と拡張性の保証

![](https://www.whatap.io/img/platform/solution_07.jpg)

統合モニタリングのための最も基本的なものは、サーバーの拡張性です。 増加する管理（モニタリング）対象サーバーを円滑に受け入れるために、モニタリングサーバーは内部拡張性を持たなければなりません。 拡張性を保証するために、WhaTapモニタリングサーバーは、細分化されたマイクロアプリケーションで構成されており、結合度の低い分散環境となっています。 また、データの安定性を確保するために、二重化およびデータ復旧機能を提供します。

### 協業機能の提供、マルチテナント

![](https://www.whatap.io/img/platform/solution_08.jpg)

WhaTapモニタリングサービスを通じて開発チームは独立的にモニタリングを利用し、運用チームはこれを統合モニタリングすることができます。 1つのモニタリング製品を利用しますが、チーム別・業務別の独立性は保証し、これを中央管理者が統合して管制することができます。 中央運営管理者は、開発チームのプロジェクトの権限を直接管理せず、各開発チームプロジェクトの所有者が権限を管理する構造です。
