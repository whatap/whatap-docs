## Oracle Proモニタリングの特長

WhaTapのOracle Proモニタリング製品は、従来のクエリ(Query) ベースのデータ収集方式を一段階上回る革新を提示します。 既存のクエリベースの方法は、データベースサーバーにかなりの負荷を与え、これはシステム性能の低下につながる可能性があります。 一方、Direct Memory Access(DMA) 技術を活用したWhaTapのアプローチは、サーバーのメモリに直接アクセスして必要なデータを抽出することによって、データベースサーバーに負荷をほとんど、または全く発生させません。 これは、特に高性能を必要とするデータベース環境において大きな利点として機能します。

また、クエリベースの収集方式はデータ更新サイクルが遅く、特定のクエリ実行時点にのみデータを収集できます。 これに比べ、Oracle Proモニタリングはメモリへのアクセスを通じてデータをほぼリアルタイムで収集でき、少なくとも1/20秒の間隔でセッション及びSQL性能データを収集できます。 これにより、データベース管理者はシステムの性能をより詳細にモニタリングし、問題が発生したときにすぐに対応できるようになります。

Oracle Proモニタリング製品のもう一つの重要な特徴は、システム障害やHang状況でもデータ収集が中断されないことです。 従来のクエリベースの方法では、これらのシナリオではデータの収集が困難になる可能性があります。 Oracle Proを使用することで、データベース管理者はいかなる状況でもデータの連続性を保証され、これによってシステムの安定性と可用性を最大化できます。

結局、WhaTapのOracle Proモニタリング製品は、データベースモニタリングの効率性と正確性を向上させると同時に、システム負荷を最小限に抑え、ビジネス継続性を保証するために欠かせない技術です。 これは、企業がより速く、より正確で、より信頼性の高い意思決定ができる基盤を提供します。

- **メモリのアクセス方式**

  WhaTapのOracle Proは**Direct Memory Access**(DMA) 技術を通じてDBサーバーメモリに直接アクセスし、データを収集します。 既存のクエリ方式の限界を超えて、サーバーの負荷なしで迅速かつ効率的なデータ収集を提供します。

- **負荷のないモニタリング情報の収集**

  Oracle ProはDBサーバーメモリに直接アクセスしてデータを抽出し、1秒に20回の性能情報を収集してもサーバー負荷をほとんど発生させません。 これは高性能DB環境でも安定的かつ効率的なモニタリングをサポートします。

- **精密なSQL性能データの収集**

  Oracle Proは1秒に20回データを収集し、精密な分析が可能であり、従来のクエリーベース方式の5秒間隔SQL収集よりもはるかに詳細な情報を提供します。 これにより、管理者はリアルタイムで問題を特定し、迅速に対応できます。

- **DB Hang状態でもモニタリング分析が可能**

  DB障害やHang状況でもOracle Proはデータ収集が中断されません。 これにより、データの連続性が保証され、障害のある状況でも分析および対応が可能になり、システムの安定性と可用性が最大化されます。
