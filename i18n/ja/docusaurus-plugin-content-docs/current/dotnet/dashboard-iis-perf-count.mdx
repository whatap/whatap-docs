**Internet Information Services**(IIS)の主要指標をリアルタイムでモニタリングしてサーバーの性能を最適化し、潜在的な問題を迅速に識別することで、システムの安定性を向上できます。 また、トラフィックの増加またはアプリケーションの変更による影響を事前に予測して措置できます。

## 性能モニタリング

ウェブサーバーの性能と安定性を評価するために使用される項目です。 各アイテムは、サーバーの動作状態と性能を追跡し、潜在的な問題を迅速に識別するのに役立ちます。 運用担当者は、これらの指標をモニタリングして、サーバーの性能を最適化し、問題を解決するためのデータとして参照できます。

- ***Pipeline Instance Count***

  現在活性化されているパイプラインインスタンスの数です。 同時に処理中のHTTPリクエストの数を示し、もしこの指標値が上昇した場合、サーバーが過負荷の状態になっている可能性があります。

- ***Worker Process Restarts***

  ワーカープロセス(Worker Process)が異常で終了し、再起動した回数です。 この指標値が予期せず増加した場合は、できるだけ早めに原因を調査すべきです。 アプリケーションの安定性と信頼性を評価するために使用されます。

- ***Requests Rejected***

  サーバーで処理が拒否されたリクエストの数です。 503 HTTPステータスコードを返したリクエストの数を示します。 サーバーの負荷が多すぎる、設定された制限を超えるなどで発生する場合があります。

- ***Requests Queued***

  現在待機中のリクエストの数です。 サーバーが現在処理できないリクエストの数であり、この指標値が高いと、サーバーの性能に影響を与える可能性があります。

- ***Current Connections***

  現在活性化されているTCP接続の数です。 サーバーの負荷を確認し、ネットワークリソースの使用を追跡するために参照できます。

- ***Requests/Sec***

  1秒あたりに処理されたHTTPリクエストの数です。 アプリケーションの現在の処理量を測定した値で、他のサーバー作業(ガベージ収集、キャッシュ整理スレッド、外部サーバーツールなど)を除き、特定の範囲内で一定に保たれている必要があります。

- ***Errors Total/Sec***

  HTTPリクエストの実行中に発生したすべてのエラーの数です。 コンパイル中のエラー、前処理中のエラー、実行中のエラーの数を合わせた値です。 アプリケーションまたはサーバーの問題を把握するために参照できます。

- ***ISAPI Extension Requests/sec***

  1秒当たりに処理されたISAPI拡張リクエストの数です。 ユーザー認証、データベースアクセス、コンテンツフィルタリングなどの作業は、すべてISAPI拡張を通じて行うことができます。 この指標により、ウェブサーバーで動的な作業を処理する頻度を把握できます。

## ネットワーク及びデータ転送の追跡

サーバーのネットワーク活動とデータ転送量を追跡するために使用される項目です。 これらの指標は、サーバーのネットワーク性能を評価し、トラフィックパターンを分析するのに役立ちます。 データ転送量とファイル転送速度をモニタリングして、ネットワークトラフィックの変化を追跡し、リソースの使用を最適化できます。

- ***Bytes Sent/sec***

  1秒あたりに転送されたバイト数です。 サーバーのネットワーク活動を追跡し、トラフィックパターンを分析するために参照できます。

- ***Bytes Received/sec***

  1秒あたりの受信バイト数を示します。 クライアントからサーバーへのデータ転送量を追跡するために参照できます。

- ***Files Sent/sec***

  転送された1秒あたりのファイル数を示します。 ファイルサービスの活動を追跡し、ディスクI/O使用量を確認する時に参照できます。

- ***Files Received/sec***

  1秒あたりの受信ファイル数を示します。 クライアントからのファイル転送の活動を追跡するために参照できます。
