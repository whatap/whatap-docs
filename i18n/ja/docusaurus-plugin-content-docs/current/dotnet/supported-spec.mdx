## .NETエージェント動作環境

エージェントをインストールする前に、.NETの対応環境を確認してください。

- **対応バージョン**

  - .NET Framework 4.5.2 ~ 4.8 (4.61 以降を推奨)
  - .NET 6、7、8

- **OS**: Windows Server 2008 R2以降

- **アプリケーションサーバー**：IIS 8.0 以降

- **ライブラリ**

  - WebClient/WebRequest

  - HttpClient/HttpClientHandler

  - ASP.NET MVC 5 (IIS)

  - ASP.NET Web API 2 (IIS)

  - ADO.NET (Database)

- **データベース**：MS SQL、Oracle、MySQLなど、ADO.NETに対応するすべてのデータベースおよびRadis

## サーバ対応環境を分析する方法

### 分析ツールダウンロード{#download-tools}

エージェントをインストールするサーバーの対応環境が不明な場合は、次のリンクから分析ツールをダウンロードしてください。

- [WhaTapDotnetScan.exe](https://repo.whatap.io/windows/WhaTapDotnetScan.exe)

- [WhaTapDotnetScan.zip](https://repo.whatap.io/windows/WhaTapDotnetScan.zip)

### 分析ツールの使用{#use-tools}

1. ダウンロードした分析ツールを実行してください。

   ![WhaTap System information scanner](/img/apm-dotnet-analystic-env-01.png)

2. ![数字1](/img/number-01.png) **Caption**項目で、ウィンドウサーバーのバージョンが2008 R2以降であることを確認します。

3. ![数字2](/img/number-02.png) **IIS Version**, **.NET Framework Version**項目で**IIS 8.0**バージョン以降、**.NET Framework 4.5**バージョン以降であることを確認します。

### ユーザーのアプリケーションのバージョンを確認

![IIS Manager](/img/apm-dotnet-application-pool.png)

**IIS Manager**から**Application Pools**をクリックします。**Application Pools**の一覧で使用するアプリケーションに対応するプール(Pool)の**.NET**が4.0バージョンであることを確認してください。
