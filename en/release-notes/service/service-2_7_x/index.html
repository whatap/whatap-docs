Service 2.7.0​

July 24, 2024

Release of new features​

It provides a guide to the new features released in Service 2.7.0.

Database​

  • New Oracle V2 services now launched!!

    • The UIs and features of Dashboard have been greatly improved.

    • You can see the database performance metrics in Dashboard.

    • Added the Log menu that can search for database logs and their trend and configure the log notifications.

    Note

    • For more information about new features in Oracle V2, see the following.

    • The minimum version of the DBX agent to use the Oracle V2 is 1.9.0. For more information about the agent update, see the following.

  • New Oracle Pro services now launched!!

    This approach uses the Direct Memory Access (DMA) technique to directly access the server's memory and extract the required data. This method minimizes or does not cause load on the database server. By accessing memory directly, data can be collected almost in real time, allowing to monitor the system performance more precisely and to immediately respond upon any problem.

    Note

    For more information about Oracle Pro, see the following.

Kubernetes​

  • New Added the Service List menu that provides comprehensive status and information on all services running in the Kubernetes cluster.

    Note

    • This feature can be seen in the Workload > Service List path. You can understand the network configuration within the cluster by detailing the service configuration and current status.

    • For more information about the Service List, see the following.

    • This function is supported in Kubernetes master agent 1.7.8 or later.

  • New Added the Node Disk List menu for disk monitoring from the node perspective.

    Note

    • This feature can be seen in the Cluster > Node Disk List path.

    • For more information about the Node Disk List, see the following.

    • This function is supported in Kubernetes master agent 1.7.8 or later.

Common​

  • Feature Added the feature that automatically focuses on the project search bar when you select All projects from the side menu to expand the project list.

  • Feature Added the feature to search for agents by IP address through the integrated search in the side menu.

  • Changed Changed the time text in the Report window of the Reservation of Report Email menu to the text that includes the time zone.

    e.g. 0406 → (UTC+9) 0406

  • Changed Changed to expose the organization icon in the organization list on the side menu.

  • Changed In Manage Accounts or Alert > Notifications, changed the success message displayed when testing mobile device notifications.

    Successfully connected to the mobile device. → A test message was requested to be sent to a mobile device.

  • Fixed In the Select Column popover of Alert > Event History, fixed the Initialize(default) button action bug.

  • Fixed In Alert > Notifications, disabled the feature to select the Test and Delete buttons for mobile devices of the users other than your account.

  • Fixed In Management > Project Member, fixed the issue where the name/organization tag colors are not reflected when changing themes.

  • Deprecated In the Anomaly detection tab of Alert > Event Configuration, removed the object merge selection UI when adding or modifying events.

  • Deprecated Integrated Dashboard / Integrated Metrics Dashboard

Application​

  • Feature Added the CSV file download feature for the search content in the following menu paths.

    • Instance Performance Management > Library Version
    • Instance Performance Management > Runtime ENV. Variable

  • Feature In Management > Agent CONF., added the feature to download the agent configuration file (whatap.conf) of the agent selected in the path.

  • Feature In the table of Instance Performance Management > Thread List/Dump, added the column to display the increased CPU time.

    Note

    If you select on the upper right of the screen, you can check the changed values ​​in the Increment column.

  • Changed In the Table view and Tree view tabs on the Trace analysis window, changed the port not to be displayed if there is no transaction host information.

  • Changed In the Multi-transaction tab on the Trace analysis window, changed not to expose transactions when there is no host information.

  • Changed In the menus under Instance Performance Management, modified to retain agent selection when moving to another submenu with the agent selected.

  • Changed In Instance Performance Management > Heap Dump, changed the guidance message regarding the dump heap to be displayed always.

  • Changed Changed the maximum number of cases (100,000 → 10,000) that can be downloaded using the CSV download feature in Statistics > Caller.

  • Changed Increased the width of the search bar under the Statistics menu.

  • Changed In Sitemap > Topology > Netstat Topology, changed the example image to English.

  • Changed In Alert > Event Configuration, changed the conditions of the TOO_MANY_ACTX event template.

    • Previous: endsWith(okindName, 'ote-virtual-0') && startsWith(onodeName, 'node-0') && type == 'java'

    • Change: active_tx_count > 100

  • Fixed In the Top 30 list of Dashboard > Transaction Map, fixed the delay issue when using the keyboard for movement.

  • Fixed In Dashboard > Transaction Map, fixed the issue where when applying a filter to a past time point and then changing to a real-time query, the filter for the previous query is applied.

  • Fixed In Dashboard > Active Transaction, fixed the problem where scrolling in the table does not work.

  • Fixed In Management > Agent Installation for the Node.js products, modified the text related to the agent downloading.

  • Fixed In Analysis > Metrics Chart, modified the display of the following widget notations.

    • Heap total(heap_tot) → Heap total use(heap_tot)

    • Heap max usage(heap_max) → Heap max size(heap_max)

  • Deprecated In Alert > Event Configuration, removed the duplicate WARNING_OLD_GENARATION_GC event templates.

  • Deprecated In Alert > Event Configuration, removed the unnecessary WARNING_TPS_DELTA event template.

  • Deprecated Removed Application dashboard of the old version.

  • Deprecated In Dashboard > Application Dashboard, removed the guide message window.

Server​

  • Changed In Server List > Server List, changed the display method of the Private IP column in the table.

    • Modified the display method of each IP with a line break to display IPS in one line.

    • Added ellipsis (…) depending on the column width.

    • Changed to display 127.0.0.1 last and expose the private IP first.

  • Fixed Fixed the issue where the Memory swap used chart does not appear, if search is performed for more than 3 hours in Memory usage Detail of the Server List > Server Detail menu.

  • Fixed In Alert > Event History, when the Search button is selected, modified to display the loading UI inside the table only when actual search logic is performed.

Kubernetes​

  • Feature In the summary view of Dashboard > Container Map, added the Container summary analysis feature under the Target information tab.

    Note

    This feature is available in Korean only.

  • Feature In the summary view of Dashboard > Container Map, added the service information under the Target information and Resource relations tabs.

  • Feature In the Log > Search Log tab (summary or detailed view) of the Dashboard > Container Map menu, added the link to go to the Live Tail menu.

  • Feature In Container > Container List, added the button to go to the list menu for the following columns.

    • Deployment → Deployment List
    • Pod → Pod List
    • Image, ImageId → Container Images
    • ApplicationName → Container Application List

    As you navigate through each menu, you can see filtered query results based on the data for the selected item.

  • Feature In Workload > Container Application List, added the button to go to the list menu for the following columns.

    • CotainerId → Container List
    • ContainerName → Container List
    • Pod → Pod List
    • Node → Node List

    As you navigate through each menu, you can see filtered query results based on the data for the selected item.

  • Feature In Workload > Container Application List, added the Image column into the Resource group.

  • Feature In the event list table of Cluster > Kubernetes Event, added the Time the event first occurred and Number of events columns.

  • Feature In Cluster > Node Detail, added the Pod list.

  • Feature In the Record summary tab of the Trace analysis window, added the feature to display information for Container Id and Pod Name and go to each container and Pod list pages when clicked for each.

  • Feature Added the agent status display feature at the top of the screen so that users can immediately recognize when the Kubernetes agent is disabled.

  • Changed In the container list of Cluster > Node Detail, improved the UI design for the container's detail button.

  • Fixed In the summary or detailed view of Dashboard > Container Map, modified not to display Application log for the blocks where the Node.js application agent has been installed in the Log > Search Log tab. (Log monitoring for Node.js products is not supported)

  • Fixed When the Dashboard > Container Map menu appears for the first time and go to the Pod or container view, fixed the problem where wrong data is output.

  • Fixed In Cluster > Node Detail, fixed the issue where the metric title at the top does not appear.
