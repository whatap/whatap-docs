Support Environment

.NET agent support environment​

Before installing the agent, check the support environment for .NET framework applications.

  • Supported version: .NET Framework 4.5 (version 4.61 or later recommended)

  • OS: Windows Server 2012 or later

  • Application Server: IIS 8.0 or later

  • Library

    • WebClient/WebRequest

    • HttpClient/HttpClientHandler

    • ASP.NET MVC 5 (IIS)

    • ASP.NET Web API 2 (IIS)

    • ASP.NET Web Forms (IIS)

    • ADO.NET (Database)

  • Database: All databases and Radis that support ADO.NET such as MS SQL, Oracle, MySQL, etc.

How to analyze the server support environment​

Downloading the analysis tools​

If you are not familiar with the supported environment of the server to install the agent on, download the analysis tools through the following links.

  • https://repo.whatap.io/windows/WhaTapDotnetScan.exe

  • https://repo.whatap.io/windows/WhaTapDotnetScan.zip

Using the analysis tools​

  1. Run the downloaded analysis tools.

  2. Click Analysis (Scan).

  3. In the area, check whether the Windows server version is 2012 or later.

  4. In the area, check whether the versions are IIS 8.0 or later, and .NET Framework 4.5 or later.

  5. Check that the .NET Framework version appears as v4.0 for each site installed in IIS.

  • .NET Framework supports up to the version 4.8.1.

  • Core, .NET 5, or later is not supported. The .NET Core versions are under development.

Checking the user's application version​

Even if the server environment satisfies the supported version, the .NET monitoring is not supported when the user's application has been developed as a .NET Core version instead of .NET Framework.

The user's application version can be seen in the IIS Manager as follows.

In IIS Manager, click Application Pools. In the Application Pools list, check that .NET of the pool for your available application is the version 4.0.

Mobile app​

Unfortunately, the WhatTap mobile app is not supported for the time being. However, the users who have installed the existing apps on their mobile devices can use the app service, but new updates are not supported. We will try to provide new, more stable versions of mobile apps as soon as possible.
