와탭 24년 안내

2024년 11월 19일
TW J
테크니컬 라이터

와탭의 2024년 1월부터 10월까지 진행된 서비스 신규 기능 및 주목할 만한 변경 사항 개요를 안내합니다. 와탭이 새롭게 제공하는 기능들을 한 눈에 살펴보세요.

공통
  • SSO 계정 연동 기능이 추가되었습니다.
  • 이벤트 설정 New 메뉴를 새롭게 제공합니다.
  • Application, Kubernetes, Database 상품군에서 메트릭스 차트 New 메뉴를 새롭게 제공합니다.
Application
  • 공통 메뉴가 개선되었습니다.
  • 트랜잭션 맵이 고도화되었습니다.
  • Go 상품 베타 서비스를 종료하고 정식 서비스를 제공합니다.
  • 애플리케이션 대시보드에 기본 프리셋 변경 기능이 추가되었습니다.
  • Java 상품이 데이터베이스 연계 분석을 제공합니다.
Server
  • Unix AIX, Oracle Solaris, HP-UX 설치 패키지를 제공합니다.
  • 서버 인벤토리 메뉴를 새롭게 제공합니다.
Kubernetes
  • 컨트롤 플레인 모니터링을 새롭게 제공합니다. 컨트롤 플레인 컴포넌트 중 kube-apiserver 모니터링을 위해 kube-apiserver 대시보드 및 kube-apiserver 메트릭스 조회 기능이 추가되었습니다.
  • 기존의 Pod 초기화 성능 기능 및 마스터 메타 정보 기능을 개편해 Pod 시작 분석과 오브젝트 매니페스트 기능을 제공합니다.
  • Deployment 기준으로 관련 정보를 조회할 수 있는 Deployment 목록, 마스터 및 노드 에이전트 설치 현황을 확인할 수 있는 쿠버네티스 에이전트 목록 기능이 추가되었습니다.
  • 마스터 및 노드 에이전트 설치 시 Helm 차트 설치를 지원합니다.
  • 에이전트 설치 화면이 보다 손쉬운 옵션 선택 방식으로 개편되었습니다.
  • Pending Pod 현황 메뉴가 새롭게 개편되었습니다.
  • 컨테이너 맵 메뉴에서 리소스 관계도를 제공합니다.
  • 노드 맵, 노드 타임라인, 노드 디스크 목록과 같은 신규 메뉴가 추가되었으며, 기존 노드 목록 메뉴가 개선되었습니다.
  • 컨테이너 요약 분석, Pod 요약 분석, 노드 요약 분석 기능을 새롭게 제공합니다.
  • Service 목록 메뉴를 새롭게 제공합니다.
  • Ingress 목록 메뉴를 새롭게 제공합니다.
Database
  • MongoDB와 Redis, Oracle, Altibase, Cubrid 상품에서 차트 및 수집 기능을 강화한 새로운 V2 서비스를 제공합니다.
  • Oracla Pro 상품 정식 서비스를 제공합니다.
  • 멀티 인스턴스 모니터링 메뉴의 개선이 이루어졌습니다.
  • PostgreSQL, MySQL, Oracle V2 및 Oracla Pro 상품이 애플리케이션 연계 분석을 제공합니다.
노트

  • 현재 데이터베이스 V2 지원 플랫폼은 다음과 같습니다.

    PostgreSQL, MySQL, MongoDB, Redis, Altibase, Cubrid, Oracle, Oracle Pro

  • 데이터베이스 플랫폼별 V2 지원 에이전트 버전은 다음과 같습니다.

    PostgreSQL(v1.6.7), MySQL(v1.6.8), MongoDB(v1.7.0), Redis(v1.7.0), Altibase(v2.0.0), Cubrid(v2.1.0), Oracle(v1.9.0), Oracle Pro(에이전트 별도 제공)

Browser
  • 사용자 세션 분석 기능을 새롭게 제공합니다.
  • 세션 리플레이 기능을 새롭게 제공합니다.
  • 페이지 로드 통계 기능을 새롭게 제공합니다.
Network

네트워크 성능 모니터링(Network Performance Monitoring) 상품이 정식으로 출시되었습니다.

Features

KAFKA, Aerospike, Apache Pulsar, NGINX, vCenter, APACHE 베타 서비스를 제공합니다.

Log

로그 검색 및 라이브 테일 메뉴에서 검색 조건 즐겨찾기 기능을 제공합니다.



공통​

SSO 계정 연동​

SSO 계정 연동 기능이 추가되었습니다. 클라우드 기반 사용자 계정 관리 및 접근 권한 관리 솔루션 Okta와 와탭 모니터링을 연동할 수 있습니다. 조직 내에서 사용 중인 Okta 계정을 이용해 와탭 서비스에 로그인 후 서비스를 이용할 수 있습니다.

이벤트 설정 New​

새로운 이벤트 설정 New 기능은 필드(Field) 중심의 사용자 경험을 강화하여 카테고리보다 필드를 먼저 선택하는 방식으로 사용자가 인지하는 지표와 이벤트 설정의 불일치를 해소했습니다. 또한 상품에 적합한 기본 이벤트 템플릿을 제공해 보다 쉽게 경고 알림 이벤트를 설정할 수 있습니다.

기존 이벤트 설정 및 새로운 이벤트 설정 New 메뉴에 검색 기능이 추가되었습니다. 이벤트 목록 오른쪽 상단의 검색 입력창을 통해 이벤트 이름, 지표 이름를 기준으로 문자열을 입력하거나 이벤트명 또는 이벤트 수신 태그 중 원하는 기준을 선택 후 문자열을 입력하여 이벤트를 검색할 수 있습니다.

메트릭스 차트 New​

Application, Kubernetes, Database 상품군 공통으로 메트릭스 차트 New 메뉴가 새롭게 개편되었습니다. 메트릭스 차트 New는 사용자 경험을 대폭 개선하여 모니터링 작업을 더욱 효율적으로 수행할 수 있도록 설계되었습니다.

사용자는 시간, 대상, 인터벌 설정을 한 번에 처리하고 쉽게 차트를 추가할 수 있습니다. 또한 대시보드 위젯의 시간 범위를 자유롭게 변경할 수 있으며 일괄 변경 기능과 프리셋 기능을 통해 설정 관리가 간편해졌습니다. 이와 더불어 대시보드 레이아웃과 위젯도 자유롭게 조정할 수 있어 차트 분석이 더욱 효율적으로 개선되었습니다.

Application​

Go 정식 출시​

Go 애플리케이션 모니터링이 베타 버전 종료 후 정식 서비스를 시작했습니다. 와탭의 Go 애플리케이션 모니터링은 웹 프레임워크를 지원하고, Go Runtime 패키지의 정보를 지속적으로 수집합니다. 또한 MSA 환경에서 연계 추적을 통해 호출 관계를 수집합니다.

공통 메뉴 개선​

통계와 인스턴스 성능 관리 메뉴의 경우 각 메뉴의 탭 메뉴 구성을 개별 하위 메뉴로 구성하는 등 동선과 안정성 향상을 위한 개선 작업이 이루어졌습니다.

트랜잭션 맵 고도화​

트랜잭션 맵은 성능 테스트 또는 장애 상황 발생 시 단기간 트랜잭션별 현황을 상세히 확인할 수 있는 메뉴입니다. 다음과 같이 트랜잭션 맵 고도화 작업이 이루어졌습니다.

기존 TX 트레이스 섹션을 제거한 대신 차트 드래그 시 트레이스 분석 팝업 창을 통해 상세 정보를 확인할 수 있도록 구조를 개선했습니다. 또한 트레이스 분석 건수를 기존 100건에서 1000건으로 확장해 이전보다 10배 많은 트레이스를 분석할 수 있게 되었습니다.

그외에도 한 화면에서 필터를 적용해 동작을 최소화하는 방향으로 필터 기능을 개선하였습니다. 예를 들어 트랜잭션 필터를 이용해 선택한 트랜잭션만 선별해 조회할 수 있습니다. 그리고 기존 실시간 최대 5분 뿐만 아니라 과거 시점 최대 10분까지 데이터를 조회할 수 있도록 변경해 전반적으로 편의성을 향상시켰습니다.

또한 응답 시간 분포도 차트에서 선택한 항목만을 조회할 수 있는 커스텀 필터 기능을 새롭게 제공합니다. 기존 TOP 30 섹션 옆의 커스텀 탭에서 조회를 원하는 특정 트랜잭션, 도메인, IP 항목에서 원하는 값을 직접 입력하거나 TOP 30 목록에서 제공하는 항목을 선택하세요. 선택한 항목들의 응답 시간 분포도가 차트에 실시간으로 필터링됩니다.

애플리케이션 대시보드 프리셋​

애플리케이션 대시보드 메뉴로 처음 진입했을 때 조회할 수 있는 대시보드 기본 프리셋을 변경할 수 있습니다. 선호하는 대시보드 구성을 기본 프리셋으로 설정함으로써 보다 개인화된 사용자 경험을 제공합니다. 위젯 선택과 배치를 원하는대로 수정 후 대시보드 오른쪽 위에 있는 프리셋 선택 상자에서 기본으로 표시 옵션을 활용해 보세요. 기본으로 지정한 대시보드는 프리셋 목록에서 기본 태그가 표시됩니다.

정보

  • 애플리케이션 .NET 상품의 지원 범위가 기존 .NET Framework뿐만 아니라 .NET Core까지 확대되었습니다.

  • .NET 상품이 멀티 트랜잭션을 지원합니다.

  • Node.js 상품이 로그 모니터링을 제공합니다.

  • 애플리케이션 대시보드 기본 프리셋 변경에 대한 자세한 내용은 다음 문서를 참조하세요.

  • IP 트랜잭션 메뉴에 대한 자세한 설명은 다음 문서를 참조하세요.

  • 스레드 목록/덤프 메뉴에 대한 활용 안내는 다음 문서를 참조하세요.


Server​

Unix 기반 운영체제 지원 확대 및 설치 개선​

와탭은 Unix AIX, Solaris, HP-UX 설치 패키지를 제공합니다. 해당 패키지를 다운로드 받은 후 실행하는 것만으로 간단하게 설치할 수 있습니다.

  • AIX 6.1 버전 이상을 지원

  • Solaris 버전 11.2 이상을 지원

  • HP-UX Itanium 버전 11.23과 PA-RISC-UX 버전 11.23을 지원

또한 서버 모니터링 상품 설치 과정이 보다 직관적인 방식으로 개선되었습니다. 기존의 운영 체제별 탭 분리 방식 대신 사용자가 운영 체제를 먼저 선택 후 해당 OS에 맞는 개별 설치 페이지로 이동합니다.

서버 구성 정보​

와탭은 새롭게 서버 인벤토리 메뉴를 제공합니다. 서버 인벤토리는 서버의 물리적 특성 및 가상적 특성, 운영 환경 등을 체계적으로 관리하기 위한 기능입니다. 자동으로 수집된 서버 구성 요소와 사용자 정의 항목을 통해 서버의 구성을 상세하게 파악하고 관리하는 것이 가능합니다.

사용자 정의 항목을 통해 커스텀 컬럼을 구성하여 서버 상태 및 성능을 더욱 세밀하게 모니터링하고 이슈 상황에 대응할 수 있습니다. 예를 들어 각 서버 담당자와 직접적인 소통이 용이하도록 운영 담당자 정보를 추가하여 문제 시 빠른 대응이 가능합니다. 또한 서버 인벤토리 메뉴는 3단계의 다중 정렬을 통해 조건에 맞는 서버를 빠르게 필터링 할 수 있습니다. 기본 정렬은 OS 타입, 모델, 그리고 시리얼 정보 기준입니다.

노트

서버 인벤토리 메뉴에 대한 자세한 내용은 다음 문서를 참조하세요.


Database​

데이터베이스 V2​

와탭 Database 상품군이 V2로 전환되고 있습니다. 23년 4분기 PostgreSQL과 MySQL에 뒤이어 24년 와탭 데이터베이스 상품군 중 Redis, MongoDB, Altibase, Cubrid, Oracle V2 서비스를 제공합니다. V2는 전반적으로 차트 및 수집 기능이 강화되었습니다.

대시보드 메뉴의 UI 및 기능을 개선하였으며, 해당 메뉴에서 데이터베이스의 성능 지표 정보를 바로 확인할 수 있습니다. 또한 멀티 인스턴스 모니터링 메뉴에서 클러스터별 에이전트를 선택할 수 있습니다. 그리고 데이터베이스 로그를 검색하고 추이를 확인하고 로그 관련 알림을 받을 수 있는 로그 메뉴를 제공합니다.

V2 전환과 동시에 기존 XOS와 XOS2로 나누어져 있던 지표를 %로 확인할 수 있는 백분률 표기로 변경하고 XOS로 통합해 제공합니다.

Oracle Pro 출시​

Oracle Pro 정식 서비스가 시작되었습니다. 와탭 Oracle Pro는 데이터베이스 서버의 메모리에 직접 접근해 성능 지표를 모니터링하며, 최대 1초에 20번까지 최신 성능 정보를 수집합니다. 디스크 I/O를 거치지 않아 시스템 부하를 최소화하고, 쿼리 실행 시간과 버퍼 캐시 상태 등의 정밀한 성능 분석을 제공합니다.

사용자 정의형 위젯​

멀티 인스턴스 모니터링 메뉴 개선과 함께 새롭게 사용자 정의형 위젯을 제공합니다. 대시보드에서 기본 제공되는 지표 외에 사용자가 원하는 지표를 추가할 수 있습니다.

애플리케이션 연계 분석​

와탭은 연계 프로젝트를 통해 애플리케이션과 데이터베이스 프로젝트 간 연계 분석 기능을 제공합니다. 데이터베이스 프로젝트 내 인스턴스 모니터링 메뉴에서 실시간으로 데이터베이스에서 활성화된 세션과 연동된 애플리케이션의 액티브 트랜잭션 정보를 조회하여 애플리케이션과 데이터베이스 간 상호작용을 살펴볼 수 있습니다.

Java 프로젝트 내 애플리케이션 대시보드에서는 연계 DB 세션 기능을 통해 액티브 트랜잭션과 연계된 데이터베이스의 SQL 성능을 조회할 수 있습니다. 연계 프로젝트 관리 기능은 여러 프로젝트의 데이터를 통합적으로 확인하여 문제 원인을 빠르게 파악하고 효율적으로 성능 관리를 할 수 있도록 돕습니다.

현재 애플리케이션과 데이터베이스 연계 분석은 Java 플랫폼만 지원합니다. 또한 연계 DB 세션 기능의 경우 Java 에이전트 2.2.33 버전 이상이 필요합니다. 데이터베이스 플랫폼의 경우 PostgreSQL, MySQL, Oracle V2 및 Oracle Pro 플랫폼이 해당 기능을 지원합니다.

사용자 세션 분석​

새롭게 제공하는 사용자 세션 분석 기능은 사용자의 웹 페이지 탐색 방식을 파악하고자 할 때 활용할 수 있습니다. 사용자가 어떻게 움직이고, 어떤 페이지에 머무르고 있는지, 화면의 어떤 요소를 클릭하는지, 화면에서 발생하는 에러 등에 관한 사용자의 전반적인 활동을 분석해 사용자가 서비스를 이용하는 방식에 대한 통찰을 얻는 것뿐 아니라 사용자의 행동 패턴을 이해하는 데에 더 가까워질 수 있습니다.

사용자 세션 로그 검색, AJAX 히트맵과 페이지 로드 히트맵을 이용해 사용자의 페이지 내 행동 분석 및 세션 중 발생하는 여러 에러 추적이 가능합니다. 세션 로그 정보를 통해 최종 사용자의 경로, 이탈 원인, 성능 및 사용자 경험 등을 분석할 수 있습니다. 또한 AJAX 히트맵과 페이지 로드 히트맵에서 실시간으로 문제를 겪고 있는 사용자를 파악할 수 있습니다. 이러한 정보를 통해 사용자의 서비스 경험과 서비스의 현재 성능 상태를 확인할 수 있습니다.

세션 리플레이​

새롭게 제공하는 세션 리플레이 기능은 사용자가 웹 사이트에서 수행하는 모든 이벤트를 자동으로 기록하고 재생할 수 있는 기능입니다.

사용자의 클릭, 스크롤, 입력, 페이지 전환 등과 같은 모든 이벤트를 정확하게 재현하여 웹 사이트와 사용자 사이의 상호작용을 분석할 수 있습니다. 이를 통해 실제 사용자 경험을 재현하여 잠재적인 문제를 빠르게 식별하고 해결할 수 있으며, 사용자의 웹 사이트 사용 방식에 대한 통찰을 제공하여 최적화를 도울 수 있습니다.

컨트롤 플레인 모니터링​

와탭 쿠버네티스 모니터링은 새롭게 컨트롤 플레인 모니터링을 제공합니다. 쿠버네티스 클러스터 전반을 관리하는 컨트롤 플레인 컴포넌트 중 클러스터 내 API 요청을 처리하는 kube-apiserver에 대한 가시성 확보를 위해 kube-apiserver 대시보드 및 kube-apiserver 메트릭스 조회 기능이 추가되었습니다.

kube-apiserver 대시보드는 kube-apiserver의 상태 및 성능 메트릭 수집, 응답 시간, 처리량, 요청 실패 등을 추적하여 클러스터의 가용성과 성능을 모니터링할 수 있습니다. kube-apiserver 메트릭스 조회를 통해서는 kube-apiserver 수집 지표 목록을 조회할 수 있습니다.

Deployment 목록과 쿠버네티스 에이전트 목록​

또한 Deployment 기준으로 관련 정보를 조회할 수 있는 Deployment 목록, 마스터 및 노드 에이전트 설치 현황을 확인할 수 있는 쿠버네티스 에이전트 목록 기능이 추가되었습니다.

개별 Pod 보다 Pod 집합 상태를 모니터링하는 것이 서비스 안정성과 성능 확보를 위해 중요합니다. 애플리케이션 배포 및 업데이트를 담당하는 Deployment 모니터링은 그러한 의미에서 필수적입니다. Deployment 목록에서 리소스 관계도 및 클러스터 내 Pod, 컨테이너, 애플리케이션 현황 및 CPU, 메모리, 트랜잭션 수 등 주요 성능 지표를 Deployment 단위로 조회할 수 있습니다.

또한 새롭게 추가된 쿠버네티스 에이전트 목록에서는 마스터 및 노드 에이전트 설치 현황을 한 눈에 살펴볼 수 있습니다.

Pod 시작 분석과 오브젝트 매니페스트​

기존의 Pod 초기화 성능 기능 및 마스터 메타 정보 기능을 개편해 각 Pod 시작 분석과 오브젝트 매니페스트 기능을 제공합니다. Pod 시작 분석은 Pod별 시작 시점 정밀 분석을 통해 기동에 소요되는 시간과 리소스 사용량에 대한 상세 데이터를 제공합니다. 오브젝트 매니페스트는 오브젝트별 과거 조회 및 비교 기능이 추가되어 클러스터 내 변동 사항을 쉽게 추적할 수 있습니다.

아울러 마스터 및 노드 에이전트 설치 시 기존 Yaml 파일 방식 외 운영 복잡도를 덜어줄 Helm 차트 설치 방식을 공식적으로 지원합니다.

Pending Pod 현황​

실험실 메뉴에서 제공되던 Pending Pod 현황 기능이 V2로 개편되면서 워크로드 하위 메뉴로 정식 배포되었습니다. Pending Pod 현황 기능은 Pending 상태의 Pod에 대한 실시간 가시성을 제공함은 물론 Pending의 원인 분석을 가능하게 하는 연계 데이터를 함께 제공합니다. 이를 통해 클러스터 및 서비스의 가용성을 높일 수 있습니다.

리소스 관계​

컨테이너 맵에 리소스 관계 기능이 추가되었습니다. 리소스 관계 기능을 사용하면 쿠버네티스 리소스 간 계층 구조 및 연결 관계를 시각적으로 확인할 수 있기에 쿠버네티스 환경 분석은 물론 문제 발생 시 원인 파악 및 영향도를 한 눈에 파악할 수 있습니다.

노드 모니터링 확장​

와탭 쿠버네티스 모니터링은 노드 모니터링 확장과 함께 여러 가지 새로운 기능을 제공합니다. 노드 맵, 노드 타임라인, 노드 디스크 목록과 같은 신규 메뉴가 추가되었으며, 기존 노드 목록 메뉴가 보다 효율적으로 개선되었습니다.

노드 맵을 통해 클러스터 관점에서 노드 상태를 모니터링할 수 있습니다. 노드의 실시간 상태, 리소스 사용 현황 및 노드 간 관계를 직관적으로 파악할 수 있으며, 그룹화, 임계치 설정, 라벨 등의 사용자 설정을 통해 사용자가 원하는 방식으로 대시보드를 구성할 수 있습니다.

노드 타임라인 메뉴에서 시간에 따라 노드의 상태 변화를 추적할 수 있습니다. 예를 들어 특정 노드의 제거 시점 및 해당 시점의 상태를 시각적으로 파악할 수 있습니다. 이를 통해 클러스터 운영 중 발생한 이슈 분석이나 과거 상황을 확인하는 데 유용합니다.

노드 디스크 목록을 통해 노드에 마운트된 개별 디스크의 상태를 실시간으로 모니터링할 수 있습니다. 그리고 기존 노드 목록 메뉴의 비교 및 상세 정보 조회 기능이 개선되었습니다. 노드 상세 메뉴로 이동하는 대신 선택한 노드의 구성 및 리소스, 메트릭스, 이벤트 정보를 노드 목록 상세 모달을 통해 확인할 수 있습니다.

구성 요소 분석 강화​
