{/* 


1. 컨트롤 플레인(Control Plane)

* 역할: 쿠버네티스 클러스터의 중앙 관리 시스템으로, 클러스터 내의 모든 작업과 리소스를 관리하고 조정합니다. 클러스터의 상태를 유지하며, 워커 노드에게 작업을 할당합니다.
* 구성 요소:
* API 서버 (kube-apiserver): 클러스터와의 모든 상호작용의 중심점으로, API 요청을 처리합니다.
* etcd: 클러스터 상태 정보를 저장하는 분산 키-값 데이터베이스입니다.
* 스케줄러 (kube-scheduler): 워크로드를 실행할 워커 노드를 선택하여 Pod를 배포합니다.
* 컨트롤러 매니저 (kube-controller-manager): 클러스터의 상태를 모니터링하고 조정하여 원하는 상태를 유지합니다.
* 클라우드 컨트롤러 매니저 (cloud-controller-manager): 클라우드 서비스와의 상호작용을 담당합니다.

2. 마스터 노드(Master Node)

* 정의: 컨트롤 플레인의 주요 구성 요소들이 실행되는 물리적 또는 가상 머신입니다.
* 역할: 클러스터의 모든 제어 기능을 수행하며, 클러스터 내의 작업과 리소스를 조정합니다.

3. 워커 노드(Worker Node)

* 정의: 애플리케이션이나 서비스가 실제로 실행되는 노드입니다.
* 역할: 워커 노드는 컨트롤 플레인의 명령에 따라 Pod를 실행하고 관리하며, 클러스터의 워크로드를 실제로 처리합니다.
* 구성 요소:
* Kubelet: 워커 노드에서 실행되며, 컨트롤 플레인과 통신하여 Pod의 상태를 관리하고 모니터링합니다.
* Kube-proxy: 네트워킹을 관리하고, 서비스 디스커버리와 로드 밸런싱을 지원합니다.
* 컨테이너 런타임: Docker나 containerd와 같은, 컨테이너를 실행하는 소프트웨어입니다.

4. 워크로드(Workloads)

* 정의: 클러스터에서 실행되는 애플리케이션이나 서비스의 작업 단위를 의미합니다. 일반적으로 Pod, Deployment, ReplicaSet, StatefulSet, DaemonSet, 그리고 Job 등의 형태로 존재합니다.
* 역할: 워크로드는 사용자 애플리케이션의 실제 실행 환경을 제공하며, 클러스터 내에서 배포되고 관리됩니다.
* Pod: 쿠버네티스에서 가장 작은 배포 단위로, 하나 이상의 컨테이너를 포함합니다.
* Deployment: 애플리케이션의 선언적 업데이트를 관리하는 컨트롤러로, 안정적인 배포와 업데이트를 지원합니다.
* ReplicaSet: 특정 수의 Pod를 항상 유지하도록 하는 컨트롤러입니다.
* StatefulSet: 상태를 유지해야 하는 애플리케이션을 위한 컨트롤러입니다.
* DaemonSet: 클러스터의 모든 또는 일부 노드에서 하나의 Pod를 실행하는 컨트롤러입니다.
* Job: 일회성 작업을 실행하고 완료를 보장하는 컨트롤러입니다.

5. 구조 요약

* 컨트롤 플레인: 클러스터의 두뇌 역할로, 모든 워크로드와 노드를 관리하고 조정합니다.
* 마스터 노드: 컨트롤 플레인이 실행되는 노드로, 클러스터의 중앙 관리 기능을 수행합니다.
* 워커 노드: 실제로 애플리케이션과 서비스를 실행하는 노드로, 워크로드를 처리합니다.
* 워크로드: 클러스터에서 실행되는 애플리케이션이나 서비스의 작업 단위로, Pod와 다양한 컨트롤러로 구성됩니다.


참고. 주요 쿠버네티스 오브젝트

* Pod: 하나 이상의 컨테이너를 실행하는 가장 작은 배포 단위입니다.
* Deployment: 파드의 배포와 관리를 자동화하는 오브젝트로, 지정된 상태를 유지하기 위해 레플리카셋을 관리합니다.
* ReplicaSet: 특정 수의 파드를 항상 유지하도록 보장하는 오브젝트로, 파드의 수를 스케일링하거나 유지하는 역할을 합니다.
* Service: 파드 간의 네트워크 연결을 관리하고, 파드가 변경되더라도 안정적인 접근을 가능하게 하는 오브젝트입니다.
* ConfigMap 및 Secret: 애플리케이션의 설정 정보를 관리하는 오브젝트로, 환경 변수나 설정 파일로 사용됩니다.
* Ingress: 외부 트래픽을 클러스터 내의 서비스로 라우팅하는 규칙을 정의하는 오브젝트입니다.
* PersistentVolume(PV) 및 PersistentVolumeClaim(PVC): 스토리지 리소스를 관리하고 요청하는 오브젝트로, 파드에 지속적인 스토리지를 제공합니다. 
  
참고. Pending Pod 현황을 통해 파드가 실행되지 않고 대기 상태에 있는 이유를 확인하고 클러스터의 리소스 상태나 스케줄링 문제를 파악

참고. Pod 시작 분석을 통해 파드가 클러스터에서 성공적으로 실행되기까지의 상태를 분석하는 과정으로 파드 실행 문제 또는 초기화 과정에서 문제 진단 

*/}

## `공통`

와탭 2024년 1분기 `공통` 부분의 주목할 만한 신규 기능으로 사이드 메뉴 개편, SSO 계정 연동 기능 추가 및 새로운 이벤트 설정 기능을 소개합니다. 

사용자 편의성 향상을 위해 5년 만에 와탭 모니터링의 **사이드 메뉴**가 새롭게 개편되었습니다. 와탭 서비스 이용에는 프로젝트 선택이 선행됩니다. 프로젝트 생성 및 선택 이후 동선을 편리하게 활용할 수 있도록 기존과 비교해 UI 혼잡도를 줄여 프로젝트 선택과 메뉴의 영역을 확보하는 구조로 개편되었습니다. 

프로젝트 목록을 선택한 부분만 펼쳐서 확인할 수 있으며, 세로 영역을 확보해 보다 편하게 다양한 기능 메뉴를 조회할 수 있습니다. 또한 간단한 별 모양(<LinkImage img='ic-star.svg' desc='비교하기 아이콘'/>) 아이콘 클릭으로 상품별 프로젝트 메뉴 내 자주 쓰는 기능들을 즐겨찾기 할 수 있습니다. 

<ImgLang img='release/release-main-ui.png' desc='사이드 메뉴 개편 cl 분기 예시' />

또한 **SSO 계정 연동 기능**이 추가되었습니다. 클라우드 기반 사용자 계정 관리 및 접근 권한 관리 솔루션 **Okta**와 와탭 모니터링을 연동할 수 있습니다. 조직 내에서 사용 중인 **Okta** 계정을 이용해 와탭 서비스에 로그인 후 서비스를 이용할 수 있습니다. 

<ImgLang img='sso-check-sso-info.png' desc='sso-info cl 분기 예시' />

<Cmdname sid="side_kubeApiserverDashboard" className="uitext" />는 kube-apiserver의 상태 및 성능 메트릭 수집, 응답 시간, 처리량, 요청 실패 등을 추적하여 클러스터의 가용성과 성능을 모니터링할 수 있습니다. <Cmdname sid="side_kubeApiserverMetricsSearch" className="uitext" />를 통해서는 kube-apiserver 수집 지표 목록을 조회할 수 있습니다.

또한 Deployment 기준으로 관련 정보를 조회할 수 있는 <Cmdname sid="side_deploymentList" className="uitext" />, 마스터 및 노드 에이전트 설치 현황을 확인할 수 있는 <Cmdname sid="side_kubernetesAgentList" className="uitext" /> 기능이 추가되었습니다.

## `Network`

와탭 2024년 1분기에 네트워크 간의 혼잡 및 장애를 파악하고 사용자의 네트워크 성능 개선을 위해 프로세스의 네트워크 정보를 수집하는 **네트워크 성능 모니터링**(**Network Performance Monitoring**) 상품이 정식으로 출시되었습니다. 

![네트워크 모니터링 sc cl ](/img/changelog/changelog-network.png)

최근 네트워크 시장은 클라우드와 가상화라는 큰 틀에서 성장하고 있습니다. 기존 장비 중심의 네트워크 모니터링 뿐만 아니라 작업을 수행하는 프로세스가 중요한 모니터링 대상이 되고 있습니다. 와탭 네트워크 성능 모니터링을 통해 프로세스 단위의 네트워크 지표를 안정적으로 모니터링해 보세요.

와탭의 `Network` 상품은 각 프로세스가 수행하는 네트워크 동작을 추적해 복잡한 구성 요소와 상태 정보를 토폴로지를 통해 직관적으로 이해할 수 있도록 시각화했습니다. **품질 지표** ***Latency***와 ***Jitter***, 그리고 **네트워크 사용량 지표** ***bps***, ***pps***, ***session count***를 조회할 수 있습니다. 또한 태그 옵션으로 가시성을 확보하였으며, <Cmdname sid="side_flexboard" className="uitext" />를 통해 다양한 사용자 커스텀 차트를 제공합니다.
