2분기 안내

2024년 7월 9일
TW J
테크니컬 라이터

와탭의 2024년 2분기 서비스 신규 기능 및 주목할 만한 변경 사항 개요를 안내합니다. 와탭이 새롭게 제공하는 기능들을 한 눈에 살펴보세요.

공통

편의성 향상을 위해 공통 메뉴 개선이 이루어졌습니다.

Application
  • 트랜잭션 맵이 지난 1분기에 이어 고도화되었습니다.
  • 애플리케이션 대시보드에 기본 프리셋 변경 기능이 추가되었습니다.
Server
  • 서버 프로젝트 생성 시 OS를 먼저 선택할 수 있도록 설치 화면이 개편되었습니다.
  • Unix Oracle Solaris 설치 패키지를 제공합니다.
Kubernetes
  • 에이전트 설치 화면이 보다 손쉬운 옵션 선택 방식으로 개편되었습니다.
  • Pending Pod 현황 메뉴가 새롭게 개편되었습니다.
  • 컨테이너 맵 메뉴에서 리소스 관계도를 제공합니다.
Database

Oracle Pro 베타 서비스를 제공합니다.

Browser

세션 리플레이 기능을 새롭게 제공합니다.

상품별 신규 기능 외 변경 사항 개요 및 해당 릴리스 버전, 에이전트 변경 사항은 다음 안내를 통해 확인할 수 있습니다.



공통​

와탭 2024년 2분기 공통 부분에서 편의성 향상을 위해 Flex 보드를 비롯한 공통 메뉴 개선 작업이 이루어졌습니다. 사이드 메뉴의 프로젝트 목록에서 쉽게 그룹을 이동할 수 있으며, Flex 보드의 경우 그룹 내 옵션을 공유할 수 있는 그룹 내 전체 위젯 적용 기능, 그룹 위젯 내 프로젝트 선택 기능이 추가되었습니다.

다음 개요 안내를 통해 2024년 2분기 공통 부분의 신규 기능 및 주요 변경 사항을 해당 릴리스 버전과 함께 확인해 보세요.

Application​

와탭 2024년 2분기 Application 부분의 주목할 만한 신규 기능으로 지난 1분기에 이어 트랜잭션 맵의 고도화와 애플리케이션 대시보드 기본 프리셋 변경 기능을 소개합니다.

트랜잭션 맵 고도화 작업의 일환으로 응답 시간 분포도 차트에서 선택한 항목만을 조회할 수 있는 커스텀 필터 기능을 새롭게 제공합니다. 기존 TOP 30 섹션 옆의 커스텀 탭에서 조회를 원하는 특정 트랜잭션, 도메인, IP 항목에서 원하는 값을 직접 입력하거나 TOP 30 목록에서 제공하는 항목을 선택하세요. 선택한 항목들의 응답 시간 분포도가 차트에 실시간으로 필터링됩니다.

애플리케이션 대시보드 메뉴로 처음 진입했을 때 조회할 수 있는 대시보드 기본 프리셋을 변경할 수 있습니다. 선호하는 대시보드 구성을 기본 프리셋으로 설정함으로써 보다 개인화된 사용자 경험을 제공합니다. 위젯 선택과 배치를 원하는대로 수정 후 대시보드 오른쪽 위에 있는 프리셋 선택 상자에서 기본으로 표시 옵션을 활용해 보세요. 기본으로 지정한 대시보드는 프리셋 목록에서 기본 태그가 표시됩니다.

Server​

와탭 2024년 2분기 Server 부분의 주목할 만한 신규 기능으로 설치 화면 개선 및 Unix Solaris 설치 패키지에 대해 소개합니다.

서버 모니터링 상품 설치 과정이 보다 직관적인 방식으로 개선되었습니다. 기존의 운영 체제별 탭 분리 방식 대신 사용자가 운영 체제를 먼저 선택 후 해당 OS에 맞는 개별 설치 페이지로 이동합니다.

이번 2분기에 새롭게 Unix Solaris 에이전트 설치 패키지를 제공합니다. 선 마이크로시스템즈가 개발한 Solaris는 고성능 Unix 운영 체제로, SPARC 시스템에서의 확장성과 ZFS 파일 시스템, DTrace와 같은 기능으로 널리 알려져 있습니다. 와탭은 Oracle Solaris 버전 11.2 이상을 지원합니다. Solaris 설치 패키지를 통해 해당 패키지를 다운로드 받은 후 실행하는 것만으로 간단하게 설치할 수 있습니다. 와탭 서버 모니터링을 통해 Solaris 운영 체제를 보다 효과적으로 관리하고 모니터링할 수 있습니다.

Kubernetes​

와탭 2024년 2분기 Kubernetes 부분의 주목할 만한 신규 기능 및 변경 사항으로 에이전트 설치 및 Pending Pod 현황 개편, 컨테이너 맵 내 리소스 관계 기능을 소개합니다.

쿠버네티스 제품에 발생하는 지속적인 변화와 다양한 컨테이너 런타임의 등장에 따라 에이전트 설치 과정 또한 복잡해지고 있습니다. 이에 와탭은 지난 1분기의 Helm 차트 설치 지원에 이어 2분기에는 새로운 쿠버네티스 에이전트 설치 기능을 제공합니다. 기존에는 설치 YAML 파일 다운로드 후 추가적으로 항목을 수정해야 하는 경우가 있었습니다. 하지만 이제는 UI 상에서 사용자 쿠버네티스 환경의 주요 구성요소나 특징, 그리고 사용하고자 하는 기능 선택 시 그에 적합한 설치 파일이 제공됩니다. 따라서 훨씬 더 빠르고 쉽게 에이전트를 설치할 수 있습니다.

실험실 메뉴에서 제공되던 Pending Pod 현황 기능이 V2로 개편되면서 워크로드 하위 메뉴로 정식 배포되었습니다. Pending Pod 현황 기능은 Pending 상태의 Pod에 대한 실시간 가시성을 제공함은 물론 Pending의 원인 분석을 가능하게 하는 연계 데이터를 함께 제공합니다. 이를 통해 클러스터 및 서비스의 가용성을 높일 수 있습니다.

와탭 쿠버네티스 모니터링은 2024년 상반기 내내 지속적인 컨테이너 맵 개선 작업을 진행하였습니다. 특히 2분기에는 컨테이너 맵에 리소스 관계 기능이 추가되었습니다. 리소스 관계 기능을 사용하면 쿠버네티스 리소스 간 계층 구조 및 연결 관계를 시작적으로 확인할 수 있기에 쿠버네티스 환경 분석은 물론 문제 발생 시 원인 파악 및 영향도를 한 눈에 파악할 수 있습니다.

Database​

와탭 2024년 2분기 Database 부분에서 주목할 만한 신규 기능 및 변경 사항으로 Oracle 상품 V2 출시와 Oracle Pro 베타 출시를 소개합니다.

와탭 데이베이스 상품군이 V2로 전환되고 있습니다. 1분기 Redis와 MongoDB에 뒤이어 2분기는 Oracle V2 전환이 시작되었습니다.

3분기 정식 출시를 앞두고 Oracle Pro 베타 서비스가 시작되었습니다. Oracle Pro는 데이터베이스 서버의 메모리에 직접 접근해 성능 지표를 모니터링하는 기술을 적용하였습니다. 최신 데이터를 반영하여 정확한 성능 정보를 최대 1초에 20번까지 모니터링할 수 있으며, 디스크 I/O를 거치지 않고 메모리에 직접 접근하여 시스템 미치는 부하를 최소화합니다. 이를 통해 특정 쿼리의 실행 시간, 버커 캐시 상태 등 정밀하게 수집한 성능 정보를 바탕으로 문제를 빠르고 정확하게 분석할 수 있습니다.

사용자의 클릭, 스크롤, 입력, 페이지 전환 등과 같은 모든 이벤트를 정확하게 재현하여 웹 사이트와 사용자 사이의 상호작용을 분석할 수 있습니다. 이를 통해 실제 사용자 경험을 재현하여 잠재적인 문제를 빠르게 식별하고 해결할 수 있으며, 사용자의 웹 사이트 사용 방식에 대한 통찰을 제공하여 최적화를 도울 수 있습니다.

Feature 페이지 로드 히트맵, AJAX 히트맵, 사용자 세션 로그 검색 메뉴에 세션 리플레이 기능 추가 Service 2.6.0​

Feature​

현재 피쳐 프로젝트 방식으로 KAFKA 모니터링 베타 서비스를 시작하였습니다. 추후 정식 배포 예정이며 KAFKA 외 지원 범위 역시 확대될 예정입니다.
