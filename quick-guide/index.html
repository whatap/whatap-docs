빠른 시작하기

와탭 모니터링 서비스 개괄 안내 페이지입니다. 와탭 모니터링 서비스를 처음 접하는 신규 사용자에게 유용합니다.

노트

와탭 모니터링 서비스 에이전트 설치 및 활용에 대한 자세한 안내는 왼쪽의 상품별 기술 문서를 참조하세요.

빠른 시작 단계별 안내​

와탭 회원 가입​

와탭 모니터링 서비스를 이용하려면 회원 가입을 진행하세요.

팁
  • 와탭 모니터링을 15일간 무료로 체험해 보세요. 결제 방식에 대한 자세한 안내는 다음 문서를 참조하세요.
  • 회원 가입, 프로젝트 생성 후 프로젝트 액세스 키를 발급한 날부터 무료 체험을 시작합니다.
  • 체험 기간 종료 후 서비스를 계속 이용하고자 한다면 결제 정보를 등록하고 프로젝트를 유료로 전환하세요.
  1. 와탭 홈페이지로 이동하세요.
  2. 오른쪽 위에 무료로 시작하기 버튼을 선택하세요.
  3. 이름, 이메일, 비밀번호, 회사명을 차례로 입력하세요.
  4. 계정 생성 버튼을 선택하세요.
노트

알림을 SMS로 수신하려면 로그인한 다음 오른쪽 위쪽에 프로필 아이콘 > 계정 관리 순으로 선택하세요. 사용자 전화번호 섹션으로 이동해 전화번호를 입력하세요.

프로젝트 생성​

에이전트를 설치하기 전에 먼저 프로젝트를 생성하세요.

  1. 와탭 모니터링 서비스로 이동한 다음 로그인하세요.

  2. 프로젝트를 생성하려면 화면 왼쪽 사이드 메뉴에서 전체 프로젝트 > + 프로젝트 버튼을 선택하세요.

  3. 상품 선택 화면에서 프로젝트에 설치할 상품을 선택하세요.

  4. 프로젝트 이름, 데이터 서버 지역, 타임 존 등의 항목을 차례로 설정하세요.

  5. 알림 언어 설정에서 경고 알림 메시지의 언어를 선택하세요.

  6. 모든 설정을 완료한 다음에는 프로젝트 생성하기 버튼을 선택하세요.

노트

  • 데이터 서버 지역은 리전(클라우드 서비스를 제공하기 위해 설치한 데이터 센터의 묶음)을 의미합니다. 특정 리전을 선택하면 해당 리전에 속한 데이터 센터에 사용자의 데이터를 저장합니다.

  • 타임 존은 알림, 보고서를 생성하는 기준 시간입니다.

  • 여러 개의 프로젝트를 그룹으로 묶어 관리하려면 프로젝트 그룹에서 그룹을 선택하거나 그룹을 추가하세요. 그룹에 대한 자세한 설명은 다음 문서를 참조하세요.
  • 조직을 선택한 상태에서 프로젝트를 추가할 경우 조직 하위 그룹을 필수로 설정해야 합니다.

프로젝트 액세스 키 확인​

프로젝트 액세스 키는 와탭 서비스 활성화를 위한 고유 ID입니다.

설치 안내 섹션에서 프로젝트 액세스 키 발급받기 버튼을 선택하세요. 프로젝트 액세스 키를 자동으로 발급 받은 후 다음 단계를 진행합니다.

팁

프로젝트를 생성한 다음에는 자동으로 에이전트 설치 페이지로 이동합니다. 에이전트 설치 페이지로 이동하지 않는다면 왼쪽 메뉴에서 전체 프로젝트를 선택한 다음 새로 생성한 프로젝트를 선택하세요.

에이전트 설치​

프로젝트 액세스 키를 발급받았다면 설치 안내 섹션에서 와탭 에이전트 설치를 진행할 수 있습니다. 관리 > 에이전트 설치 메뉴를 통해 설치 안내 섹션에 접근할 수도 있습니다.

  1. 지원 환경을 확인하세요.

  2. 관리 > 에이전트 설치 메뉴의 에이전트 다운로드 탭을 선택하세요. 안내에 따라 직접 에이전트를 다운로드 받거나 패키지 설치를 진행하세요.

    노트

    에이전트 다운로드

    • 환경에 따라 다운로드 버튼을 선택하거나 명령어를 이용해 서버에서 직접 다운로드합니다.
    • 환경에 따라 리포지토리를 등록하고 패키지를 설치합니다.

  3. 프로젝트 액세스 키와 와탭 서버 IP를 입력하고 해당 설정 스크립트를 실행하세요.

  4. 에이전트를 재시작하세요.

이제부터 와탭 모니터링 서비스가 시작됩니다.

다운로드 및 설정 과정에 대한 자세한 안내는 상품별 설치 안내 문서를 참조하세요. 왼쪽에서 상품별 메뉴를 선택하거나 다음의 바로가기를 확인하세요.

대시보드 조회​

에이전트 설치 후 데이터가 와탭 서버로 전달되면 대시보드 메뉴에서 프로젝트의 전체 현황을 실시간으로 조회할 수 있습니다. 와탭 모니터링 서비스는 수집된 모든 데이터를 통합 대시보드로 구성할 수 있으며, 사용자에 의해 자유 편집이 가능한 개인화된 대시보드를 제공합니다.

  • 애플리케이션은 애플리케이션 대시보드와 트랜잭션 맵, 액티브 트랜잭션, 애플리케이션 토폴로지 그리고 Flex 보드의 각 위젯에서 수집된 성능 지표를 확인할 수 있습니다.

  • 서버는 리소스 보드와 컴파운드 아이 그리고 Flex 보드의 각 위젯에서 수집된 성능 지표를 확인할 수 있습니다.

  • 데이터베이스는 인스턴스 목록과 인스턴스 모니터링, 멀티 인스턴스 모니터링, 슬로우 쿼리 그리고 Flex 보드의 각 위젯에서 수집된 성능 지표를 확인할 수 있습니다.

  • 쿠버네티스는 컨테이너 맵의 각 위젯을 통해 노드, 컨테이너, 애플리케이션, 로그, 이벤트 정보를 동적으로 확인할 수 있습니다.

노트

사용자 정의형 통합 대시보드인 Flex 보드 생성 및 설정에 대한 자세한 설명은 다음 문서를 참조하세요.

외부 데이터 수집​

Telegraf와 Focus 등을 활용해 와탭 서비스 수집 데이터를 확장할 수 있습니다. 자세한 설명은 다음 문서 Telegraf, Focus를 참조하세요.

  • 와탭 플러그인을 설치해 Telegraf에서 수집한 데이터를 와탭 수집 서버로 전달할 수 있습니다.
  • 와탭 Focus를 설치해 사용자가 자체 개발한 체크 스크립트 혹은 데이터베이스 쿼리를 작동하여 수집된 값 등을 사용자의 와탭 프로젝트에 업로드할 수 있습니다.

알림 설정​

와탭 모니터링 서비스는 수집되는 모든 지표에 대해 임계치 기반 알림 설정이 가능합니다. 경고 알림 메뉴에서 이벤트와 수신 설정 및 지난 기록을 확인할 수 있습니다. 개인별로 알림 수신 설정이 가능할 뿐 아니라 이메일, SMS, 메신저 등 다양한 알림 수신 채널을 제공합니다.

알림 설정에 대한 자세한 설명은 상품별 다음 문서를 참조하세요.

보고서 조회​

보고서는 서비스 이용 현황과 장애 발생 기록을 보고하는 문서로 모니터링 데이터 분석은 서비스의 개선 방향을 정하는 지표가 됩니다. 와탭은 보고서 메뉴에서 보고서를 생성해 업무 자동화를 지원합니다. 개별 프로젝트 단위 보고서와 다중 프로젝트 단위의 통합 보고서를 제공합니다.

노트

보고서에 대한 자세한 설명은 다음 문서를 참조하세요. 상품별 보고서 안내는 상품별 기술 문서를 참조하세요.

멤버 관리​

사용자는 관리 > 프로젝트 멤버 메뉴에서 프로젝트 멤버를 추가할 수 있습니다. 프로젝트와 그룹별로 멤버의 권한을 설정하고 또 멤버를 삭제할 수 있습니다.
