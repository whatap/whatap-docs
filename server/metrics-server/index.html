서버 지표

서버 모니터링 지표를 다음과 같이 안내합니다.

Category V1​

agent_status_summary​

에이전트 상태와 관련한 지표를 10초 간격으로 수집한 카테고리입니다.

Fields​

FieldTypeUnitDescription
inActTime-밀리초(ms)에이전트가 비활성화된 상태로 유지된 시간
isActive--현재 에이전트의 활성 상태 여부 (true / false)
isRebooted--서버가 최근에 재시작되었는지 여부 (true / false)
lastActTime-밀리초(ms)마지막으로 에이전트가 활성화된 상태의 시각 (0: 비활성화된 경우)
oid--프로젝트에 포함된 각 에이전트의 고유 식별자
startTime-밀리초(ms)에이전트가 시작된 시점의 타임스탬프
status--현재 에이전트의 상태
uptime-초(sec)서버가 가동 중인 시간
노트

status 필드의 값은 다음과 같이 구분합니다.

  • ok: 서버가 활성화되어 있는 정상 상태입니다.

  • inactive: 서버가 비활성화된 상태입니다.

  • warning: 서버에 경고(warning) 이벤트가 진행 중인 상태입니다.

  • fatal: 서버에 위험(critical) 이벤트가 진행 중인 상태입니다.

  • paused: 에이전트가 데이터 수집을 일시 중지한 상태입니다.

필드명설명OS

nice전체 CPU 사용률 중 낮은 우선 순위(niced) 프로세스의 사용률을 의미합니다. 높은 우선순위 및 기본 우선순위 프로세스 사용률은 user(퍼센트)에 표시됩니다.HP-UX, Linux
wait전체 CPU 사용률 중 CPU가 I/O 작업 완료를 기다리는 데 소비한 시간 비율을 의미합니다.HP-UX, AIX, Solaris, Linux
total_wait전체 CPU 사용률 중 사용자 모드(User), 커널 모드(Sys), I/O 대기(wait)의 사용률 합계를 의미합니다.HP-UX, AIX, Solaris, Linux
idleCPU가 유휴 상태로, 작업을 수행하지 않은 시간 비율을 의미합니다.HP-UX, AIX, Solaris, Linux, Windows
lCoresOS에 할당된 Logical Core 수를 의미합니다.HP-UX, AIX, Solaris, Linux, Windows
irq하드웨어 Interrupt Request를 처리하는 데 사용된 CPU 사용률을 의미합니다.Linux
softirq소프트웨어 Interrupt Request를 처리하는 데 사용된 CPU 사용률을 의미합니다.Linux
steal가상화 환경에서 하이퍼바이저가 다른 프로세스를 서비스하는 동안 가상 CPU가 비자발적으로 대기하는 시간 비율을 의미합니다.Linux
procUtilWindows 전용 지표로, 시스템의 모든 프로세서의 총 CPU 사용률을 의미합니다 (Windows 작업 관리자 CPU 사용률).Windows
loadavg_1m최근 1분 동안의 시스템 평균 부하를 의미합니다.HP-UX, AIX, Solaris, Linux
loadavg_5m최근 5분 동안의 시스템 평균 부하를 의미합니다.HP-UX, AIX, Solaris, Linux
