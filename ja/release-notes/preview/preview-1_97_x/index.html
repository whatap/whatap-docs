  • Feature ログトレンドから検索キーを最大2つまで選択できるように修正(検索値は制限なし)

  • Change Server プロジェクトの ログ設定 メニューのエージェント設定オプションを変更

アラート通知​

  • Feature イベント設定 メニューから エージェント重複通知 オプションを追加

    モニタリング中のエージェント間で、同じIDを設定したエージェントが2つ以上ある場合、メトリクスを重複して収集することによって発生するアラート通知です。エージェント名が重複するように設定されていないことをエージェント設定で関連オプションを確認してください。青/緑などのデプロイ状況でエージェントが重複してアラート通知が発生した場合、この機能を利用することをお勧めします。同じ名前のエージェントが同時に2つ以上起動した場合、選択した時間に通知はされません。

  • Feature イベント受信設定 メニューから 受信タグの未設定通知を受け取る オプションを追加

    イベント受信タグが設定されていないアラート通知を受信しない場合は、そのオプションを選択します。
