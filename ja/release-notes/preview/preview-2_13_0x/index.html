Preview 2.13.0.a

ノート

preview.whatap.ioで、次の新機能と変更した内容を事前に体験できます。

Preview 2.13.0.a3​

2025年01月14日

共通​

  • Feature 分析 > メトリクスチャート Newメニューの機能を改善

    • チャートデータにマウスオーバーする時、ヒントデータのMax値が記録された特定の時間を追加

    • 凡例一覧の全体を選択/解除できるボタンを追加

    • チャートラインのカラー選択機能を追加

    • もっと見るメニューにCSVコピーボタンを追加

    • もっと見るメニューにチャート画像のコピーボタンを追加

  • Feature アラート通知 > 通知設定メニューからサードパーティのプラグインによる通知受信レベル選択機能を追加

  • Changed 統合レポートメニューのタイムセレクター機能を交替

    • 日刊報告書: 開始と終了の日付、時間を選択できます。

    • 週間報告書: 開始日のみを選択すると、終了日を計算して自動的に選択します。

    • 月間報告書: 開始日のみを選択すると、終了日を計算して自動的に選択します。

    • 開始日が1日の場合: 当該月の月末までに自動的に選択します。

    • 開始日が1日でない場合: 来月の前日までに自動的に選択します。

    ノート

    例

    • 開始日が1月1日であれば、1月1日00時00分00秒から1月31日23時59分59秒を自動選択します。

    • 開始日が1月10日であれば、1月10日00時00分00秒から2月9日23時59分59秒を自動選択します。

  • Fixed 統合Flexボード又はFlexボードメニューでリアルタイム照会モード時に通知一覧ウィジェットでアップデートごとに通知音が発生する現象を修正

  • Fixed 利用状況 > 前払金メニューから一覧が表示されない問題を修正

Application​

  • Feature ダッシュボード > TXマップメニューからチャートのY軸の最大値を直接入力して調整できる機能を追加

  • Changed ダッシュボード > TXマップメニューでまたはボタンでY軸の最大値を調整する時、30秒を超えると10秒間隔でしか調整できない機能を改善

    • 10秒から50秒の区間は10秒間隔で移動

    • 50秒から300秒は50秒間隔で移動

    • 300秒以上は100秒間隔で移動

  • Fixed トレース分析ウィンドウのテーブルビュータブでSQLステップを確認する時、SQL文の置換変数またはバインド変数のいずれか一つでも解除すると、すべてが解除される問題を修正

Server​

  • New過去の性能データをより直観的に分析できるメトリクスエクスプローラーメニューを追加

    • サーバー インベントリメニューのカテゴリー情報を追加

    • メトリクスエクスプローラー機能案内モーダルを追加

    ノート

    この機能は、実験室 > メトリクスエクスプローラーメニューで確認できます。

  • Feature ダッシュボード > サーバーインベントリマップメニューのグループ要約パネルと詳細表示から現在時刻基準で24時間以内の進行中のイベントを照会できる進行中イベントの状況タブを追加

  • Changed ダッシュボード > サーバーインベントリマップメニューのグループパネル上段にグループ別のサーバー状態別個数と状態別イベント数を表示するように変更

  • Changed サーバーリストメニューでサーバーの状態を判別するロジックを変更

    • 既存：進行中のイベントのみサーバー状態を判別

    • 変更：実施中のイベント+単発性イベント(解消された通知オプションがオフになったイベント)にサーバーの状態を判別

      単発性イベントは、発生後のアクティブ時間(基本1時間)の間は、実施中のイベントと同様に処理します。

  • Changed サーバーリストメニューでサーバー状態アイコンをクリックすると表示されるサーバー状態画面で、スナップショットのチャートが有効なイベントカテゴリの場合にのみチャートを表示するように変更

  • Changed ダッシュボード > サーバーインベントリマップメニューのグループ要約パネルと詳細表示でサーバーインベントリテーブルを横スクロールする時、サーバー名のカラムが固定されるように改善

  • Changed Serverモニタリング製品紹介ページのコンテンツを変更

Kubernetes​

  • NewKubernetesのシステムコンポーネントの1つであるkube-schedulerに対するモニタリングデータを照会できるkube-scheduler ダッシュボードメニューを追加

    ノート

    この機能は、クラスター > kube-scheduler ダッシュボードメニューで確認できます。 クバネティスエージェント1.8.1バージョン 以降から対応します。

  • New時間の流れによるコンテナの状態を確認できるコンテナタイムラインメニューを追加

    ノート

    この機能は、コンテナー > コンテナタイムラインメニューで確認できます。

  • New時間の経過に伴うPod状態を確認できるPodタイムラインメニューを追加

    ノート

    この機能は、ワークロード > Podタイムラインメニューで確認できます。

  • Feature ダッシュボード > コンテナマップメニューの対象情報タップでPod開始性能分析情報も確認できる機能を追加

  • Feature クラスター > kube-apiserver ダッシュボードメニューのウィジェットが管理型Kubernetesの場合データなしに対する案内を追加

  • Changed次のメニューで要約分析機能をタブに分離

    • ダッシュボード > コンテナマップ
    • ダッシュボード > ノードマップ

  • Fixed ダッシュボード > コンテナマップメニューでフィルターの指標名が正しく表示されない問題を修正

  • Fixed コンテナー > コンテナイメージメニューでイメージ名に@が含まれる場合、タグが適切に表示されない問題を修正

  • Fixed クラスター > Ingress リストメニューで誤って設定されたルールが一部抜けている問題を修正

  • Fixedコンテナ要約分析でRequest、Limit設定メッセージのエラーを修正

  • Deprecated次のメニューパスで構成図をクリックすると、タブに移動する機能を削除

    • クラスター > ノードタイムライン

    • クラスター > ノードリスト > 詳細

Database​

共通​

Fixed 分析 > カウント推移比較メニューでアクティブセッションテーブルの行をクリックすると表示されるセッション詳細ウィンドウのデータが出力されない問題を修正

New IBM DB2モニタリングの商品発売

Oracle Pro​

  • Feature SQL詳細ウィンドウで統計値の表現デザインを変更

  • Fixed SQL詳細ウィンドウで直近1時間または直近1日を変更する時、最初のデータを選択するように修正

  • Fixed 分析 > SQL分析メニューからフィルター条件を初めて生成する時、条件のない状態で適用ボタンをクリックするとフィルターが追加される問題を修正

  • Fixed 分析 > SQL分析メニューから同じキーの条件を複数生成すると、AND条件であるにもかかわらずOR条件のように処理される問題を修正

MySQL​

Browser​

Preview 2.13.0.a2​

2025年01月09日

共通​

Fixed 統合Flexボード又はFlexボードメニューでダッシュボード変更後、時間セレクターを操作すると変更前のダッシュボードに初期化される問題を修正

Application​

  • Feature 統計 > トランザクションメニューでトランザクション件数を基準に計算した平均HTTP呼出し時間, 平均SQL時間, 平均SQLフェッチ時間のカラムを追加

    ノート

    変更事項については、以下を参照してください。

    カラム名の変更

    • 平均HTTP呼出し時間 → HTTP呼出し当りの平均時間：HTTP呼び出しの合計時間/トランザクションで発生したHTTP呼び出しの合計件数

    • 平均SQL時間 → SQL当りの平均時間：SQLの合計時間/トランザクションで発生したSQLの合計件数

    • 平均SQLフェッチ時間 → SQLパッチ当りの平均時間：SQLパッチの合計時間/トランザクションで発生したSQLパッチの合計件数

    新たに追加された平均値のカラム項目

    • 平均HTTP呼出し時間：HTTP呼び出しの合計時間/トランザクションの発生件数

    • 平均SQL時間: 총 SQL 시간 / 트랜잭션 발생 건수

    • 平均SQLフェッチ時間：SQLパッチの合計時間/トランザクションの発生件数

  • Changed サイトマップ > Netstatトポロジーメニューの初期画面に案内メッセージを追加

  • Fixed ダッシュボード > アプリケーションメニューでエージェントフィルターの適用事項がアクティブTXポップアップウィンドウに適用されない問題の修正

    ノート

    アクティブTXポップアップウィンドウはアクティブTXスピードチャートウィジェットとアクティブTXウィジェットをクリックすると確認できます。

  • Fixed 分析 > スタックメニューでチェックボックスが一部切られて表示される問題を修正

  • Fixed 分析 > TX検索メニューでOSカラムの内容が表示されない問題を修正

  • Fixed組織を選択した状態で分析 > TX検索メニューに入る時、時間セレクターで発生するエラーを修正

Server​

  • Feature サーバーリスト > サーバー インベントリメニューでサーバーのアクティブ状態を表示するデフォルトカラムを追加

  • Changed サーバーリスト > サーバー インベントリメニューでOSタイプカラムにオペレーティングシステム別のアイコンを表示

  • Fixed サーバーリスト > サーバー インベントリメニューのカラム設定でサーバー名カラムのチェックを解除できる問題を修正、基本カラムに変更

ノート

基本カラムはカラム設定で追加または削除することはできず、順序を変更することもできません。

Database​

共通​

  • Feature ダッシュボード > マルチインスタンスメニューでSession Tableウィジェットのオプションをプリセットに保存する機能を追加

    プリセットに保存されるオプションは次のとおりです。

    • カラム選択機能で選択したテーブルカラム

    • ユーザーが設定した個別カラムの幅

    • テーブルに適用したフィルター

  • Feature 管理 > クラウド設定メニューでAWSリージョン選択のオプションを追加

    ノート

    適用プラットフォーム：PostgreSQL、Oracle、MySQL、SQL Server

  • Changed ダッシュボード > インスタンスリストメニューでテーブルのヘッダーに指標名を表示するように変更

  • Changed ダッシュボード > インスタンスモニタリングメニューでチャート領域の高さが小さくなるとスクロールが使えるように変更

  • Changed 管理 > エージェントアップデートメニューでWindowsオペレーティングシステムを使用する環境についての案内メッセージを追加

  • Deprecated 管理 > エージェントインストールメニューでXOSエージェントの実行権限付与のコマンドを削除

Oracle Pro​

SQL Server​
