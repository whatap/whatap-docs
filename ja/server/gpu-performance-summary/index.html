GPUパフォーマンスサマリー

ホーム画面 > プロジェクト選択 > サーバーリスト > GPUパフォーマンスサマリー

システムに接続されたGPUの状態およびパフォーマンス指標をリアルタイムでモニタリング・管理できます。 物理GPUとMIGインスタンスの情報を同時に確認でき、各項目を並び替えたりフィルターして効率的に使用できます。

ノート

サポートエージェントバージョン

Linuxでのみサポートされており、エージェントバージョン 2.9.3 以上が必要です。

基本画面案内​

Live をクリックすると、リアルタイムデータの表示・一時停止ができます。 検索ボックスでGPU情報を検索できます。

データリスト内の項目名をクリックすると、昇順または降順に並べ替えることができます。

  • Total: 全GPUインスタンス数（Physical + MIG含む）
  • Physical: 実GPUデバイス数
  • MIG: MIG（Multi-Instance GPU）インスタンス数

GPU情報​

カラム名説明
Status該当GPUインスタンスの現在の状態（例：Inactive、N/A）
HostNameGPUが接続されているサーバーのホスト名
GPU Index物理GPUまたはMIGインスタンスの固有インデックス（MIGインスタンスは 0/6/0 などの形式で表示）
Model NameGPUモデル名（例：NVIDIA A100-SXM4など）
GPU TypePhysical または MIG のうち、該当インスタンスの種類

GPUパフォーマンス指標​

カラム名説明
GPU_Util (%)GPU使用率（%）
- 物理GPUでのみ表示され、MIGインスタンスは除外
Encoder_Util (%)ハードウェアエンコーダ使用率（%）
- 物理GPUでのみ表示
Decoder_Util (%)ハードウェアデコーダ使用率（%）
- 物理GPUでのみ表示
GR_Engine_Active_Util (%)GPUのグラフィックエンジンまたはコンピュートエンジンがアクティブな時間の割合
- GPU全体の使用率を測定するための指標で、MIG環境ではインスタンス間のオーバーヘッドも物理GPU使用率に反映されて表示されます
SM_Active_Util (%)SMで少なくとも1つのワープ（Warp）が実行された時間の割合
- GPUではスレッドを32個単位のWarpでまとめて実行します
SM_Occupancy (%)SMが実行可能な最大Warp数に対する実際のWarp実行数の割合
- GPUではスレッドを32個単位のWarpでまとめて実行します
Tensor_Core_Util (%)Tensorコアがアクティブだった時間の割合
Memory_Copy_Util (%)GPUのメモリコピーエンジン（memory copy engine）の使用率（%）
- GPUとホスト間、またはGPU内部でのメモリ転送がどの程度行われているかを示す値
- Physical GPUでのみ表示され、MIGインスタンスでは表示されません
DRAM_Active_Util (%)DRAMのRead/Write活用率
- Physical GPUでのみ表示され、MIGインスタンスでは表示されません
FP64_Compute_Util (%)FP64（64ビット浮動小数点）演算パイプのアクティブ時間の割合
FP32_Compute_Util (%)FP32（32ビット浮動小数点）演算パイプのアクティブ時間の割合
FP16_Compute_Util (%)FP16（16ビット浮動小数点）演算パイプのアクティブ時間の割合
BAR1_Total_Memory (Bytes)BAR1メモリ総量
- BAR1メモリはGPUとCPU間のデータ転送時に使用されます
BAR1_Used_Memory (Bytes)使用中のBAR1メモリ容量
- BAR1メモリはGPUとCPU間のデータ転送時に使用されます
BAR1_Free_Memory (Bytes)残りのBAR1メモリ容量
- BAR1メモリはGPUとCPU間のデータ転送時に使用されます
FB_Total_Memory (Bytes)フレームバッファの総メモリ容量
FB_Free_Memory (Bytes)フレームバッファの使用可能メモリ容量
FB_Used_Memory (Bytes)フレームバッファで使用中のメモリ容量
FB_Reserved_Memory (Bytes)予約されたフレームバッファメモリ容量
FB_Memory_Usage (%)フレームバッファメモリ使用率（%）
ECC_SBE_TotalECC SBE（Single Bit Error）累積合計
ECC_DBE_TotalECC DBE（Double Bit Error）累積合計
GPU_Temperature (°C)GPUの現在の温度
Power_Usage (W)GPUが現在消費している電力量
Performance_State (P)現在のGPUパフォーマンスステート（P-State）を0～15の数値で表示
- P0～P15の値を持ち、P0が最速
Fan_Speed (%)GPUファンの現在の動作速度比率
SM_Clock (MHz)SM（Streaming Multiprocessor）の現在のクロックスピード
Memory_Clock (MHz)メモリの現在のクロックスピード
Video_Clock (MHz)ビデオ処理を担当するクロック速度
PCIE_TX (Bytes/s)GPUがPCIeインターフェースを介して送信したデータ量
- 物理GPUのみ値が表示され、MIGはN/Aと表示され収集されない場合があります
PCIE_RX (Bytes/s)GPUがPCIeインターフェースを介して受信したデータ量
- 物理GPUのみ値が表示され、MIGはN/Aと表示され収集されない場合があります
NVLink_TX (Bytes/s)NVlinkを介して送信したデータ量
NVLink_RX (Bytes/s)NVlinkを介して受信したデータ量

CSVダウンロード​

検索結果をCSVファイルとしてダウンロードできます。

カラム設定​

画面右上の カラム設定 ボタンをクリックすると、カラム設定 ウィンドウが開きます。
このウィンドウで、テーブルに表示するカラムを選択できます。

  • 全選択: すべてのカラムを選択

  • デフォルト順にリセット: カラム順を初期状態に戻す

  • デフォルト選択にリセット: 選択状態を初期設定に戻す

ノート

選択したカラムはブラウザのクッキーに保存され、ページを更新しても保持されます。
クッキーが削除された場合や設定エラーが発生した場合は、初期状態にリセットされます。

  1. カラム設定 ウィンドウで、GPU情報 および GPUパフォーマンス指標 の表示カラムを選択してください。

    • カラムはグループ単位で選択したり、表示順を調整することができます。
    注意

    GPU情報のうち、Status と HostName は削除できません。

  2. 適用 ボタンをクリックして、設定を保存してください。

フィルターを追加する​

フィルター条件を追加することで、目的のGPUを素早く見つけることができます。

  1. フィルター 検索バーをクリックすると、フィルターを追加 ウィンドウが表示されます。

  2. フィルターを追加 で、GPU条件（フィルターキーと条件）を選択し、適用 ボタンをクリックします。

GPUを比較する​

複数のGPUを選択すると、GPUリストの下にある 比較 パネルで、チャートによる視覚的な比較が可能になります。
直近10分間のデータトレンドを追跡し、パフォーマンスの異常やボトルネックを特定できます。

  • 比較できる項目は最大20個までです。
