HTTPステータスのコードタイプ

WhaTap URLモニタリングを通じてウェブサービスの障害有無を確認できます。 ウェブサービスからのリターンされるステータスコードが4xx、5xxの場合、警告通知を送信します。 このようなステータスコードは、HTTPステータスコード(HTTP Status Code)と呼びます。 代表例として"404 Not Found"があります。

ステータスコードは3桁の数字で構成され、最初の桁は1から5までです。 最初の桁が4または5の場合は、正常な状況ではないため、サイト管理者が直ちに認知する必要があります。

  • 1xx(情報): リクエストを受けており、プロセスは継続中です。

  • 2xx(成功): リクエストは正常に受付、認識、処理されました。

  • 3xx(リダイレクション): リクエストを完了するには、追加の作業が必要です。

  • 4xx(クライアントエラー): リクエストの文法が正しくないか、リクエストを処理できません。

  • 5xx(サーバーエラー): サーバーは有効なリクエストの処理に失敗しました。

ノート

HTTPステータスコードの詳細については、次のリンクを参照してください。

  • W3 RFC 2616ステータスコードのまとめ

  • モジラ財団ステータスコードのまとめ

  • IETFのRFC 2616

1XX: Information responses​

ステータスコードが'1'で始まる場合は、サーバーがリクエストを受けており、サーバーに接続されたクライアントが作業を続行する必要があることを意味します。 このコードはHTTP 1.0には対応していません。

  • 100 Continue

    進行中であることを意味するレスポンスコードです。 現在までの進行状況に問題はなく、クライアントがリクエストを継続するか、すでにリクエストを完了した場合は無視してもよいことを通知します。

  • 101 Switching Protocol

    101レスポンスコードは、クライアントが送信したアップグレードリクエストヘッダーへのレスポンスとして送信されます。 このコードは、サーバーがクライアントのUpgradeリクエストを受け入れ、プロトコルを変更することを通知します。 主にWebSocketプロトコルの切り替えに使用されます。

  • 102 Processing(WebDAV)

    このレスポンスコードは、サーバーがリクエストを受信し、処理中だが、まだ最終レスポンスを提供できないことを意味します。

2XX: Successful responses​

  • 200 OK

    リクエストが正常に処理されました。 リクエストに対するレスポンスとして情報が返されます。

  • 201 Created

    リクエストは正常に処理され、その結果で新しいリソースが作成されました。 このレスポンスは、主にPOSTリクエストや一部のPUTリクエスト以降に使用されます。

  • 202 Accepted

    リクエストは受信されましたが、まだ処理されていません。 このレスポンスは、リクエスト処理が非同期的に行われ、その後HTTPレスポンスで結果がリターンされたことを明示しません。 別のプロセスがリクエストを処理しているか、サーバーがリクエストをバッチプロセスとして処理しているときに使用されます。

  • 203 Non-Authoritative Information

    返されたメタ情報はオリジンサーバーのものではなく、ローカルまたはサードパーティのコピーから収集されたことを意味します。 この場合、200 OKのレスポンスが優先される必要があります。

  • 204 No Content

    リクエストを正常に処理しましたが、返すコンテンツがありません。 ただ、ヘッダーは有効である可能性があり、ユーザーエージェントはキャッシュされたヘッダーを最新情報に更新できます。

  • 205 Reset Content

    リクエストが完了した後、ユーザーエージェントがリクエストを送信したドキュメントビューをリセットする必要があることを意味します。

  • 206 Partial Content クライアントがスコープヘッダーを介して部分コンテンツをリクエストしたときに使用されます。 ウェブサーバーは'206 Partial Content'レスポンスコードとともに、Range ヘッダーに指定されたデータ部分を送信します。

  • 207 Multi-Status

    マルチステータスレスポンスは、複数のリソースの複数のステータスコードを含む状況で使用されます。 WebDAV(Web Distributed Authoring and Vesioning)で主に使われます。

  • 208 Already Reported

    Prostat(PropertyとStatusの合成語) レスポンス属性として、同じコレクションにバインディングされた複数の内部メンバーを繰り返し並べることを避けるために使用されます。 WebDAV(Web Distributed Authoring and Vesioning)で使われます。

  • 226 IM Used (HTTP Delta encoding)

    サーバーがGETリクエストを処理し、レスポンスが1つ以上のインスタンス操作が適用された現在のインスタンスを含んでいることを通知します。

3XX: Redirection messages​

  • 300 Multiple Choice

    リクエストに対して1つ以上のレスポンスが可能であることを意味します。 ユーザーエージェントまたはユーザーは、それらのいずれかを選択する必要があります。 標準化されたレスポンスの選択方法はありません。

  • 301 Moved Permanently

    リクエストしたリソースのURIが永久に変更されたことを意味します。 新しいURIがレスポンスに提供されることもあります。

  • 302 Found

    リクエストしたリソースのURIが一時的に変更されたことを意味します。 新しいURIは後で提供されることがあり、クライアントはその後のリクエストでも必ず同じURIを使用する必要があります。

  • 303 See Other

    サーバーがクライアントにリクエストしたリソースを他のURIからGETリクエストを通じて取得するように指示します。

  • 304 Not Modified

    キャッシュ目的で使用されます。 クライアントにレスポンスが修正されていないことを通知し、クライアントはキャッシュされたバージョンを引き続き使用できます。

  • 305 Use Proxy

    以前のHTTP仕様で定義されており、リクエストされたレスポンスが必ずプロキシを通じて接続する必要があることを通知します。 プロキシのin-band設定に対するセキュリティ上の懸念により、使用が徐々に減少しています。

  • 306 Unused

    現在は使用されておらず、今後の使用のために予約された状態です。 HTTP 1.1以前のバージョンで使用されました。

  • 307 Temporary Redirect

    クライアントがリクエストしたリソースが他のURIにあり、以前のリクエストと同じメソッドを使用してリクエストする必要があることを意味します。 302 Foundと似ていますが、307 Temporary Redirectの場合、使用されたHTTPメソッドを変更してはなりません。 最初のリクエストがPOSTなら、2番目のリクエストもPOSTにする必要があります。

  • 308 Permanent Redirect

    リソースが永続に異なるURIに位置することを意味します。 301 Moved Permanentlyと似ていますが、308 Permanent Redirectの場合、使用されたHTTPメソッドを変更してはなりません。 最初のリクエストがPOSTなら、2番目のリクエストもPOSTにする必要があります。

4XX: Client error responses​

  • 400 Bad Request

    このレスポンスは、リクエストの文法が正しくないため、サーバーがリクエストを理解できないことを意味します。

  • 401 Unauthorized

    HTTP標準では'未承認(unauthorized)'を明確にしているが、実際にこのレスポンスは'非認証(unauthenticated)'を意味します。 要求されたレスポンスを受け取るために、クライアントは認証を行う必要があります。

  • 402 Payment Required

    このレスポンスコードは、後で使用されるために予約されています。 デジタル決済システムに使用するためのものですが、現在は使用されていません。

  • 403 Forbidden

    クライアントは、コンテンツにアクセスする権限がありません。 サーバーは、リクエストを拒否するための適切なレスポンスを送信します。 401 Unauthorizedと異なる点は、サーバーがクライアントを認識していることです。

  • 404 Not Found

    サーバーがリクエストしたリソースが見つかりません。 ブラウザーでは知られていないURLを意味します。 APIでは、終点が適切ですが、リソースが存在しないことを示します。 サーバーは、認証されていないクライアントからリソースを非表示にするために、このレスポンスを403の代わりに送信する場合もあります。 このレスポンスコードは、ウェブ上で最も広く知られています。

  • 405 Method Not Allowed

    リクエストしたメソッドはサーバーで認識できますが、使用が禁止されています。 例えば、特定のAPIでのリソースの削除が禁止されることがあります。 必須メソッドであるGETとHEADは削除できず、このエラーコードを返すことはできません。

  • 406 Not Acceptable

    サーバーからユーザーエージェントが指定した規格に適合するコンテンツを見つけられなかったときに、このレスポンスを送信します。

  • 407 Proxy Authentication Required

    このレスポンスは、401 Unauthorizedと似ていますが、プロキシ認証が必要であることを意味します。

  • 408 Request Timeout

    サーバーがアイドル状態の接続を終了しようとしているときに、このレスポンスを送信します。 Chrome、Firefox 27+、またはIE 9などのブラウザーは、HTTP事前接続メカニズムを使用してこのレスポンスを受信できます。 一部のサーバーは、このメッセージを送信せずに切断する場合があります。

  • 409 Conflict

    リクエストが現在のサーバーのステータスと衝突したときにこのレスポンスを送信します。

  • 410 Gone

    リクエストしたコンテンツがサーバーから永久に削除され、使用できなくなったときにこのレスポンスを送信します。 クライアントは、キャッシュとリソースに対するリンクを削除する必要があります。

  • 411 Length Required

    Content-Lengthヘッダーフィールドが定義されていないリクエストをサーバーが拒否します。

  • 412 Precondition Failed

    クライアントのヘッダーにある前提条件がサーバーの前提条件と一致しない場合にこのレスポンスを送信します。

  • 413 Payload Too Large

    リクエストエンティティがサーバーで定義した制限を超えています。 サーバーは接続を切断するか、Retry-Afterヘッダーフィールドを含めてレスポンスできます。

  • 414 URI Too Long

    クライアントがリクエストしたURIがサーバーで処理できる長さを超えています。

  • 415 Unsupported Media Type

    リクエストしたメディア形式がサーバーで対応していないため、サーバーはそのリクエストを拒否します。

  • 416 Requested Range Not Satisfiable

    Rangeヘッダーフィールドにリクエストした範囲を満たしません。 リクエストした範囲が対象URIのデータサイズを超えている可能性があります。

  • 417 Expectation Failed

    このレスポンスは、Expectリクエストヘッダフィールドの要件がサーバーで満たされていないときに送信されます。

  • 418 I'm a teapot

    サーバーは、コーヒーをティーポットで沸騰させることを拒否します。 HTCPCPプロトコルステータスコードの1つです。

  • 421 Misdirected Request

    サーバーに誘導されたリクエストは、レスポンスを生成できません。 これは、サーバがリクエストURIと接続されたスキームと権限を構成し、レスポンスを生成できないときに送信されます。

  • 422 Unprocessable Entity (WebDAV)

    リクエストには異常はありませんが、文法上の誤りにより従うことはできません。

  • 423 Locked (WebDAV)

    リクエストしたリソースがロックされているため、アクセスできません。

  • 424 Failed Dependency (WebDAV)

    以前のリクエストが失敗したため、現在のリクエストも失敗しました。

  • 426 Upgrade Required

    サーバーは現在プロトコルを使用してリクエストを処理しませんが、クライアントが他のプロトコルにアップグレードすると処理できます。 サーバーは、必要なプロトコルをUpgrade ヘッダーに含めてレスポンスします。

  • 428 Precondition Required

    サーバーはリクエストが条件的である必要があります。 これは、クライアントがリソースをGETした後に修正し、PUTでサーバに戻す間、サードパーティによってサーバの状態が変更される'アップデート喪失'を防止するためです。

  • 429 Too Many Requests

    ユーザーが指定された時間内にあまりにも多くのリクエストを送信したことを意味します。 ("rate limiting")

  • 431 Request Header Fields Too Large

    リクエストしたヘッダーフィールドが大きすぎて、サーバーがリクエストを処理できません。 リクエストを減らして再送信する必要があります。

  • 451 Unavailable For Legal Reasons

    リクエストしたリソースが政府によって検閲されたウェブページなど、違法であることを示します。

5XX: Server error responses​

  • 500 Internal Server Error

    サーバーに問題がありますが、正確な問題を明確に説明できないことを意味します。

  • 501 Not Implemented

    サーバーがリクエストを実行するために必要な機能に対応していないことを意味します。

  • 502 Bad Gateway

    サーバーがゲートウェイからの誤ったレスポンスを受信したことを意味します。 インターネット上のサーバーが他のサーバーから有効でないレスポンスを受けた場合に発生します。

  • 503 Service Unavailable

    サーバーがリクエストを処理する準備ができていないことを意味します。 これは、主にサーバーのメンテナンス中または過負荷がかかったときに発生します。 このレスポンスとともにユーザーフレンドリーなページを提供する必要があり、可能な場合はRetry-Afterヘッダーにサービスの復旧予想時間を含める必要があります。

  • 504 Gateway Timeout

    サーバーがゲートウェイとして機能し、適時にレスポンスを得られなかったことを意味します。 これは主に、サーバー間のネットワークの問題が原因で発生します。

  • 505 HTTP Version Not Supported

    これは、クライアントがサーバーで対応していないHTTPバージョンをリクエストしたことを意味します。 ほとんどのウェブ ブラウザは、サーバーがHTTP 1.x バージョンに対応していると想定しています。 最新バージョンのプロトコルを使用しないことは、セキュリティと性能上の理由で推奨されません。

  • 506 Variant Also Negotiates

    サーバーに内部構成エラーが発生し、リクエストに対する透明なコンテンツ交渉が循環参照につながるときに発生します。

  • 507 Insufficient Storage

    サーバーがリクエストを保存する十分なスペースを持っていない場合に発生します。 これは主にWebDAVで発生します。

  • 508 Loop Detected (WebDAV)

    サーバーがリクエストの処理中に無限ループを検出したときに発生します。

  • 510 Not Extended

    サーバーがリクエストを実行するために追加の拡張が必要な場合に発生します。

  • 511 Network Authentication Required

    クライアントがネットワークにアクセスするために認証が必要であることを意味します。
