XOS2 지표 폐기 안내

데이터베이스 서버에 XOS 에이전트를 설치하면 서버의 자원 상태를 모니터링할 수 있습니다. 수집하는 주요 지표는 다음 문서를 참조하세요.

  • 인스턴스 모니터링 - Process information 탭을 확인하세요.
  • 메트릭스 지표 - XOS 카테고리의 지표

XOS 지표 변경 사항​

데이터베이스 모니터링에서는 CPU 사용량과 관련해 CPU(xos), CPU(xos2) 지표 두 가지를 제공하고 있었습니다. DBX 에이전트 1.7.0 버전 이상에서는 두 지표가 하나의 단위로 통합될 예정입니다.

변경 전 지표 이름변경 후 지표 이름단위
cpu(xos2)
cpu(xos)		cpu(xos)사용률(%)
cpu_idle(xos2)
cpu_idle(xos)		cpu_idle(xos)%
cpu_sys(xos2)
cpu_sys(xos)		cpu_sys(xos)%
cpu_user(xos2)
cpu_user(xos)		cpu_user(xos)%
cpu_iowait(xos2)
cpu_iowait(xos)		cpu_iowait(xos)%

DBX 에이전트 1.7.0 버전 이하에서 cpu(xos) 지표의 값이 9000이라면 cpu(xos2) 지표의 값은 90으로 수집되었습니다.

DBX 에이전트를 1.7.0 버전으로 업데이트한 후에 기존 지표로 경고 알림을 설정했거나 대시보드를 구성했다면 지표 설정을 변경해야 합니다. 경고 알림 > 이벤트 설정 메뉴에서 설정한 임계값은 자동 전환됩니다. 이미 설정한 경고 알림 중 위 5가지 지표를 사용한 알림이 제대로 전환됐는지 확인하세요. 다음 내용을 참고하세요.

DBX 에이전트 업데이트하기​

관리 > 에이전트 업데이트 메뉴에서 1.7.0 버전 이하의 에이전트를 업데이트하세요.

에이전트 업데이트에 대한 자세한 내용은 다음 문서를 참조하세요.

변경된 지표 확인하기​

DBX 에이전트 업데이트 후 경고 알림 > 이벤트 설정 메뉴의 데이터베이스 탭에서 cpu(xos2) 지표가 cpu(xos) 지표로 변경됐는지 확인하세요.

  • DBX 에이전트 업데이트 전

  • DBX 에이전트 업데이트 후

사용자가 설정한 메트릭스 알림과 대시보드의 설정들은 직접 수정해야 합니다.

  • 경고 알림 > 이벤트 설정 메뉴의 메트릭스 탭에서 cpu(xos2) 지표로 설정한 이벤트를 cpu(xos) 지표로 변경하세요.

  • 다음 메뉴 경로에서 cpu(xos2) 지표를 cpu(xos) 지표로 변경해야할 설정이 있는지 확인하세요.

    • 대시보드 > 인스턴스 모니터링
    • 대시보드 > Flex 보드

  • 3월 정기 서비스 배포(2024.3.27 18:00) 후 대시보드에서 cpu(xos) 지표가 기존과 다르게 표시된다면 DBX 에이전트를 1.7.0 버전 이상으로 업데이트하세요. 에이전트 업데이트에 대한 자세한 내용은 다음 문서를 참조하세요.

  • 도움이 필요한 고객의 경우 와탭 모니터링 서비스 화면 오른쪽 위에 채팅 문의를 통해서 지원받을 수 있습니다.
